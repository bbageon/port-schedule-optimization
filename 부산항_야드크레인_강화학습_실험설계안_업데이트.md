# 부산항 야드크레인 강화학습 최적화 실험 설계안 업데이트

## 0. 업데이트 목적

본 문서는 기존 `부산항 야드크레인 강화학습 최적화 실험 설계안`에 다음 내용을 추가·보완한 업데이트본이다.

1. 예측 차량 도착정보·COPINO·Q-learning 기반 야드크레인 스케줄링 선행연구를 바탕으로 **MDP·비용 최소화 설계 구조**를 부산항 야드크레인 작업순서 최적화 문제에 적용
2. 기존 Baseline, Exp-1, Exp-2, Exp-3 구조에 **Exp-4: 다중 야드크레인 협조 최적화** 추가
3. Exp별로 강화학습 Q-learning 적용 시 필요한 **State / Action / Reward**를 구체화
4. “야드크레인 배치”의 의미를 물리적 레일 배치가 아닌 **운영상의 레인–야드크레인 동적 할당·작업순서·재조작 선처리 최적화**로 명확화
5. 정보 확보 시점, 제어범위, 다중 크레인 협조범위를 분리하여 실험효과를 식별할 수 있도록 보완
6. 탄소는 직접 최적화 항목이 아니라 대기·이동·재조작 감소에 따라 발생할 수 있는 **운영단계 기대효과**로 구분

## 1. 실험 목적

ETA는 부산항에서 제공되는 외생 입력값으로 가정하며, 본 연구는 ETA 생성모형이나 예측 정확도를 검증하지 않는다. 본 연구의 목적은 제공된 ETA와 사전 반출입정보를 야드크레인 작업순서 최적화에 활용했을 때 차량 대기시간·크레인 이동거리·재조작 횟수가 얼마나 개선되는지를 검증하는 데 있다.

핵심 검증 질문은 다음과 같다.

> 차량 정보를 더 이른 시점에 확보할수록 야드크레인 작업순서 최적화 효과가 커지는가?

기존 실험은 정보 확보 시점에 따라 Exp-1, Exp-2, Exp-3으로 구성한다. 여기에 부산항 스마트항만 전환 관점에서 **동일 블록 또는 실제 서비스영역 내 다중 야드크레인 간 상호 영향**을 반영한 Exp-4를 추가한다.

최종적으로 본 실험은 다음 알고리즘을 고안하는 것을 목표로 한다.

> **부산항 스마트항만 전환을 위한 차량 사전정보 활용 야드크레인 작업순서·협조 최적화 알고리즘**

여기서 “배치”는 야드크레인 레일을 물리적으로 재배치한다는 의미가 아니다. 본 연구에서의 배치는 다음을 의미한다.

```text
- 물리적으로 선택 가능한 작업차선 또는 작업 큐 중 어디에 할당할지
- 어느 야드크레인이 어떤 작업을 먼저 처리할지
- 어떤 컨테이너를 차량 도착 전에 선처리할지
- 본선작업과 외부트럭 작업의 우선순위를 어떻게 조정할지
- 여러 야드크레인이 서로 병목을 만들지 않도록 어떻게 협조할지
```

## 2. 기존 연구 기반 적용 방향

### 2.1 직접 선행연구와 본 연구의 위치

본 연구의 핵심 질문은 항만 운영 분야의 직접 선행연구와 연결된다.

| 선행연구 | 핵심 내용 | 본 연구와의 관계 |
|---|---|---|
| Kim, Lee and Hwang (2003) | 연속적으로 도착하는 외부트럭을 대상으로 학습 기반 YC 작업순서 규칙 비교 | 동적 YC 작업선택에 학습정책을 사용할 수 있다는 근거 |
| Guo et al. (2011) | 예측 차량 도착시각을 활용하여 평균 차량 대기시간을 최소화하는 YC dispatching | Exp-2~3의 가장 직접적인 이론 근거 |
| Fotuhi et al. (2013) | YC 작업자를 Q-learning agent로 모델링하여 트럭 대기시간 최소화 | Q-learning 기반 작업선택의 직접 근거 |
| 황성범·정석재·윤성욱 (2019) | 실제 부산컨테이너터미널 레이아웃·데이터와 COPINO 정보를 사용하여 공차이동·간섭대기·트럭지연 감소 검증 | 부산항 적용성과 사전정보 활용의 직접 근거 |

따라서 본 연구의 신규성은 제공된 차량 사전정보를 Q-learning 작업정책에 입력하는 것 자체가 아니라 다음에 둔다.

```text
- 정보 확보시점과 정보 정확도의 효과를 분리해 검증
- 현행 작업규칙 대비 학습형 priority-rule 선택정책의 추가효과 검증
- 본선작업과 외부트럭 작업이 공존하는 상황에서 서비스 수준 보호
- 동일 블록 또는 실제 서비스영역 내 다중 크레인의 간섭·안전 제약 반영
- 대기시간 개선과 이동·재조작·운영효율의 동반 개선 여부를 개별 KPI로 검증
```

### 2.2 MDP·비용 최소화 구조

본 연구는 현재 운영상태를 관측하고, 실행 가능한 작업 우선순위 또는 다음 작업을 선택한 뒤, 그 결과로 발생한 비용을 이용해 장기 누적비용이 낮아지는 정책을 학습한다.

```text
현재 상태 관측
→ 실행 가능한 행동 생성 및 제약 위반 행동 제거
→ 작업 우선순위 또는 다음 작업 선택
→ 작업 수행시간 동안 발생한 대기·이동·재조작·지연 비용 계산
→ 다음 의사결정 시점에서 상태 갱신
→ 장기 누적비용이 낮아지는 정책 학습
```

### 2.3 본 연구에서 최소화할 비용

본 연구의 직접 최적화 대상은 야드 운영비용이다.

```text
Yard Operation Cost =
정규화된 외부트럭 누적 대기비용
+ 장기대기 또는 상위대기 패널티
+ 정규화된 야드크레인 이동·작업비용
+ 재조작 비용
+ 대상 터미널에 실제 존재하는 내부 이송장비 대기비용
+ 본선작업 지연 패널티
+ 크레인 간 간섭·작업불균형 비용
+ 작업계획 변경 패널티
```

분, m, 횟수, 지연시간처럼 단위가 다른 항목은 그대로 더하지 않고 Baseline 평균, SLA 또는 운영 허용치로 나누어 무차원화한다. 안전거리, 비통과, 장치 가능 슬롯, 장비 도달영역과 같은 조건은 비용항이 아니라 Hard Constraint와 Action Masking으로 처리한다.

구현 방식은 두 가지 중 하나로 정리할 수 있다.

```text
방식 1. 논문처럼 Cost를 직접 최소화
Q(s, a) = 해당 상태에서 행동 a를 선택했을 때의 장기 누적비용
가장 작은 Q값을 갖는 행동 선택

방식 2. 일반 강화학습 구현 방식
Reward = -Cost
누적 Reward를 최대화하도록 학습
```

본 문서에서는 설명의 직관성을 위해 `Reward = -Cost` 구조를 사용한다.

## 3. 전체 실험 구성

| 구분 | 정보 확보 시점 | 추가되는 정보/파라미터 | 최적화 범위 | 검증 목적 |
|---|---|---|---|---|
| Baseline | 현재 방식 | 블록 도착 차량 중심 처리 | 도착순 또는 현장 판단 | 기준선 확보 |
| Exp-1 | 블록 진입 시점 | 블록 대기열, 크레인 위치, 컨테이너 위치 | 단일 블록 내부 작업순서 | 블록 내 순서 변경 효과 검증 |
| Exp-2 | 게이트 진입 시점 | 게이트 진입시각, 블록까지 거리·소요시간 | 게이트–야드 연계 | 블록 도착 전 선제 조정 효과 검증 |
| Exp-3 | 사전 반출입정보 시점 | 부산항 제공 ETA, 반출입정보, 컨테이너 위치, 미래 작업량 | 차량 도착 전 사전 최적화 | 제공된 차량 사전정보의 활용효과 검증 |
| Exp-4 | 실제 서비스영역 내 다중 야드크레인 환경 | 크레인 도달영역, 비통과·안전거리, 다중 크레인 상태, 실제 이송장비 대기 | 동일 블록 또는 운영구역 협조 최적화 | 다중 야드크레인 협조 알고리즘 효과 검증 |

Exp-1~3은 정보 확보시점의 선제성 효과를 검증하는 주 실험이다. Exp-4는 정보를 더 일찍 확보하는 단계가 아니라 **단일 야드크레인 작업순서 최적화를 실제 서비스영역 내 다중 야드크레인 협조 문제로 확장하는 별도 실험**이다.

따라서 `Baseline < Exp-1 < Exp-2 < Exp-3 < Exp-4`를 사전 결론으로 두지 않는다. 제공된 사전정보를 활용한 선처리와 포지셔닝이 실제 대기·이동·재조작 개선으로 이어지는지는 검증가설로 취급한다.

핵심 질문인 “정보 확보시점 효과”를 식별하기 위해 정보시점과 제어범위를 다음처럼 구분한다.

| 실험요인 | 수준 예시 |
|---|---|
| 정보 확보시점 | 블록 진입 / 게이트 진입 / 사전 반출입정보 |
| 제어범위 | 순서변경만 / 사전 포지셔닝 / 사전 재조작 |
| 협조범위 | 단일 YC / 동일 서비스영역 내 다중 YC |

## 4. Baseline: 현재 방식 / 블록 도착순 기반 처리

### 4.1 개념

Baseline은 대상 터미널의 현행 YC 작업지시 또는 dispatching rule을 우선 재현한다. 실제 규칙을 확보하지 못한 초기 PoC에서만 도착순을 단순 기준선으로 사용하며, 이 경우 이를 “현재 운영”으로 단정하지 않는다.

```text
차량 블록 도착
→ 블록별 작업 대기열 등록
→ 도착순 또는 현장 판단 기반 처리
→ 야드크레인 이동 및 상·하차 수행
```

### 4.2 역할

Baseline은 강화학습 적용 효과를 판단하기 위한 비교 기준이다.

### 4.3 비교 기준 지표

- 기사 평균대기시간
- 중앙값 대기시간
- 90/95퍼센타일 대기시간
- 장기대기 차량 비율
- 야드크레인 총 이동거리
- 야드크레인 빈 이동거리
- 재조작 횟수
- 시간당 처리량
- 본선작업 지연 여부
- 작업순번 변경 횟수

## 5. Q-learning 공통 설계 원칙

### 5.1 이벤트 기반 MDP/SMDP 정의

본 연구에서 야드크레인 작업순서 최적화 문제는 다음과 같은 MDP로 정의한다.

| MDP 요소 | 야드크레인 적용 정의 |
|---|---|
| State | 현재 야드 운영상태: 차량, 컨테이너, 크레인 가용시각·위치, 작업 큐, 본선작업, 실제 이송장비 대기 상태 |
| Action | 실행 가능한 priority rule 선택 또는 다음 처리 작업·재조작 작업 선택 |
| Reward | 대기시간·이동거리·재조작·본선지연·레인혼잡 등 비용의 음수값 |
| Transition | 가변 작업시간 경과 후 변화한 대기열, 크레인 위치·가용시각, 컨테이너 위치, 장비 대기 상태 |
| Policy | 각 상태에서 장기 비용이 가장 작아지는 행동 선택 규칙 |

야드크레인 작업은 이동거리·Tier·재조작 수에 따라 소요시간이 달라진다. 따라서 작업완료, 신규 긴급작업 발생, 장비가용 상태변화를 의사결정 시점으로 하는 이벤트 기반 SMDP가 기본이다. 고정 1~5분 주기를 혼용하지 않는다.

Exp-1~3의 비교가능성을 위해 핵심 비용함수는 동일하게 유지한다. 단계별로 달라지는 것은 정보와 허용 행동이며, 같은 결과를 여러 이름으로 중복 패널티하지 않는다.

```text
Core_Cost_t =
  wW    × [의사결정 구간의 외부트럭 queue-area / S_wait]
+ wTail × [장기대기 차량 queue-area / S_tail]
+ wMove × [(적재이동거리 + (1 + η_empty) × 빈 이동거리) / S_move]
+ wR    × [재조작 횟수 / S_rehandle]
+ wV    × [본선 및 내부이송장비 지연시간 / S_vessel]
+ wC    × [작업계획 변경 횟수 / S_change]

Reward_t = -Core_Cost_t
```

여기서 `S_*`는 Baseline 평균 또는 운영 허용치이며, `η_empty`는 빈 이동에 추가 가중을 주려는 경우에만 사용한다. 평균대기시간은 완료 차량만 집계하지 않고 의사결정 구간의 대기차량 수를 시간적분한 queue-area로 계산하여 종료시점 미처리 차량의 대기도 포함한다.

### 5.2 Q-learning 업데이트 구조

비용 최소화 관점에서는 다음처럼 이해할 수 있다.

```text
Q(s_t, a_t) ← (1 - α)Q(s_t, a_t)
              + α[Cost(s_t, a_t) + γ min_a Q(s_{t+1}, a)]
```

일반적인 reward 최대화 관점에서는 다음처럼 변환한다.

```text
Reward_t = -Cost_t

Q(s_t, a_t) ← (1 - α)Q(s_t, a_t)
              + α[Reward_t + γ max_a Q(s_{t+1}, a)]
```

### 5.3 학습 파라미터

| 파라미터 | 의미 | 초기 제안값 |
|---|---|---|
| α | 학습률 | Tabular PoC에서 0.05~0.20 탐색 후 검증자료로 선정; DQN 학습률과 구분 |
| γ | 미래 비용 반영 정도 | 기준 시간단위와 운영 horizon으로 산정. 가변시간은 `γ(τ)=exp(-ρτ)` 형태로 보정 |
| ε | 탐색 비율 | 오프라인 시뮬레이션에서만 초기값·감쇠율 튜닝; 평가·운영은 greedy 정책 사용 |
| Δt | 의사결정 주기 | 작업완료·신규 긴급작업·장비가용 이벤트 시점 |
| K | 상태에 포함할 후보 작업 수 | 실측 후보수 민감도 분석으로 선정. 최장대기·긴급·마감임박 작업은 강제 포함 |

### 5.4 Action 설계 원칙

초기 PoC에서는 “전체 후보 차량 중 정확히 한 대 선택” 방식보다 **작업 우선순위 정책을 행동으로 정의하는 Q-learning hyper-heuristic**이 적절하다.

이유는 다음과 같다.

```text
차량·컨테이너 수가 많아지면 행동 공간이 급격히 커진다.
Q-table 기반 Q-learning은 상태·행동 공간이 너무 커지면 학습이 어렵다.
따라서 초기에는 행동을 우선순위 정책 단위로 축소한다.
```

각 Action은 그 자체가 물리 작업이 아니라, 실행 가능한 작업집합에서 다음 작업을 결정하는 priority rule이다. `A8 작업순서 재배열`처럼 결과가 불명확한 행동은 제외하거나, 재배열 알고리즘과 적용범위를 별도로 정의해야 한다.

초기 공통 Action 후보는 다음과 같다.

| Action Code | 행동 | 설명 |
|---|---|---|
| A0 | FIFO 유지 | 현재 도착순 또는 큐 순서 유지 |
| A1 | 장기대기 우선 | 가장 오래 기다린 차량 또는 작업 우선 |
| A2 | 근접 작업 우선 | 현재 크레인 위치에서 이동거리가 짧은 작업 우선 |
| A3 | 재조작 적은 작업 우선 | 바로 처리 가능한 컨테이너 우선 |
| A4 | 본선 연계 우선 | 대상 터미널의 STS·내부 이송장비 지연 가능성이 있는 작업 우선 |
| A5 | 도착예정 임박 작업 우선 | 제공된 차량 도착예정시각을 활용하여 곧 도착할 작업 우선 준비 |
| A6 | 재조작 선처리 | 차량 도착 전 대상 컨테이너 주변 정리 |
| A7 | 동일 Bay 묶음 처리 | 가까운 Bay 또는 같은 블록 작업을 연속 처리 |
| A8 | 작업순서 재배열 | 비용이 낮아지는 경우 큐 순서 조정 |
| A9 | 대기/양보 | 다중 크레인 또는 레인 혼잡 시 일부 작업 지연 |

Exp별로 적용 가능한 Action은 달라진다.

## 6. Exp-1: 블록 진입 후 Q-learning 최적화

### 6.1 개념

차량이 지정 야드블록에 진입한 이후, 해당 블록의 야드크레인 작업 대기열 안에서 다음 작업순서를 Q-learning으로 최적화한다.

```text
차량 블록 진입
→ 블록별 작업 대기열 등록
→ 현재 대기열·크레인 위치·컨테이너 위치 관측
→ Q-learning이 다음 작업 우선순위 선택
→ 야드크레인 이동 및 상·하차 수행
→ 비용 계산 및 Q값 업데이트
```

### 6.2 검증 목적

> 차량이 이미 블록에 도착한 뒤라도, 단순 도착순보다 작업순서를 지능적으로 조정하면 기사 대기시간과 크레인 이동거리를 줄일 수 있는가?

### 6.3 State 설계

Exp-1은 블록 내부 정보만 사용한다.

| State 항목 | 설명 | 이산화 예시 |
|---|---|---|
| 현재 시간대 | 피크/비피크, 주간/야간 | 3~4개 구간 |
| 야드블록 ID | 작업이 발생한 블록 | 블록 단위 |
| 야드크레인 현재 위치 | 현재 Bay 또는 작업 위치 | Bay 구간 |
| 크레인 가용상태 | 작업 중 여부와 남은 처리시간 | idle / remaining-time bucket |
| 블록 내 대기 차량 수 | 현재 블록 작업 큐 길이 | 실측 분포의 분위수 기반 bucket |
| 차량별 대기시간 | 후보 차량의 누적 대기시간 | 실측 분위수 또는 SLA 기반 bucket |
| 작업흐름 | 게이트·본선·환적·재조작 흐름 | gate-in, gate-out, load, discharge, transshipment, rehandle |
| 적공·규격·특수속성 | 컨테이너 취급속성 | full/empty, 20/40/45ft, reefer/DG/OOG 등 |
| 작업가능·Hold 상태 | 세관·검사·운영상 작업 가능 여부 | available / hold / inspection / not-released |
| 컨테이너 장치 위치 | 대상 컨테이너 Bay/Row/Tier | Bay 구간 중심 |
| 목표 Tier·상부 방해물 수 | 반출 작업의 재조작 판단 | 실제 Tier와 target 위 blocker 수 |
| 합법적 재조작 슬롯 | 재조작 컨테이너를 옮길 수 있는 슬롯 | 가용 슬롯 수·거리·적합성 |
| 본선작업 연계 여부 | 본선작업과 연결된 작업인지 | 0/1 |

초기 Q-table 안정화를 위해 전체 차량을 모두 State에 넣기보다 **상위 K개 후보 작업만 요약**한다.

```text
State_Exp1 = [
  time_slot,
  block_id,
  crane_position,
  crane_remaining_time,
  queue_length_bucket,
  topK_wait_time_bucket,
  topK_job_flow_and_container_attributes,
  topK_container_distance_bucket,
  topK_blocker_and_feasible_slot_summary,
  vessel_priority_flag
]
```

### 6.4 Action 설계

Exp-1에서는 블록 내부 현재 대기열에 대해서만 행동을 선택한다.

| Action Code | 행동 | Exp-1 적용 의미 |
|---|---|---|
| A0 | FIFO 유지 | 블록 도착순 유지 |
| A1 | 장기대기 우선 | 오래 기다린 차량 먼저 처리 |
| A2 | 근접 작업 우선 | 크레인 이동거리가 짧은 컨테이너 먼저 처리 |
| A3 | 재조작 적은 작업 우선 | 바로 꺼낼 수 있는 컨테이너 먼저 처리 |
| A4 | 본선 연계 우선 | 본선 작업과 연결된 작업이 있으면 우선 처리 |
| A7 | 동일 Bay 묶음 처리 | 가까운 위치의 작업을 묶어서 처리 |
| A8 | 작업순서 재배열 | 현재 큐 안에서 순서 변경 |

### 6.5 Reward 설계

Exp-1의 비용은 블록 내부 작업 효율 중심으로 공통 핵심비용을 사용한다.

```text
Cost_Exp1 = Core_Cost_t

Reward_Exp1 = -Cost_Exp1
```

### 6.6 기대효과

| 지표 | 기대효과 |
|---|---|
| 기사 평균대기시간 | 블록 내 순서변경으로 감소 가능 |
| 크레인 이동거리 | 근접·동일 Bay 규칙 선택 시 감소 가능하나 별도 검증 필요 |
| 장기대기 차량 | 장기대기 보호규칙 적용 시 감소 가능 |
| 재조작 횟수 | 현재 대기열 범위에서 제한적 개선 가능 |
| 터미널 작업 효율 | 블록 단위 동반 개선 여부 검증 |

### 6.7 한계

차량이 이미 블록에 도착한 뒤 최적화를 시작하므로, 게이트에서 블록까지 이동하는 동안의 혼잡, 특정 블록 쏠림, 차량 도착 전 재조작 준비는 반영하기 어렵다.

## 7. Exp-2: 게이트 진입 후 Q-learning 최적화

### 7.1 개념

차량이 게이트에 진입한 순간부터 목적 야드블록까지의 이동시간과 거리를 반영하여, 차량이 블록에 도착하기 전에 야드크레인 작업순서를 선제적으로 조정한다.

```text
차량 게이트 진입
→ 목적 블록 확인
→ 게이트~블록 거리/소요시간 계산
→ 블록 도착예상시각 산출
→ 현재 블록 큐 + 도착 예정 차량 정보 반영
→ Q-learning이 선제 작업순서 선택
```

### 7.2 검증 목적

> 차량이 블록에 도착하기 전 게이트 진입 정보를 활용하면, 블록 진입 후 최적화보다 더 큰 대기시간 감소 효과가 있는가?

### 7.3 State 설계

Exp-2는 Exp-1 State에 게이트 진입 정보와 블록 도착예상정보를 추가한다.

| State 항목 | 설명 | 이산화 예시 |
|---|---|---|
| Exp-1 공통 State | 블록 대기열, 크레인 위치, 컨테이너 위치 등 | 동일 |
| 게이트 진입시각 | 차량이 터미널에 들어온 시각 | 시간대 구간 |
| 목적 야드블록 | 차량이 향하는 블록 | 블록 ID |
| 게이트~블록 거리 | 차량 이동거리 | 실제 도로망 최단거리 또는 구간 |
| 게이트~블록 예상 이동시간 | 블록 도착예상시각 계산 | 연속 time-to-arrival 또는 실측 분위수 bucket |
| 내부 도로 혼잡도 | 터미널 내부 이동 지연 가능성 | 낮음/중간/높음 |
| 블록별 현재 대기열 | 현재 큐 상태 | 큐 길이 bucket |
| 블록별 예상 대기열 | 도착 예정 차량까지 포함한 큐 | 큐 길이 bucket |
| 차량 도착예상순서 | 앞으로 블록에 들어올 순서 | 순위 bucket |

```text
State_Exp2 = [
  State_Exp1,
  gate_in_time_slot,
  destination_block,
  travel_time_to_block_bucket,
  internal_road_congestion,
  expected_queue_length,
  arrival_order_bucket
]
```

### 7.4 Action 설계

Exp-2에서는 현재 블록 대기열뿐 아니라 곧 도착할 차량을 고려한다.

| Action Code | 행동 | Exp-2 적용 의미 |
|---|---|---|
| A0 | FIFO 유지 | 현재 블록 도착순 유지 |
| A1 | 장기대기 우선 | 이미 오래 기다린 차량 보호 |
| A2 | 근접 작업 우선 | 차량 도착 전 가까운 작업 처리 |
| A3 | 재조작 적은 작업 우선 | 짧은 시간 안에 처리 가능한 작업 우선 |
| A5 | 도착예정 임박 작업 우선 | 게이트 진입 차량의 블록 도착예상시각 반영 |
| A7 | 동일 Bay 묶음 처리 | 차량 도착 전 근접 작업 묶음 처리 |
| A8 | 작업순서 재배열 | 도착 예정 차량을 고려해 큐 재정렬 |

### 7.5 Reward 설계

Exp-2도 Exp-1과 동일한 핵심비용을 사용한다. 선제 이동이 불필요했다면 실제 이동거리와 계획변경 비용에, 작업준비가 늦었다면 queue-area에 반영되므로 별도의 “사전조정 실패”를 중복 부과하지 않는다.

```text
Cost_Exp2 = Core_Cost_t

Reward_Exp2 = -Cost_Exp2
```

불필요한 선제 이동과 계획변경 횟수는 원인분석용 보조 로그로 별도 기록한다.

### 7.6 기대효과

| 지표 | 기대효과 |
|---|---|
| 기사 평균대기시간 | 도착 전 정보 활용에 따른 추가 감소 가능성 검증 |
| 크레인 이동거리 | 선제 포지셔닝의 순이동 감소 여부 검증 |
| 장기대기 차량 | 기존 대기차량 보호조건 아래 감소 가능성 검증 |
| 터미널 작업 효율 | 게이트 진입정보와 야드 작업순서 연계효과 검증 |

## 8. Exp-3: 사전 반출입정보 및 부산항 제공 ETA 활용 최적화

### 8.1 개념

부산항에서 차량 도착 예정시각이 제공된다는 전제하에, 사전 반출입정보와 제공 ETA를 야드크레인 작업순서·재조작 선처리·크레인 포지셔닝에 활용한다.

```text
사전 반출입정보 등록
→ 부산항 제공 ETA 입력
→ 목적 블록 / 컨테이너 위치 확인
→ 미래 블록별 작업량 예측
→ Q-learning이 사전 작업순서 선택
→ 차량 도착 전 재조작·위치정리·크레인 위치 조정
```

### 8.2 검증 목적과 범위

> 부산항에서 제공된 차량 도착 예정시각과 사전 반출입정보를 야드크레인 작업순서·재조작 선처리·크레인 포지셔닝에 활용했을 때, 게이트 진입 후에만 최적화하는 것보다 운영성과가 개선되는가?

| 구분 | 연구범위 포함 여부 |
|---|---|
| ETA 직접 예측 | 제외 |
| ETA 산출방식 개발 | 제외 |
| ETA 예측 정확도 평가 | 제외 |
| 부산항 제공 ETA를 외생 입력값으로 사용 | 포함 |
| 제공 ETA를 활용한 야드크레인 최적화 효과 검증 | 포함 |

### 8.3 State 설계

Exp-3은 Exp-2 State에 사전 반출입정보와 미래 작업량 정보를 추가한다.

| State 항목 | 설명 | 이산화 예시 |
|---|---|---|
| Exp-2 공통 State | 게이트 진입/도착예정정보 포함 | 동일 |
| 부산항 제공 ETA | 터미널 또는 블록 도착예정시각 | 제공값 또는 현재시각 기준 잔여시간 |
| 예약시간/예정 반출입시간 | 작업 예정 시간창 | time window |
| 컨테이너 번호/작업 ID | 대상 컨테이너 식별 | ID는 직접 학습보다 속성으로 변환 |
| 작업흐름 | 사전 작업 유형 | gate-in/out, load/discharge, transshipment, rehandle |
| 적공·규격·특수속성 | 컨테이너 취급속성 | full/empty, 20/40/45ft, reefer/DG/OOG 등 |
| 목적 야드블록 | 작업 대상 블록 | 블록 ID |
| 컨테이너 장치 위치 | 작업 위치 | Bay/Row/Tier bucket |
| 시간대별 게이트 혼잡도 | 도착량 집중 가능성 | 낮음/중간/높음 |
| 블록별 미래 작업량 | 미래 대기열 예측 | 블록별 실측 분포의 분위수 bucket |
| 본선작업 마감시간 | 선적/양하 마감 또는 우선순위 | 임박/보통/낮음 |
| 재조작 선처리 가능 여부 | 도착 전 주변 컨테이너 정리 가능성 | 가능/불가 |

```text
State_Exp3 = [
  State_Exp2,
  provided_ETA_time_to_arrival,
  reservation_window,
  future_block_workload_bucket,
  target_container_position_bucket,
  pre_rehandle_feasibility,
  vessel_cutoff_priority
]
```

### 8.4 Action 설계

Exp-3에서는 차량 도착 전 사전 조치가 가능하다.

| Action Code | 행동 | Exp-3 적용 의미 |
|---|---|---|
| A0 | 현재 계획 유지 | 기존 작업계획 유지 |
| A1 | 장기대기 우선 | 기존 대기 차량 보호 |
| A2 | 근접 작업 우선 | 이동거리 최소화 |
| A3 | 재조작 적은 작업 우선 | 빠른 처리 가능 작업 우선 |
| A5 | 도착예정 임박 작업 우선 | 제공된 차량 도착예정시각을 활용해 곧 도착할 작업 준비 |
| A6 | 재조작 선처리 | 차량 도착 전 대상 컨테이너 주변 정리 |
| A7 | 동일 Bay 묶음 처리 | 위치가 가까운 작업을 묶어 처리 |
| A8 | 작업순서 재배열 | 현재+미래 대기열 통합 재정렬 |
| A10 | 사전 위치정리 | 향후 반출입을 고려해 컨테이너 위치 정리 |
| A11 | 크레인 사전 포지셔닝 | 차량 도착 예정 위치 근처로 크레인 이동 |

`A6 재조작 선처리`와 `A10 사전 위치정리`는 컨테이너가 이미 야드에 있고 작업가능 상태이며 합법적 목적 슬롯이 존재할 때만 허용한다. 아직 야드에 들어오지 않은 반입 컨테이너에는 대상 컨테이너 선처리를 적용할 수 없으므로, 반입 작업은 사전 장치슬롯 계획과 크레인 포지셔닝으로 구분한다.

### 8.5 Reward 설계

Exp-3도 동일한 핵심비용을 사용한다. 사전 재조작과 포지셔닝의 유용성은 대기·이동·재조작·본선지연의 실제 결과로 평가하며, 예측 실패라는 추상적 항목을 별도 중복계상하지 않는다.

```text
Cost_Exp3 = Core_Cost_t

Reward_Exp3 = -Cost_Exp3
```

불필요한 사전 포지셔닝 거리와 차량 도착 후 즉시 처리 성공률은 정책 원인분석용 보조지표로 기록한다.

### 8.6 기대효과

| 지표 | 기대효과 |
|---|---|
| 기사 평균대기시간 | 제공된 사전정보 활용의 추가가치 검증 |
| 장기대기 차량 | 선처리와 기존 대기차량 보호의 균형 검증 |
| 크레인 이동거리 | 선제 이동이 총이동을 줄이는지 별도 검증 |
| 재조작 횟수 | 합법적 재조작 슬롯과 미래 작업을 고려한 순감소 여부 검증 |
| 터미널 작업 효율 | 차량 도착 전 사전 최적화의 동반효과 검증 |

### 8.7 처리 예시

```text

```

## 9. Exp-4: 다중 야드크레인 협조 최적화

### 9.1 개념

Exp-4는 여러 야드크레인이 **동일 블록, 동일 레일 또는 운영상 작업을 교환할 수 있는 실제 서비스영역**에서 서로 영향을 주는 경우를 대상으로 한다. 수직형과 수평형 터미널을 하나의 연결 환경으로 혼합하지 않으며, 대상 터미널의 실제 도면·장비 도달영역·차량 인계지점을 먼저 고정한다.

```text
사전 반출입정보 / 부산항 제공 ETA 입력
→ 대상 서비스영역의 작업·차량·야드크레인 상태 통합 관측
→ 각 야드크레인을 Agent로 정의
→ Agent별 다음 작업 후보 산출
→ 중앙집중 학습 또는 공동 보상 기반 협조 학습
→ 대상 운영영역의 공동 비용 최소화
```

Exp-4의 핵심은 다음이다.

> 개별 야드크레인의 지역 최적이 아니라, 실제 작업 공유범위 안에서 야드크레인 간섭·본선작업·외부트럭 대기열을 함께 고려한 협조정책을 학습한다.

### 9.2 왜 Multi-Agent 문제인가?

야드크레인이 여러 대라는 이유만으로 Multi-Agent 문제가 되는 것은 아니다. 다음 조건이 있을 때 Multi-Agent 문제가 된다.

```text
- 여러 야드크레인이 동일 레일·블록 또는 작업교환이 가능한 운영영역을 공유함
- 한 크레인의 작업 선택이 다른 크레인의 이동·대기·작업순서에 영향을 줌
- 대상 터미널의 내부 이송장비 대기열이 여러 크레인의 선택에 의해 변함
- 비통과, 안전거리, Bay·인계지점 점유가 다른 크레인의 행동을 제한함
- 본선작업과 외부트럭 작업이 여러 크레인에 걸쳐 우선순위 충돌을 일으킴
```

즉, Exp-4는 단순히 “크레인이 여러 대”인 문제가 아니라, **물리적 간섭과 공동 대기열 때문에 크레인 간 선택이 서로 영향을 주는 협조 문제**이다.

### 9.3 검증 목적

> 여러 야드크레인이 동일 서비스영역에서 상호 영향을 갖는 경우, 각 크레인을 개별적으로 최적화하는 것보다 공동 제약과 공동비용을 반영한 협조 최적화가 더 효과적인가?

### 9.4 Agent 정의

```text
Agent_i = i번째 야드크레인
```

예를 들어 야드크레인이 N대라면 다음과 같이 정의한다.

```text
YC-1 Agent
YC-2 Agent
...
YC-N Agent
```

초기 구현에서는 개별 야드크레인이 독립적으로 판단하기보다 중앙 시뮬레이션 학습기가 전체 상태를 보고 각 크레인에 추천 작업을 내려주는 구조를 우선한다. 개별 agent 구조는 실제 분산실행 필요성이 확인된 이후 비교한다.

### 9.5 State 설계

Exp-4는 Exp-3 State에 실제 크레인 도달영역과 다중 야드크레인 상호 영향을 추가한다.

| State 항목 | 설명 | 목적 |
|---|---|---|
| Exp-3 공통 State | 부산항 제공 ETA, 미래 작업량, 컨테이너 위치, 사전 재조작 가능성 | 차량 사전정보 활용 |
| 크레인 도달영역 | 각 크레인이 실제 처리할 수 있는 Bay·인계지점 | 불가능한 재할당 제거 |
| 비통과·안전거리 상태 | 크레인 순서, 최소간격, 차단구간 | 충돌·간섭 방지 |
| 작업차선·인계지점 대기열 | 외부트럭과 실제 내부 이송장비의 대기 | 인계지점 병목 반영 |
| 야드크레인별 현재 위치·가용시각 | 각 크레인의 Bay·작업 위치와 남은 작업시간 | 이동거리·작업가능시점 산정 |
| 야드크레인별 작업 예정 목록 | 각 크레인의 다음 작업 후보 | 작업 충돌·중복 방지 |
| 인접 야드크레인 위치 | 주변 크레인 상태 | 상호 영향 파악 |
| 인접 야드크레인 작업 방향 | 이동·작업 흐름 | 양보/대기 판단 |
| 작업 점유 상태 | 특정 Bay·차선·인계지점 점유 여부 | 중복 작업 방지 |
| 내부 이송장비 대기 상태 | 대상 터미널에 실제 존재하는 YT·SC·AGV 중 해당 장비 | 본선 연계 효율 반영 |
| STS 작업 우선순위 | 본선작업 지연 가능성 | 선박 하역 생산성 보호 |
| 블록별 작업량 불균형 | 특정 블록 작업 쏠림 | 균형 배분 |
| 장기대기 차량 분포 | 레인·블록별 장기대기 | 공정성 확보 |

```text
State_Exp4_Agent_i = [
  State_Exp3,
  crane_i_position,
  crane_i_queue,
  neighbor_crane_positions,
  neighbor_crane_planned_actions,
  crane_reachability_and_non_crossing_state,
  transfer_point_occupancy,
  actual_transport_equipment_waiting_state,
  STS_priority_state,
  workload_balance_state,
  long_wait_distribution
]
```

### 9.6 Action 설계

Exp-4에서는 단일 크레인의 다음 작업 선택뿐 아니라, 물리적으로 가능한 범위의 다중 크레인 협조 행동을 포함한다.

| Action Code | 행동 | Exp-4 적용 의미 |
|---|---|---|
| A0 | 현재 계획 유지 | 기존 작업계획 유지 |
| A1 | 장기대기 우선 | 장기대기 차량 또는 작업 우선 |
| A2 | 근접 작업 우선 | 각 크레인별 이동거리 최소화 |
| A3 | 재조작 적은 작업 우선 | 전체 처리량 향상 |
| A4 | 본선 연계 우선 | STS와 실제 내부 이송장비의 대기 방지 |
| A5 | 도착예정 임박 작업 우선 | 제공된 차량 도착예정시각을 활용한 작업 선처리 |
| A6 | 재조작 선처리 | 도착 전 컨테이너 주변 정리 |
| A7 | 동일 Bay 묶음 처리 | 크레인 이동거리 감소 |
| A8 | 작업순서 재배열 | 정의된 재배열 규칙으로 여러 크레인 큐 조정 |
| A9 | 대기/양보 | 비통과·안전거리 또는 타 크레인 작업 보호 |
| A12 | 작업 크레인 재할당 | 두 크레인의 실제 공통 도달영역 안에서만 작업 재배정 |
| A13 | 작업차선·인계지점 선택 | 동일 작업에 복수의 물리적 선택지가 있을 때만 적용 |
| A14 | 작업량 균형화 | 특정 크레인 과부하 완화 |
| A15 | 본선 우선 구간 확보 | 본선 연계 흐름을 보호하기 위한 큐 조정 |

### 9.7 Reward 설계

Exp-4에서는 개별 크레인 성능과 대상 운영영역의 공동 성능을 함께 반영해야 한다. 모든 비용항은 공통 핵심비용과 같은 기준으로 정규화한다.

```text
Local_Cost_i =
  c1 × 해당 크레인의 정규화 이동비용
+ c2 × 해당 크레인 담당 작업의 정규화 대기비용
+ c3 × 해당 크레인의 정규화 재조작비용

Global_Cost =
  Core_Cost_t
+ g1 × 정규화된 크레인 간 간섭·상호대기시간
+ g2 × 정규화된 크레인별 작업량 불균형

Cost_Exp4_Agent_i = β × Local_Cost_i + (1 - β) × Global_Cost
Reward_Exp4_Agent_i = -Cost_Exp4_Agent_i
```

중앙집중형 단일 agent를 사용할 때는 `Global_Cost`만 사용한다. 다중 agent에서는 `β` 민감도 분석을 수행하여 지역비용의 중복가중과 전역보상의 credit assignment 문제를 확인한다. 처리량과 본선 서비스 수준은 `Baseline 대비 허용저하 δ 이내`의 제약조건으로 보호한다.

### 9.8 Exp-4 알고리즘 방향

Exp-4에서 바로 완전한 Multi-Agent RL을 도입하기보다, 다음 단계로 접근한다.

| 단계 | 방식 | 설명 |
|---|---|---|
| 4-1 | 중앙집중형 스케줄러 | 전체 상태를 보고 각 크레인의 다음 작업 추천 |
| 4-2 | Independent Q-learning 비교군 | 비정상성·credit assignment 문제를 확인하기 위한 비교군 |
| 4-3 | Cooperative Multi-Agent RL | 공동 보상 기반 협조 학습 |
| 4-4 | CTDE 구조 | 학습은 중앙에서, 실행은 크레인별 추천으로 수행 |
| 4-5 | 제약 기반 스케줄링 + RL | 불가능한 행동은 Action Masking으로 제거 |

제안서에서는 특정 알고리즘 하나를 “전역 최적 보장 알고리즘”으로 단정하지 않고 다음처럼 표현하는 것이 안전하다.

> 다중 야드크레인 간 실제 작업영역·안전제약·이송장비 대기 상태가 상호 영향을 갖는 경우, 중앙집중형 스케줄러 또는 중앙집중 학습·분산 실행 구조의 협조형 Multi-Agent RL로 확장한다. 공동 보상과 Action Masking을 통해 개별 크레인의 지역 최적화보다 대상 운영영역의 공동 성능을 개선하는 것을 목표로 한다.

### 9.9 기대효과

| 지표 | 기대효과 |
|---|---|
| 외부 트럭 평균대기시간 | Exp-3 대비 추가 감소 가능 |
| 장기대기 차량 비율 | 레인·크레인별 공정성 반영으로 감소 |
| 야드크레인 총 이동거리 | 작업 묶음·크레인 배분으로 감소 |
| 야드크레인 빈 이동거리 | 불필요한 이동 감소 |
| 재조작 횟수 | 차량 도착 전 선처리와 크레인 협조로 감소 |
| 작업차선·인계지점 혼잡도 | 실제 선택 가능한 지점의 협조 배정으로 감소 가능 |
| STS·내부 이송장비 대기 | 본선 연계 우선순위 반영으로 감소 가능 |
| 터미널 처리량 | Baseline 대비 유지 또는 증가 여부 검증 |

위 항목은 보장된 결과가 아니라 Exp-3 대비 동반 개선 여부를 검증할 기대효과이다.

### 10.1 State 요약

| 구분 | State 핵심 |
|---|---|
| Exp-1 | 블록 대기열, 차량 대기시간, 크레인 위치, 컨테이너 위치, 재조작 예상, 본선 연계 여부 |
| Exp-2 | Exp-1 + 게이트 진입시각, 목적 블록, 블록 도착예상시각, 내부도로 혼잡도, 예상 대기열 |
| Exp-3 | Exp-2 + 사전 반출입정보, 부산항 제공 ETA, 미래 작업량, 사전 재조작 가능성, 본선작업 마감시간 |
| Exp-4 | Exp-3 + 크레인 도달영역, 비통과·안전거리, 다중 크레인 위치·가용시각, 인계지점 점유, 실제 이송장비 대기 |

### 10.2 Action 요약

| 구분 | Action 핵심 |
|---|---|
| Exp-1 | FIFO 유지, 장기대기 우선, 근접 작업 우선, 재조작 적은 작업 우선, 본선 연계 우선, 작업순서 재배열 |
| Exp-2 | Exp-1 + 게이트 진입 차량의 블록 도착 전 근접 작업 처리, 예상 대기열 기반 순서 조정 |
| Exp-3 | Exp-2 + 재조작 선처리, 사전 위치정리, 크레인 사전 포지셔닝, 미래 작업량 기반 작업 묶음 처리 |
| Exp-4 | Exp-3 + 공통 도달영역 내 작업 재할당, 작업차선·인계지점 선택, 대기/양보, 작업량 균형화 |

### 10.3 Reward 요약

| 구분 | Reward/Cost 핵심 |
|---|---|
| Exp-1 | 정규화된 queue-area, 장기대기, 적재·빈 이동, 재조작, 계획변경, 본선지연 |
| Exp-2 | Exp-1과 동일한 Core Cost 사용; 선제 이동·계획변경은 원인분석 로그로 기록 |
| Exp-3 | Exp-1과 동일한 Core Cost 사용; 불필요한 포지셔닝·즉시처리 성공률은 원인분석 로그로 기록 |
| Exp-4 | Core Cost + 정규화된 크레인 간 간섭·상호대기 + 작업량 불균형 |

## 11. 평가 지표

기사 불만 감소와 터미널 운영 효율을 동시에 판단하기 위해 다음 지표를 사용한다.

### 11.1 핵심 성능지표

| 평가 지표 | 설명 |
|---|---|
| 평균대기시간 | 전체 기사 대기시간 감소 효과 |
| 중앙값 대기시간 | 일반적인 기사 체감 수준 |
| 90/95퍼센타일 대기시간 | 장기대기 차량 감소 여부 |
| 최대대기시간 | 극단적 불만 발생 가능성 |
| 장기대기 차량 비율 | 일정 시간 이상 대기한 차량 비율 |
| 종료시점 미처리 작업 수·누적대기 | 어려운 작업을 평가구간 밖으로 미루는 정책 방지 |
| 야드크레인 총 이동거리 | 크레인 작업 이동 효율 |
| 야드크레인 빈 이동거리 | 컨테이너 없이 이동한 거리 |
| Gantry·Trolley·Hoist별 이동·작업시간 | 이동거리만으로 표현되지 않는 크레인 사이클 분석 |
| 재조작 횟수 | 반출 작업 효율 |
| 시간당 처리량 | 터미널 생산성 |
| 작업순번 변경 횟수 | 기사 안내 안정성 |
| 본선작업 지연 여부 | 선박 하역 작업 영향 |
| 작업차선·인계지점 혼잡도 | 실제 차량 인계구간의 병목 발생 여부 |
| 크레인별 작업량 편차 | 특정 크레인 과부하 여부 |
| 내부 이송장비 대기시간 | 대상 터미널에 실제 존재하는 YT·SC·AGV의 병목 여부 |
| STS 대기시간 | 본선작업 생산성 영향 |

대기시간은 하나의 값으로만 기록하지 않고 `게이트 진입→블록 도착`, `블록 도착→서비스 시작`, `서비스 시작→완료`로 분해한다. 본 연구의 직접 정책효과는 주로 블록 대기와 작업 서비스 구간에서 해석한다.

### 11.2 환경 기대효과용 보조지표

탄소는 강화학습 State·Action·Reward에 직접 적용하지 않는다. 다만 대기시간·이동거리·재조작 감소가 환경효과로 이어질 가능성을 사후 평가하기 위해, 자료 확보 시 다음 지표를 보조적으로 기록한다.

| 보조지표 | 용도 |
|---|---|
| 외부트럭 엔진가동 공회전시간 | 대기시간 감소가 실제 공회전 감소로 연결되는지 확인 |
| 야드크레인 연료 또는 전력사용량 | 이동·재조작 감소의 운영에너지 기대효과 확인 |
| 대상 내부 이송장비 연료·전력사용량 | 병목이 다른 장비로 전가되는지 확인 |
| 추정 운영단계 CO2e | 동일 물동량에서 Baseline 대비 순감축 가능성의 탐색적 평가 |

## 12. 산출 방식

### 12.1 대기시간 감소율

```text
대기시간 감소율 =
(기준 평균대기시간 - 실험 평균대기시간) / 기준 평균대기시간 × 100
```

### 12.2 크레인 이동거리 감소율

```text
크레인 이동거리 감소율 =
(기준 이동거리 - 실험 이동거리) / 기준 이동거리 × 100
```

### 12.3 장기대기 차량 감소율

```text
장기대기 차량 감소율 =
(기준 장기대기 차량 수 - 실험 장기대기 차량 수) / 기준 장기대기 차량 수 × 100
```

### 12.4 재조작 감소율

```text
재조작 감소율 =
(기준 재조작 횟수 - 실험 재조작 횟수) / 기준 재조작 횟수 × 100
```

### 12.5 작업차선·인계지점 혼잡 감소율

```text
혼잡 감소율 =
(기준 혼잡지수 - 실험 혼잡지수) / 기준 혼잡지수 × 100
```

### 12.6 환경 기대효과의 사후 산출

```text
탐색적 운영단계 CO2e =
외부트럭 엔진가동 공회전량 × 해당 배출계수
+ 야드크레인 연료사용량 × 연료 배출계수
+ 야드크레인 전력사용량 × 대상연도 전력 배출계수
+ 실제 내부 이송장비의 연료·전력 기반 배출량

기대 감축량 = Baseline 추정 CO2e - 실험정책 추정 CO2e
```

이 값은 정책의 직접 목적함수가 아니라 사후 기대효과 지표이다. 엔진 ON/OFF, 연료·전력 실측치가 없으면 대기시간 또는 이동거리만으로 탄소감축을 확정하지 않고 시나리오 범위로 제시한다.

## 13. 야드크레인 운영·제어 설계

### 13.1 최적화 대상

강화학습 최적화 알고리즘의 적용 대상은 **크레인의 물리적 레일 방향 변경**이 아니다. 최적화 대상은 다음이다.

```text
- 야드크레인의 작업순서
- 야드크레인의 이동순서
- 블록별 대기열 처리순서
- 차량 도착 전 재조작 선처리 여부
- 물리적으로 선택 가능한 작업차선·인계지점 배정
- 실제 공통 도달영역 내 다중 야드크레인 작업량 균형화
```

### 13.2 터미널 유형별 고려사항

| 구분 | 야드블록 배치 | 야드크레인 주행 방향 | 최적화 대상 |
|---|---|---|---|
| 수직터미널 | 안벽과 직각 | 블록 길이방향 이동 | 해측·육측 작업분리, SC 또는 AGV 연계, 외부트럭 인계지점 대기 최소화 |
| 수평터미널 | 안벽과 평행 | 블록 길이방향 이동 | YT·외부트럭 작업순서, 블록 측면 대기열, 크레인 간섭 최소화 |

수직형과 수평형을 하나의 연결 환경에 섞지 않고 별도 시나리오로 검증한다. 부산항은 터미널별 장비구성이 다르므로 한 실험 안에서 YT·SC·AGV를 동시에 일반 상태변수로 사용하지 않는다.

### 13.3 부산항 대상 터미널 고정 원칙

| 유형 예시 | 공개 확인되는 주요 특성 | 설계 시 적용방향 |
|---|---|---|
| HJNC형 수평 자동화 | 21블록, ARMGC 42기, YT 중심 | 동일 블록 또는 실제 공통 도달영역의 다중 ARMGC 협조 |
| PNIT·HPNT형 수평 자동화 | RMG/ATC와 YT 중심 | 고정 레일 서비스영역과 외부·내부차량 대기열 반영 |
| BNCT·BCT형 수직 야드 | 수직형 블록과 SC 중심 | 해측·육측 인계영역과 양측 작업흐름 분리 |
| DGT형 완전자동화 | ARMG와 AGV 중심 | 육측 외부트럭·해측 AGV의 분리된 큐와 자동화 제약 반영 |

연구 시작 전에 대상 터미널, 기준연도, 블록 ID, 크레인 형식·대수, 크레인별 도달영역, 차량 인계방식, 장치단수, 안전거리와 비통과 규칙을 확정한다. 공개자료는 구조 확인에만 사용하고 이동속도·가감속·작업시간·고장률은 장비 PLC 또는 운영로그로 보정한다.

### 13.4 환경·탄소중립 기대효과 해석

본 연구는 차량 평균대기시간과 야드 운영비용을 줄이는 운영 최적화 연구이며, 탄소배출량을 직접 최소화하는 연구로 정의하지 않는다. 다음 관계는 자동으로 성립하는 결론이 아니라 사후 검증할 기대효과이다.

```text
외부트럭 대기 감소
→ 엔진가동 공회전시간이 함께 줄어든 경우 배출 감소 가능

야드크레인 이동·재조작 감소
→ 실제 연료 또는 순전력 사용량이 함께 줄어든 경우 배출 감소 가능

전체 운영단계 추정 CO2e 감소
→ 부산항 탄소중립 목표에 기여할 수 있는 운영효율화 기대효과
```

따라서 결과 문구는 “탄소중립을 달성한다”가 아니라 “운영단계 온실가스 감축 및 탄소중립 목표에 대한 기여 가능성을 확인한다”로 제한한다.

## 14. 시뮬레이션 설계

강화학습 적용 전 실제 운영 투입이 아니라 시뮬레이션 환경에서 검증한다.

### 14.1 시뮬레이션 구성 요소

| 구성 요소 | 설명 |
|---|---|
| 야드블록 | 선택한 한 터미널의 수직형 또는 수평형 블록 구조 반영 |
| 도로·인계지점 | 실제 도로망, 작업차선, 인계지점 점유와 선택 가능성 반영 |
| 야드크레인 | Gantry·Trolley·Hoist 속도와 가감속, 적재·무부하, 가용상태 반영 |
| 다중크레인 제약 | 도달영역, 비통과, 최소 안전거리, 충돌·간섭 반영 |
| 차량 | 외부트럭과 대상 터미널에 실제 존재하는 내부 이송장비 도착 이벤트 |
| 컨테이너 | Bay/Row/Tier, blocker, 합법적 재조작 슬롯, 작업흐름, 적공·규격·특수속성, Hold·작업가능 상태 |
| 본선작업 | STS 작업순서, 선적/양하 우선순위 |
| 큐 | 블록별 작업 대기열, 본선 연계 작업 큐, 레인별 대기열 |
| 이벤트 | 사전 반출입정보, 부산항 제공 ETA 입력, 게이트 진입, 블록 도착, 장비 지연 |
| 강화학습 에이전트 | Exp-1~3 단일/중앙 Agent, Exp-4 다중 Agent 또는 중앙집중형 협조 Agent |

### 14.2 시뮬레이션 이벤트 흐름

```text
사전 반출입정보 발생
→ 부산항 제공 ETA를 외생 입력값으로 반영
→ 게이트 진입 이벤트
→ 블록 도착 이벤트
→ 본선작업 이벤트
→ 야드크레인 작업 후보 생성
→ Q-learning 행동 선택
→ 크레인 이동
→ 상차/하차/재조작
→ 작업 완료
→ 대기시간·이동거리·처리량·인계지점 혼잡 기록
→ 자료 확보 시 공회전·연료·전력을 환경 기대효과용으로 별도 기록
→ Q값 업데이트
```

## 15. 실험 통제 조건

각 실험은 동일한 조건에서 비교해야 한다.

```text
- 같은 일자 반출입 물량
- 같은 야드 블록 구조
- 같은 크레인 수
- 같은 컨테이너 장치 위치
- 같은 차량 도착 분포
- 같은 본선작업 조건
- 같은 장비 처리속도
- 같은 재조작 조건
- 같은 도로·인계지점·크레인 도달영역
- 같은 부산항 제공 ETA 입력값
```

이렇게 해야 알고리즘 차이로 인한 효과를 비교할 수 있다.

추가 검증조건은 다음과 같다.

```text
- 정보시점과 제어범위를 분리한 요인실험
- 정상·피크 물량, 낮음·중간·높음 장치율, 장비고장 시나리오
- 일자 기준 train / validation / test 분리
- 정책별 동일 이벤트와 공통난수 사용
- 복수 seed 반복, 평균차이·95% 신뢰구간·효과크기 보고
- 평가 종료시점 미처리 작업과 누적대기 포함
```

## 16. 적용 로드맵

### 16.1 단계별 적용

```text
1단계: Baseline 재현
- 대상 터미널의 현행 YC 작업규칙 재현
- 규칙 미확보 시 FIFO·최장대기·근접작업 휴리스틱을 비교기준으로 구현

2단계: Exp-1 구현
- 블록 내부 대기열 기반 Q-learning
- 단일 야드크레인 또는 단일 블록 기준 PoC

3단계: Exp-2 구현
- 게이트 진입 차량의 블록 도착예상시간 반영
- 현재 큐 + 도착 예정 큐 통합 최적화

4단계: Exp-3 구현
- 사전 반출입정보 및 부산항 제공 ETA를 외생 입력값으로 반영
- 차량 도착 전 재조작·위치정리·크레인 포지셔닝 최적화

5단계: Exp-4 구현
- 동일 블록 또는 실제 서비스영역 내 다중 야드크레인 협조 최적화
- 중앙집중형 스케줄링 → 협조형 Multi-Agent RL로 확장
```

### 16.2 알고리즘 고도화 방향

| 단계 | 알고리즘 | 적용 범위 |
|---|---|---|
| 초기 | Tabular Q-learning | Exp-1 단순 PoC |
| 중간 | Q-learning + Action Masking | Exp-2~3 제약 반영 |
| 고도화 | DQN/PPO 또는 후보집합 인코딩 모델 | 상태공간·가변 후보집합이 커지는 경우 |
| 최종 | Cooperative Multi-Agent RL / CTDE | Exp-4 다중 크레인 협조 |
| 운영 적용 | 제약 기반 스케줄링 + RL 추천 | TOS 연계 및 안전 제약 반영 |

## 17. 최종 판단

본 실험 설계의 연구방향은 사전 차량 도착정보를 활용한 야드크레인 작업순서 최적화 선행연구와 부합한다. 다만 타당한 결과를 얻으려면 대상 터미널의 물리구조를 고정하고, 정보 확보시점·제어범위·다중크레인 협조범위를 분리하며, 모든 실험에 동일한 정규화 비용함수를 적용해야 한다.

기존 핵심 메시지는 다음이다.

> 정보 확보 시점이 빠를수록 야드크레인 작업순서 최적화 효과가 커지는지 검증한다.

이 문장은 사전 결론이 아니라 다음 검증가설로 해석한다.

```text
H1. 동일한 제어범위에서 차량 정보를 더 일찍 확보하면 외부트럭 대기비용이 감소하는가?
H2. 부산항에서 제공된 ETA를 활용한 사전 작업준비가 게이트 진입 후 최적화보다 우수한가?
H3. 대기시간 감소가 야드크레인 이동·재조작·본선지연의 동반 개선으로 이어지는가?
H4. 실제 공통 서비스영역의 다중 크레인 협조가 단일 크레인별 최적화보다 우수한가?
H5. 대기·이동·재조작 개선이 실제 에너지와 운영단계 온실가스 감축 기대효과로 이어지는가?
```

업데이트 후 최종 핵심 메시지는 다음이다.

> 본 연구는 부산항의 특정 터미널·블록을 대상으로 블록 진입정보, 게이트 진입정보, 사전 반출입정보와 부산항 제공 ETA를 야드크레인 작업순서 최적화에 활용했을 때의 효과를 검증하고, 실제 크레인 도달영역·비통과·안전거리·본선 서비스 제약을 반영한 다중 야드크레인 협조정책으로 확장한다. 외부트럭 대기시간을 중심으로 이동·재조작·본선지연의 동반 개선 여부를 평가하며, 탄소는 직접 최적화 대상이 아니라 운영효율 개선에 따른 사후 기대효과로 확인한다.

ETA 오차·누락·갱신은 본 실험의 핵심 검증대상이 아니며, 필요 시 본 연구 이후 별도의 민감도 분석 항목으로 분리한다.

## 18. 제안서용 문장

> 본 연구는 부산항에서 사전 반출입정보 기반 차량 ETA가 제공된다는 전제하에, 해당 ETA를 야드크레인 작업순서 최적화의 입력값으로 활용한다. ETA의 산출 방식이나 예측 정확도는 연구범위에 포함하지 않으며, 제공된 ETA를 활용했을 때 차량 도착 전 재조작 선처리, 크레인 사전 포지셔닝, 작업순서 조정을 통해 기사 평균대기시간과 크레인 이동거리를 줄일 수 있는지를 검증한다. 모든 실험에는 정규화된 외부트럭 queue-area, 장기대기, 크레인 이동, 재조작, 본선 및 내부이송장비 지연, 작업계획 변경으로 구성한 동일 비용함수를 적용한다.

> Exp-4에서는 수직형과 수평형 터미널을 혼합하지 않고, 선정된 터미널의 동일 블록 또는 실제 공통 서비스영역에서 다중 야드크레인의 도달영역·비통과·안전거리·인계지점 점유를 반영한다. 중앙집중형 또는 협조형 학습정책이 개별 크레인별 정책보다 외부트럭 대기, 크레인 간 간섭, 재조작, 본선작업 지연을 개선하는지 검증한다. 탄소는 보상함수에 직접 적용하지 않으며, 대기·이동·재조작 개선이 실제 공회전과 장비 연료·전력 감소로 연결되는 경우 운영단계 온실가스 감축 및 부산항 탄소중립 목표에 기여할 수 있는 기대효과로 평가한다.
