# 부산항 야드크레인 강화학습 구현계획서

## 1. 문서 목적

본 문서는 [실험설계안](./부산항_야드크레인_강화학습_실험설계안_업데이트.md)을 코드와 실험으로 구현하기 위한 **상위 인덱스**이다. 상세 명세는 구현 책임 단위별 문서로 분리한다.

본 연구는 야드크레인(YC)의 작업순서, 사전 포지셔닝, 조건부 사전 재조작을 최적화한다. 외부트럭 평균·상위 대기시간 감소가 주목표이며, YC 이동거리·재조작·처리량·서비스 수준은 함께 평가한다.

ETA는 부산항에서 제공된다고 가정하는 외생 입력값이다. ETA 생성모형, 예측 정확도, 오차분포 평가는 구현 범위에 포함하지 않는다. 탄소는 State·Action·Reward에 넣지 않고 운영효율 개선에 따른 사후 기대효과로만 평가한다.

## 3. 핵심 구현 결정

| 항목 | 결정 |
|---|---|
| 시뮬레이션 | 연속시간 이산사건 방식 |
| 문제형식 | 작업별 가변 경과시간을 반영한 SMDP |
| 1차 PoC | 실제 자료로 보정 가능한 1개 블록·YC 1기 |
| 확장 | 실제 공통 서비스영역의 YC 2기 이상 |
| 행동 | 다음 작업 자체가 아니라 작업을 고르는 priority rule 선택 |
| 초기 정책 | 집계상태 기반 Tabular Q-learning hyper-heuristic |
| 고도화 | Action Masking 기반 DQN/PPO; 필요 시 다중 agent |
| 비교 | 동일 초기상태·실제 도착흐름·이벤트·난수표본 사용 |
| 안전 | 도달영역, 슬롯 적합성, 비통과, 안전거리 등을 Hard Constraint로 강제 |
| ETA | Exp-3에서만 정책에 공개되는 제공 입력값; 예측모델·정확도 평가는 제외 |
| 탄소 | 정책 목적이 아닌 사후 기대효과; 실측자료가 있을 때만 정량화 |
| UI | 시뮬레이터와 분리된 읽기 전용 replay 구조 |

## 4. 실험 단계

| 구분 | 정책에 공개되는 차량 정보 | 검증 목적 |
|---|---|---|
| Baseline | 현행 작업지시 시점의 정보 | 현행 dispatching rule 재현 |
| Exp-1 | 블록 도착 이후 정보 | 도착 이후 반응형 최적화 효과 |
| Exp-2 | 게이트 진입 이후 정보 | 더 이른 정보 제공 효과 |
| Exp-3 | 사전 반출입정보와 제공 ETA | 차량 도착 전 준비행동 효과 |
| Exp-4 | Exp-3 정보 + 복수 YC 상태 | 실제 공통 서비스영역의 협조 효과 |

Exp-1~3은 정보 공개시점 외 조건을 동일하게 둔다. 작업순서만 허용하는 실험과 포지셔닝·사전 재조작까지 허용하는 실험을 분리해, 정보 효과와 행동범위 효과를 혼동하지 않는다.

---

## 5. 전체 실행 흐름

```text
운영자료·터미널 설정
        ↓
정제·익명화·날짜별 데이터 분리
        ↓
이산사건 시뮬레이터 + Hard Constraint
        ↓
공통 RL 환경(State·Mask·Rule·Reward)
        ↓
Baseline / Q-learning / 고도화 정책
        ↓
동일 시나리오 반복평가
        ├─ KPI·통계·사후 탄소 기대효과
        └─ snapshot/event log → 검증 UI
```

UI는 시뮬레이터 상태를 변경하지 않는다. 동일 seed로 산출된 snapshot과 event log를 읽어 단일 정책을 재생하거나 Baseline과 RL을 같은 시각축에서 비교한다.

---

## 6. 단계별 산출 결과

1. 실제 로그의 이동·서비스시간을 재현하는 단일 YC 시뮬레이터
2. 현행 규칙과 최소 3개 휴리스틱 Baseline
3. Exp-1~3을 동일 비용함수로 실행하는 Tabular Q-learning 환경
4. 단일 시나리오 재생 및 Baseline/RL 동기 비교 UI
5. 다중 YC 공통 서비스영역과 간섭 제약을 포함한 Exp-4
6. 평균·P90/P95 대기, 이동, 재조작, 처리량, 본선·내부작업 보호지표 보고서
7. 실측 에너지·공회전 자료가 있는 경우에 한한 탄소 감축 잠재력 보고서

---

## 7. 구현 원칙

> 먼저 실제 운영을 충분히 재현하는 시뮬레이터와 강한 Baseline을 구축한다. 그 위에서 동일 사건과 동일 비용함수를 사용해 정보 공개시점과 제어범위의 효과를 분리한다. 제공 ETA는 야드크레인 최적화에 이용되는 외부 입력일 뿐 예측 연구대상이 아니다. 안전제약은 학습할 대상이 아니라 항상 지켜야 하는 규칙이며, UI는 결과를 검증하고 설명하기 위한 읽기 전용 도구이다. 탄소는 실제 공회전·연료·전력 감소가 확인될 때의 사후 기대효과로만 평가한다.
