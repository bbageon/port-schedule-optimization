# YR-054 — RL 재감사: 게이트 전패의 원인은 "누락"이 아니라 "협조 실패"

> YR-045 §7 지시("feature·비용·후보 누락 재감사") 이행. 분석 원자료는 locked run 의
> `outputs/reports/yr045_locked_rerun/phase_d/` (36조건 × 60 seed, 항별 비용·행동·후보 포함).
> 새 실험 없음 — 기존 원자료의 정량 분해다.

## 1. 결론 (쉬운 말)

RL이 rollout baseline에 +25~28% 뒤진 원인은 정보나 후보가 모자라서가 아니다.
**두 크레인이 서로 부딪히는 선택을 해서 생기는 양보 비용(interference)이 격차의 약 85%**다.
rollout은 매 결정마다 두 크레인의 조합을 통째로 시뮬레이션해 충돌을 피하는 반면, RL은
크레인별로 독립 점수(argmin)를 매겨 충돌을 예측하지 못한다 — 중앙 resolver가 안전은
지켜주지만(위반 0) 진 크레인은 양보(WAIT)로 시간을 태운다.

## 2. 정량 근거 (dueling vs JointRollout, FULL/forbid — RL 최선 조건)

| 비용 항 | JR | dueling | Δ | 해석 |
|---|---:|---:|---:|---|
| **interference** (양보 크레인 벌점) | 10.38 | 24.87 | **+14.48** | 격차 +17.1 의 **85%** |
| lane_cong (레인 혼잡) | 47.09 | 48.67 | +1.58 | 소폭 |
| truck_wait (트럭 대기) | 2.61 | 2.86 | +0.24 | 대기는 거의 대등 |
| 기타 (sts·transfer 등) | — | — | +0.83 | 소폭 |

- 행동 분포: WAIT 선택 **1638 vs 655 (2.5배)** — forbid 모드라 전략 대기가 아니라 **경합
  패배(구조적 양보)**가 다수. 실작업 가능 시 SERVE 선택률은 둘 다 건전(0.44 vs 0.48).
- **후보 누락 없음**: RL이 본 후보 kind 합이 JR 이상(PRE 9675 vs 8418 등 — 결정 수가 많아서).
- **비용 누락 없음**: 지배도 guard 통과(최대 항 48.6%), 13항 전부 발현.
- **feature 누락 판단**: 계약 feature 는 양 정책 동일. 단 "상대 크레인이 다음에 무엇을 할지"는
  어떤 feature 로도 직접 주어지지 않는다 — rollout 은 그것을 시뮬레이션으로 얻는다(구조 차이).

## 3. 부가 가설 검증

- **전략적 WAIT 오용 (allow 조건)**: 실재하나 2차 요인 — allow→forbid 개선 −4.0~−11.9,
  개선의 최대 단일 사례는 미완주 seed 420037(+160, 트럭 1대 방치). 미완주 6건 전부 allow.
- **NO_ETA 폭주 (dqn 134.9)**: 같은 뿌리의 극단 사례 — 최악 seed 420057 의 비용은
  interference **216.1** (전체의 74%)이 지배. ETA feature 가 결측(known=0)이 되면 학습 분포
  밖에서 두 크레인이 반복 충돌한다. 입력 결측 강건성 문제라기보다 협조 실패의 증폭.
- **checkpoint 선택 노이즈**: val 20-seed 곡선 변동폭 205~227(평균의 1.5배) — 선택이 노이즈에
  민감하고 val→test 일반화 격차 +3~+9. 실재하는 개선 여지지만, 격차(+20)의 주인은 아니다.
- **학습예산**: rollout 은 결정당 최대 64조합 × 600초 미래를 시뮬레이션하는 온라인 계획자다.
  500ep 반응형 정책이 같은 정보로 이를 이기려면 협조를 내재화해야 한다 — 예산 확대만으로
  좁혀질 성질이 아님을 위 분해가 시사한다 (truck_wait 는 이미 대등).

## 4. 권고 (모두 board row 로 등록 — 실행은 별도 결정)

1. **협조 구조 재판정 (YR-013 QMIX 의 착수 근거 데이터)**: §7 재감사 결과, 원인은 누락이
   아니라 **독립 학습자의 조정 실패**로 특정됐다. 이는 정확히 중앙 mixer(QMIX)·joint 표적이
   풀려는 문제다. YR-013 재판정은 사용자 결정 사항 — 본 감사가 그 판단 재료다.
   (대안 경량책: 상대 크레인 의도 feature 추가·resolver 경합 이력 feature — QMIX 전 실험 가능.)
2. **전략적 WAIT 행동 제외 (YR-052)**: 신규 seed 에서 전 정책 손해 + RL 미완주 유발 확정.
   데이터상 권고 = RL 행동공간에서 제외(구조적 양보만 유지). 사용자 보류 중.
3. **checkpoint 선택 안정화 (부차)**: val seed 확대(20→40+)·이동평균 선택 등 — 격차 주인이
   아니므로 우선순위 낮음.
