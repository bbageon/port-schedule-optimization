## 본 세션 추가 발견 (검토 강화 + 처방 정정)

- **greedy→H1 이득(−8.8)의 출처 분해**: truck_wait −5.5·lane_cong −3.8·rehandle
  −1.6·interference +2.0·empty_travel **+0.29(H1 이 더 이동)**. → H1 은 이동을
  줄여서가 아니라 **미래 재조작을 줄여 하류 혼잡·대기를 낮춰** 이긴다. "3층(최소
  이동)"은 목적함수에 거의 무관(이동 1%).
- **검토 처방 1건 정정**: 검토는 "좌우 이동 거리를 비용 원자료에 넣고 재측정"을
  권했으나 — 이동이 1% 라 **이동 상세를 더해도 총비용 거의 안 변함.** 진짜 지렛대는
  하류(lane_cong+truck_wait 83%)이고, 그건 배치의 **다운스트림 효과**라 **rollout
  기반 비용 최적 오라클**로만 측정된다 (이동항 보강이 아님).

## 결론 정정

- **철회**: "H1==오라클 천장 100% → K-후보 RL 무의미(격차 0)." 오라클이 목적함수
  상한이 아니라 **future_blocked-최소화 계열 천장**일 뿐 — 실제 비용을 직접 최적화
  하는 학습/rollout 은 배포형 규칙·이 오라클 **둘 다 이길 여지가 원리적으로 남음.**
- **약화 후 유지**: "배포형 H1 이 이 휴리스틱 계열의 완전정보판을 따라잡음" 까지만.
  당시 H1 채택 근거를 greedy 대비 개선·완주 최견고·다운사이드 0으로 적었으나,
  완주·다운사이드 문구는 아래 후속 일반화 감사에서 폐기했다. 단 **"이 문제 규칙으로
  충분·RL 불필요"는 미증명**이라는 결론과 H1의 잠정 채택은 유지한다.
- **숫자 인용 규칙**: 전 수치에 "미실측 임시 보정(assumed) 하" 단서 필수.
- **0b(강한 오라클) 회귀**도 재해석: 그것도 프록시(future_blocked+이동)라, "위치선점
  나쁨"은 맞되 "비용 최적 배치에 헤드룸 없음"의 근거는 못 됨.

## 후속 일반화 감사 정정 (2026-07-20)

원자료를 selector별로 다시 집계하면 FT+H1은 high·fill0.65에서 18/20만 완주했고,
fill0.70에서 20/20 완주했다. 따라서 위의 "완주 최견고·다운사이드 0"은 더 이상
유효한 근거가 아니다. FT의 greedy·oracle도 fill0.65에서 각각 18/20만 완주해
H1 단독 결함으로 단정하지 않되, 채택 정책의 고적재율 guard는 부분 실패로 기록한다.
상세 seed와 backlog는 [0·1a 결과 감사](2026-07-20-YR-075a-재조작목적지-0단계-prereg.md#일반화-감사-정정-2026-07-20)를 따른다.

YR-075-c도 후속 설계에서 **K후보·30분 지평의 국소 rollout benchmark**로 범위를
좁혔다. 후보 생성과 유한 지평 뒤 base policy에 의존하므로 전역 비용 최적 상한이
아니다. 이 benchmark가 H1을 못 이겨도 검사한 K·지평·조건에서만 H1 충분으로 말한다.
