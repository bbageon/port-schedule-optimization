# YR-061 사전등록 — 미완료 잔존 페널티 (학습 표적 전용) 판정 실험 (2026-07-18)

> 실행 전 동결. 결과·해석은 `## 실행 결과` 절로 append (원문 수정 금지 — 규약 준수).
> 배경: [단일 DQN 11조건 진단 판정](2026-07-18-단일DQN-11조건-진단-보상신용할당-판정.md).

## 1. 가설과 처방

- **진단**: 전 11조건에서 학습 후 SERVE 붕괴(0.04~0.09)·REPOSITION 퇴화. 원인 특정 =
  학습신호(총비용)가 미서비스·미완료를 과소처벌 — 에피소드가 시간 한도에서 끝나면 대기
  청구가 멈추고 미완료 벌점이 없어 "일 안 하는" 정책이 총비용을 게이밍한다. 평가 게이트
  (완료율·건전성)는 이를 탈락시키지만 **학습 중에는 신호로 들어가지 않는 불일치**.
- **가설 H-061**: 학습 표적에만 미완료 잔존 페널티를 넣으면 — 평가 지표·ledger·게이트
  불변 — 퇴화가 사라지고 SERVE·완료율이 회복된다.
- **처방**: `LearnerConfig.unserved_terminal_cost` (기본 0.0 = off). 에피소드 종료 시
  `페널티 × 미완료 job 수`를 종결 구간 비용에 가산(total_normalized 단위, cost_scale 로
  나눠 표적화) → 각 크레인의 말단 SMDP 표본이 자기 결정시점 할인으로 흡수, 부트스트랩
  사슬로 역전파. **checkpoint 선택도 같은 목적함수(val 실제비용 + 페널티×backlog)** 로 —
  게이밍된 지표로 정책을 뽑는 순환을 선택 단계에서도 끊는다.

## 2. 동결 설정 (실행 전 고정)

- 하네스: `src/yard_rl/experiments/yr061_reward_redesign.py` (`run_yr061`, full mode).
- arm = 페널티 사다리 **{0.0(control), 2.0, 5.0, 10.0}** — 소형 시나리오 에피소드 실제비용
  ~65-100·job당 ~4-6 기준으로 "완만→지배" 스펙트럼. control 은 기존 거동 재현.
- 학습: ddqn·lr 1e-3·cost_scale=train 5-seed BaselinePreference fit·γ 0.95·hidden 128 —
  진단이 "기본값 최선"으로 판정한 그대로. 전 arm 동일 초기화 seed(61,000)·동일 탐험
  rng(61,100)·동일 train seed 열 (paired).
- 시나리오: `TerminalGenParams(n_external=16, n_vessels=2)` (나머지 기본 — horizon 4h+drain
  2h·가우시안 변주·ETA ±300s). 소형 = 빠른 판정 루프 (진단과 같은 취지).
- seed 대역 (신규 600k — 기사용 0~250k·300k·400k·500k~560k 배제, config guard 강제):
  train 600000~600149 (150ep, 에피소드당 신규 시나리오), val 610000~610007 (8),
  test 620000~620019 (20). checkpoint 15ep 마다 val 평가, val 목적함수 최소 선택.
- 판정 지표: test 20-seed — 실제(비페널티) total_cost·완료율·backlog·serve_share·
  `serve_when_available`(swa, YR-044 건전성 문턱 0.25)·paired bootstrap (10k resamples,
  seed 75,061). 비교군: SF_SPT·FIFO (동일 후보·resolver·비용 config).
- 비목표: QMIX 경로(joint_sink)에는 페널티 미적용 — 별도 트랙(YR-013/059, 병행 세션).
  본선 λ 중립·전략 WAIT 기본 제외(YR-052) 등 기존 계약 그대로.

## 3. 사전 판정 기준

- **P1 (퇴화 해소, primary)**: 페널티 arm 중 ≥1개에서 test swa ≥ 0.25 **그리고** 평균
  완료율 ≥ 0.95. control 은 진단 재현(swa < 0.25 예상) — control 이 이미 건전하면
  "이 프로토콜에선 퇴화 미재현"으로 P1 판정 불가, 재설계.
- **P2 (성능, secondary)**: P1 통과 arm 의 실제 total_cost paired vs control — 문서화.
  퇴화 정책의 게이밍된 낮은 비용과 비교이므로 **P2 열세는 실패가 아니다** (완주 비용 vs
  미완주 비용). vs SF_SPT 가 실질 기준: 이기면 강한 성공, CI 겹침이면 "회복 성공·성능
  동급", 크게 열세면 "회복은 성공, 성능 추격 과제 잔존".
- **실패 조건**: 전 페널티 arm swa < 0.25 → "미완료 과소처벌이 퇴화 원인" 가설 기각,
  진단 문서의 되돌릴 조건 발동 (상태/후보 표현 재수사).
- 노이즈 주의: 소형 시나리오·단일 학습 궤적 — 수치 몇 단위 차이는 해석 금지, 정성
  패턴(swa·완료율·행동분포)이 1차 판정 축.
