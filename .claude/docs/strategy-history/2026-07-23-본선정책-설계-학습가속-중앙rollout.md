# 본선 정책 설계 v2 — rollout 기반 본선보호 (적대검토 반영) (2026-07-23)

> v1 초안을 4관점 적대검토(코드·보고서 대조)로 정정. **4/4 영역 is_defensible=false** —
> 방향(증류 실패 + rollout 본선보호)은 실측 견고하나 **배포·게이트·선택·RL 주장은 과대**였다.
> 원칙: 근거는 **지지범위 안에서만** 인용, 미측정은 선결실험으로 명시.

## 1. 실측이 견고히 지지하는 것 (코어 — 확정)

- feed-forward+요약벡터(214/246) 증류 실패 — 학생 berth 교사 2~5배·완주 실패·분기재현 0.167.
- 순수 기준재 @3600 **full** rollout 이 SF 대비 **선석초과 −10~−22분 4/4 유의**(berth CI 상한<0).
- 사전식(트럭 무조건 1순위)은 창 5400s 늘려도 배 무의(2a); 같은 창 기준재 목적은 배 유의보호.

→ **본선 보호 기전 = 미래를 내다보는 rollout**(증류된 net 아님). 이 결론만 확정.

## 2. 정정된 주장 (과대 → 지지범위로 하향)

| v1 주장 | 실측 | v2 정정 |
|---|---|---|
| 순수@3600 **트럭 P95 유의개선** | 2/4 셀만 유의(4/4는 평균대기) | "평균대기 4/4·P95 2/4 유의" |
| 증류 **구조적 차단** | 더 큰/그래프 학생 미시도 | "**이 인코딩(feed-forward+요약벡터)에서** 차단·잠정" |
| ~1-2s/결정 | 에피소드 wall 만 있음(미측정) | **선결실험서 실측 전 인용 금지** |
| **배포 정책 본체** | 2크레인·단일블록 장난감 근거 | "**2크레인 공식 성능 기준점 동결**"까지만 |

## 3. 배포 — "본체"는 유보 (스케일 미검증)

−10~−22분은 **2크레인·단일블록·이송3대**에서만. 실터미널 21블록·42기(블록당 2기). "중앙"의
가치=교차블록 YT/STS 결합인데 검증모델에 없음. 42기 공동조합 k^42 폭발·max_combos=64 는 2기서도
이미 절단·조합마다 deepcopy → 규모서 초선형. **∴ 배포 본체 주장은 아래 선결실험(§7) 뒤로.**

## 4. 목적함수 — 방향 맞음, 안전장치·규격 미완

`[하드 제약] + 가중합 경제비용(트럭 1.0 + ρ_vessel·선석초과 + ρ_rehandle + ρ_crane)` 구조 유지. 단:

- **★"하드 제약 필터"는 현재 코드에 없다** — `JointRolloutGreedy.decide` 는 feasibility+비용 argmin
  뿐. 완주·backlog·마감은 ρ_vessel 이 클 때만 나타나는 **비용-창발(soft) 결과**(rholow v0=완주실패·
  backlog6). 모델에 "위반 불가 하드마감" 개념 자체 부재(planned_completion 은 비용 적립 soft anchor).
  → **도면을 soft 로 정정**하거나, 비용과 독립된 **admissibility 필터**(물리 최소완료 ≤ 마감 위반 조합
  제거·미완 backlog 조합 제거)를 **실제 구현**. "도달가능성" 판정식(물리 최소완료시간) 먼저 정의.
- ρ_vessel: "보호시작 ~4·33 고원"은 **high 셀만** 지지 — mid 미측정(하필 트럭 P95 악화 셀)·LEX/full
  대비(SELECTIVE 아님)·v0 완주절벽(불변 하한 ~1-4). → 불변구간을 **[하한, 상한]으로 정직 한정**.

## 5. 트럭 게이트 — 서술어 → 검정 규격 (현 n=10 부족)

"비열등"만으론 목적이 트럭을 게이트 경계까지 갉음(트럭 깎아 배 삼). 규격화 필수:
- 고정 **SF anchor** + 비열등 마진 **δ**(예: 트럭 P95 +2분) 사전등록 + **단측 non-inferiority 검정**.
- **현 n=10 검정력 부족** — P95 CI 폭 ~13분·2/4 셀 0 포함 → 수 분 열화도 "비열등" 통과. δ 탐지 필요
  seed 수 사전계산. **P99·max 도 게이트에**(P95 이후 꼬리 무방비).

## 6. 선택적 rollout — 결정 아님, 열린 실험으로 격하

현 SELECTIVE 는 배포근거 0(firmup 에 arm 없음·발동창 1800·paired CI 없음). 실측 불리:
- 발동률이 **압박 tight 셀서 0.87~0.93**(선택성이 full 로 수렴)·벽시계 절약 tight 서 ~7-8% 무의.
- "위험 놓치면 full 복귀"는 **순환**(놓친 위험 감지할 독립 detector 없음)·SYMPTOM 본선(planned
  결측)은 trigger·비용 양쪽 **사각**. full@1800 은 SF 보다 berth 나쁨(4/4) → 발동창 3600 필수.

→ **채택 금지**. `SELECTIVE@3600` 을 firmup(4셀×10seed·paired CI)에 정식 arm 으로 넣어 gated 유의성
확보 뒤에만. tight 발동률 >0.7 이면 full 채택(사전등록). SYMPTOM → **무조건 발동**(안전측).

## 7. RL 역할 — 안전영향으로 4분할 ("가속기" 한 우산 금지)

"resolver 가 검증"은 feasibility 만 잡지 본선열화·최적탈락은 못 잡는다. 역할별:
- ① **후보정렬** = 안전 가속기(rollout 이 생존자 전수평가) — 채택.
- ② **가지치기** = 증명가능 지배·보수형만(학습된 "명백 나쁨" 컷 금지 — 별칭이 최중요 분기서 최대
  오류). **beam==전수 회귀테스트 의무**.
- ③ **잔여비용 추정** = 가속기 아님, **목적변경** — argmin 에 편향 유입 시 full rollout 도 오선택.
  held-out 검증 + 채택게이트(선석·P95 비열등) 뒤에만.
- ④ **발동예측** = fallback 있음 — 채택.
- 집합/그래프 입력 = "가속기/강등" 아님, **YR-081 급 그린필드**. 별칭이 값/랭킹 타깃에서 풀린다는
  측정 선결(현 별칭 진단은 분류에만 존재).

## 8. 선결 실험 (배포 주장 전 필수) — YR-087

1. per-decision 벽시계·n_decisions 직접 로깅 → 2기 "1-2s" 실측.
2. 크레인 4→8·블록 2→4 규모 스윕 — 조합수·절단율·결정지연·본선우위 **감쇠곡선**.
3. **중앙 vs 블록분해 rollout 대조**(교차블록 YT/STS 결합 손실 정량).
4. `SELECTIVE@3600` firmup arm(paired CI) — gated 정책 유의성.
5. ρ_vessel∈{11,33,99} × **mid 포함** × SF 기준 × SELECTIVE 재확인(게이트 breach 여부).
6. admissibility 필터 구현 or §4 soft 정정 · train-fit 0.192 산출 아티팩트 박제(재현성).
