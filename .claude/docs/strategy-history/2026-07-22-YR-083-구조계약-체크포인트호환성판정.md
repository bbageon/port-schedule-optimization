## 3. 판정 근거 (코드)

채택 FT(`CentralJointValuePolicy`, student_ft.pt)는 **2크레인 대칭·SHARED 역할·214입력 고정**이다
(`encoding.py`: 크레인별 g·yc·queue·cand 텐서, 역할 feature 없음 / `candidates.py`: 후보 생성에
역할·차종 제약 없음 — 두 크레인이 같은 후보풀).

- **수평형 ZERO_SHOT**: 계약이 대칭 단일블록 2크레인(SHARED×2·YT·assumed 기하) **실행 substrate
  로 환원** → 입력 텐서 차원·의미 불변이라 동결 FT 무재학습 실행. 확인됐으나 엔진에 자리 없는
  구조필드(HJNC orientation=수평·block_count 21)는 **`engine_ignored` 로 명시 기록**(조용한 유실 0,
  실행 불변). "무손실"이라 하지 않는 이유 = 그 필드들은 엔진 모델이 거칠어 소비 못 할 뿐 실재하기
  때문(적대검증 반영). PNIT 는 확인값 오버레이(열폭 2.84)로 궤적이 base 와 달라지나 텐서는 불변 →
  여전히 zero-shot (golden 완전동일은 HJNC 열폭 3.1=base 에 한함).
- **DGT 현재 UNSUPPORTED**: 육측(외부트럭)/해측(AGV) **역할 mask 미구현**, AGV→WSTP **이송 과정
  자체가 엔진에 없음**, 인계점 용량·제어 미모형. 지금 넣으면 이 구조가 dead field 로 조용히
  무시돼 "DGT에서 검증"이 거짓이 됨 → 실행 금지.
- **DGT 목표 SCHEMA_ADAPTATION(재학습)**: 엔진이 역할을 지원해도 고정 FT 는 zero-shot 불가 —
  (1) 두 크레인 슬롯이 대칭→비대칭(육/해 역할타입)으로 바뀌어 대칭 학습 분포를 벗어남,
  (2) 해측 크레인의 작업은 AGV/본선이라 **FT의 1차 목적(트럭 대기)이 적용되지 않음**(의미 붕괴),
  (3) 입력에 역할 신호가 생기면 214입력 차원·의미가 변함. → 재증류/재학습 경로.
- **S/C·북항 UNSUPPORTED**: physics_base 없음(bay/좌표 미확보)이라 실행 프로파일 자체 불가 +
  S/C 이동·인계는 YT/AGV 와 역학이 달라 별도 과정 필요.

## 4. 적대적 검증 (판정 3건 반박 시도 — 코드 근거)

병렬 회의자 3명이 encoding·joint_distill·candidates·compiler 를 읽고 각 판정을 반박 시도 →
**3건 전부 upheld(반박 0)**. 라벨은 코드 경로로 확증. HJNC 검증관이 정당한 균열 하나를
지적(orientation 확인값을 compiler 가 조용히 버림 = 자기규약 위반) → **`engine_ignored` 버킷
신설로 수정**(확인 구조필드를 실행불변이라도 명시 기록, ZERO_SHOT 은 유지). DGT 두 판정은
high confidence — schema.py yc 그룹에 육/해측 채널 부재 + 증류 표적 `tier_a`(트럭대기 전용) +
214입력이 vessel 그룹(sts_wait·remaining_moves·마감) **통째 제외**(22+96+96=214 검산)로
"해측 크레인 목적 미학습" 확정.

**체크포인트 실측 검증**: 동결 FT `in_dim=214`(스키마 유도값 일치) 로드 후 HJNC·PNIT stress
프로파일에서 `run_joint_episode` 실행 — **완주 1.0, 차원오류 0** → zero-shot 텐서 호환을
구조적 주장이 아니라 체크포인트로 확인. (대기 절대값 HJNC 3.975/PNIT 2.051 은 현 엔진
YR-080 1단계 반영분이라 성능 주장 아님 — 재판정은 YR-080 2단계.)
