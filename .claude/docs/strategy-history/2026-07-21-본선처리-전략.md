# 본선 처리 전략 — YR-080 본선 축 (개정 v5: 기준재 상대비용)

> 2026-07-21. 개정 이력: v1(mandatory 단일원인) → v2(하드 절벽/필터) → v3(선석 초과 **경제비용**
> 재정의) → v4(행동공간 STORE/RETRIEVE 통일 + 실행 must-fix) → **v5(비용 단위 = 실화폐 아닌
> 기준재 상대비율 — 문헌조사 확정, 사용자 승인)**. 검증: 아래 코드 line·YR-080d/e.
> 핵심: **본선은 새 행동이 아니라 SERVE 후보의 한 종류.** RL 은 트럭·본선의 **경제적 우선순위만**
> 배우고, 실행계획·목적지·YT 준비·안전은 기존 결정론 계층이 담당. 트럭 정책 폐기 없이 통합비용 확장.

## 1. 행동공간 — 새 본선 action 불필요, 물리는 STORE/RETRIEVE 2모드

행동 종류는 SERVE / PRE_REHANDLE / REPOSITION / WAIT 그대로. 본선 작업은 **SERVE 후보**로 처리.
야드크레인 관점에서 4개 업무는 물리적으로 **2가지**로 통일된다:

| 작업 방향 | TRUCK | VESSEL |
|---|---|---|
| **STORE**(인계점→스택) | 트럭 반입 | **본선 양하** |
| **RETRIEVE**(스택→인계점) | 트럭 반출 | **본선 적하** |

**데이터 구조** (새 action 대신 속성 추가):
```
action_kind    = SERVE                  # 정책 행동은 하나
service_mode   = STORE | RETRIEVE       # 물리 실행 모드 (이동방향·계획 생성이 다름)
requester_type = TRUCK | VESSEL         # 업무 정보 (비용·통계·인계용, 삭제 금지)
```
→ 신경망은 같은 물리작업을 공통 학습하되, **우선순위는 비용으로 다르게 판단**(트럭=누적 대기비용,
본선=예상 선석 초과비용). 물리 실행은 동일 적재/반출 계획기, 안전은 동일 resolver.

## 2. 목적 — 선석 초과 경제비용을 **기준재 상대비율**로 (v5 확정)

겨누는 것은 해상 위험이 아니라 **접안 후 터미널 작업이 계획 선석 종료를 넘긴 경제손실**. 단
**실화폐(원)로 쓰지 않는다** — 문헌조사(2026-07-21) 결론: 실화폐는 소수파(자연 가격출처 있을 때만),
지배 관행은 상대 계수(Imai 2003 TR-B 가중시간·Meisel&Bierwirth 2009 TR-E 명목 cost rate 15년 재사용).
이론 근거 3겹: 경제학 기준재(numeraire)·OR 임계비율·RL 보상 양의 선형변환 불변(Ng et al., ICML 1999).

```
통합비용 = Σ 트럭대기시간                    [기준재: 계수 1.0 고정]
         + ρ_vessel  × Σ 선석초과시간         [ρ = 접안료요율 ÷ 트럭 시간비용 — 공개요율 anchor]
         + ρ_rehandle × 재조작 횟수           [재조작 1회 = 크레인 작업시간 등가 환산]
         + ρ_crane   × Σ 크레인 이동·운영시간  [장비 운영단가 비율]
초과시간 = max(0, 실제 작업완료 − 계획 선석 종료)
```

RL 은 "트럭 먼저? 본선 먼저?"가 아니라 **두 선택의 상대적 경제손실을 비교**(마감 3h+작업적음→트럭 /
20분+작업많음→본선 / 이미 초과→우선순위 급상승).

**임의 스케일 함정과의 차이 (방어 조건 2 — 필수)**:
1. **비율 근거 명시** — ρ 마다 출처·산출식·중심값 표 (공개요율: 부산항만공사 접안료 GT·시간당 ÷
   화물차 안전운임 시간환산). 근거 없는 가중치는 문헌의 공인 약점(JCDE 2023 리뷰).
2. **비율 민감도** — ρ 를 ×⅓~×3 그리드로 흔들어 **정책 우열 안정성** 보고. 뒤집히는 구간은 명시.
+ **결정 비율 vs 학습 스케일 분리** — 보상 전체 상수배(정책 불변)로만 학습 스케일 조정, 정규화
  상수가 결정 비율을 겸하지 않게(과거 함정의 재발 방지). 항별 비용 기여 분해 상시 보고(지배 탐지).
+ 원화 해석 필요 시 anchor 요율 곱해 복원 가능(문서화 — 정규화 유효조건 충족).

명명: `본선 위험도`→`선석 사용시간 초과 위험`. `vessel_deadline_mult`=현실 입력 아님·스트레스 실험만.

## 3. 반드시 고칠 것

**정의·계약**
1. 선박별 **실제 접안·계획 작업완료·선석 종료시각** 명시.
2. **선석 초과 실비용을 총 터미널 비용에 포함**.

**코드·실행 (검증됨)**
3. **본선 양하 방향 역전 수정** — 현재 scenario_gen.py:276 이 VESSEL_DISCHARGE 에도 기존 야드
   `target_container` 를 넣어(free_targets.pop) engine.py:374(GATE_IN 만 STORE) 밖의 **반출 경로**로
   처리 → 양하가 반대로 모델링됨. 수정: `VESSEL_LOAD=야드 반출→YT`, `VESSEL_DISCHARGE=YT 인수→
   야드 신규 적재`. 엔진 분기를 `flow==GATE_IN` 이 아니라 **`service_mode==STORE`** 기준으로
   (트럭 반입 ∪ 본선 양하 공통 / 트럭 반출 ∪ 본선 적하 공통).
4. **인과 연결 (crux)** — `YC 본선작업 → YT/AGV 이송 → 안벽 버퍼 → 안벽크레인 → 선박 작업량 감소 →
   실제 완료시각 → 선석 초과비용`. 안 연결되면 야드가 본선을 먼저 해도 최종 비용 불변 → RL 이 무엇이
   지연을 줄였는지 못 배움(YR-080d 가중치 1→4 행동동일의 진짜 이유; 현 disconnect: kpis.py:86 야드지각
   vs engine.py:741 VesselProcess 완료지연 vs :685 독립진행).
5. **신규 적재 위치 규칙** — 채택 H1(engine.py:399)은 **방해물 재조작 위치**만 고른다. 트럭 반입·본선
   양하의 **최초 STORE 위치**(engine.py:374 find_slot)에도 미래 출고순서·본선계획 기반 **비방해 규칙**
   적용. 두 규칙(재조작 목적지 / 신규 적재 목적지) 모두 초기 규칙기반, 개선 여지 확인 시에만 RL 확장.
6. **YT/버퍼 준비 = 실행가능성 마스크** — 후보 생성 **전** 조건: `본선작업 준비완료 = 컨테이너 준비 +
   YT/인계버퍼 준비 + YC 실행계획 가능`. 준비 안 된 작업은 후보에서 가리거나 RELEASED 안 함. **RL 학습
   대상이 아니라 실행가능성 조건** (기존 안전층은 YC 물리충돌만 막고 YT 흐름은 자동 보장 안 함).

## 4. RL 이 배우는 것 = 경제적 우선순위만 (나머지는 기존 결정론 재사용)

RL 몫: 트럭 누적 대기비용 vs 예상 선석 초과비용의 **우선순위**.
기존 결정론 계층 그대로 재사용: 반출·반입 실행계획 · 재조작/신규적재 목적지 · 적재높이·규격 확인 ·
서비스범위 · 동일작업 중복차단 · 통로 충돌차단 · 슬롯/레인 예약 · 안전거리 · resolver dry_run 공동
실행가능(resolver.py:69) · 실행 중 재검사(engine.py:818).

## 5. 상태·후보 신호 (v3)

- **선석 초과 위험(여유)**: `예상 완료 = 현재 + 남은 본선작업 예상 처리시간`, `여유 = 계획 선석 종료 −
  예상 완료`(+60분=여유·+5분=위험·−30분=이미 초과 예상). 후보별 `이 작업 선택 시 줄어드는 예상 초과
  비용`(현재 빈 vessel_risk_delta 에). 공동가치망 입력에 전달(현재 DecisionEncoding 미포함).
- 여러 선박이면 평균 아닌 **가장 위험한 선박** 보호 (Liu·Wan·Wang, NRL 2006).

## 6. 정책 방향·검증·순서·요율

- **트럭 정책 확장**(폐기 아님): 통합비용으로 교사 재수집→증류→반사실 FT 재검증. 교사 라벨 = (본선
  초과비용, 트럭 SLA 초과, 트럭 대기, 이동·재조작). listwise 증류 유지·보조회귀만 다중출력.
- **검증**: ①계약(학습비용=보고 KPI 동일정의) ②인과(YC 늦추면 선박 완료 실제 늦나) ③정확해(소셀
  전수/CP·MILP) ④절제(가중치만/상태만/필터만/재학습) ⑤성능(본선 정시율·최대지각·QC 유휴 + 트럭
  평균/P95 동시) ⑥완주·backlog·위반 0 ⑦강건성. (표본 충분 시 CVaR — Chow, JMLR 2018 — 현 소수선박엔 이름.)
- **순서**: YR-080(양하 방향·인과·신규적재규칙 수정 → 통합비용 재학습) → YR-075-c → YR-041(신규 seed
  locked, 재수정 금지) → YR-042.
- **요율**: 접안료 12h 기준 + 초과 GT·시간당(부산항만공사 공개). 실 부담엔 계약 페널티·연장비·다음 선박
  기회비용 별도 → **실 계약자료 우선, 없으면 공개요율 assumed 명시**.
