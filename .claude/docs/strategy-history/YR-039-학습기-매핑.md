## 1. 정책 골격 (최종전략 §계약 승계)

- `Q_cost(global, yc, queue_summary, candidate) → 스칼라` — **후보별 독립 점수 공유망**
  (YR-031-b: 집합 요약은 필요, 후보 독립 구조는 충분). K_max 후보를 같은 망으로 채점.
- 선택: resolver `Preference` seam 에 **QPreference** 주입 — rank = `(Q값, *Baseline
  rank)`. mandatory-우선·feasibility·dry_run 오라클은 resolver 골격 그대로 (안전은
  학습 밖, YR-034 계약).
- **미학습 계약**: 출력 head zero-init → 전 후보 Q=0 → tie-break 로 rank 가
  BaselinePreference 와 완전 동일 → **미학습 정책 ≡ 기존 baseline resolver** (테스트 고정).

## 2. 인코딩 (contract → tensor)

- FeatureVector 3채널 계약 준수: 그룹별 `x = [value ⊙ known ‖ known]` (결측 중화 +
  결측지시자). 그룹 = global / yc / queue_summary / candidate(K_max 행).
- 마스크: CandidateSet.pad_mask(패딩)·feasible — 채점은 전 행, **argmin·batch min 은
  feasible∧pad 만** (masked +inf).
- 차원은 SCHEMA_VERSION 의 fv.names 길이에서 유도 (스키마 변경 시 자동 추종).
- 선박 per-vessel 벡터는 v1 미사용 — queue_summary.vessel_urgency_max·global 집계가
  대변 (ablation 은 YR-014). vessels 가변길이 인코딩은 후속.

## 3. 학습 (Stage B — 본 문서로 동결)

- **SMDP per-crane 전이 스티칭**: TransitionRecord 열에서 크레인 c 의 연속 결정
  (k, next_k) 사이 구간비용을 할인 합산 `C = Σ γ^{(t_i−t_k)/ref}·cost_i.total`,
  할인 `γ^{(t_next−t_k)/ref}` (ref=60s, γ=0.95 승계). 팀 공유비용을 각 크레인에
  동일 귀속 (독립 학습자 v1 — mixer 는 YR-013).
- 대상 표본: 실행된 (crane, candidate) 만 (WAIT/yield 결정은 v1 제외 — 문서화).
- **Double DQN 기본**: y = C + γ_dt · Q_target(s', argmin_{feasible} Q_online(s')).
  variant: DQN(target argmin=target)·Dueling(V(ctx)+A(cand)−mean_feasible A). 동일
  조건 비교 (spec).
- replay uniform·target sync N step·grad clip·Adam. GPU 있으면 사용, CPU parity
  테스트 (결정론: CPU 단일스레드 기준 재현, GPU 는 성능용).
- checkpoint 프로토콜 = **YR-033**: 선택은 validation 만 (대표본 90일급), test 절대
  미사용. device-independent 저장 (state_dict + scaler/None + config).

## 4. 시나리오 생성기 (integrated/scenario_gen.py)

- 단일야드 GenParams 대응 `TerminalGenParams(n_external, n_vessels, fill_ratio,
  rehandle_risk, horizon_s, drain_window_s, …)` — seed 결정론 (RNG 는 생성기에만,
  엔진은 결정론 입력 소비 — YR-036 계약).
- 산출: 초기 스택(충돌·공중적재 없음), 외부트럭 반입/반출 도착열, 본선
  DISCHARGE(RISK)+LOAD(SYMPTOM 혼합) + 대상 컨테이너·연계 Job, injected 없음(기본).
- 검증 테스트: 같은 seed 재현 + `record_episode` 완주 (validate_all·invariant 통과).

## 5. 평가 (Stage B)

- baseline: BaselinePreference resolver (동정보 강 휴리스틱) — validation 에서
  SPT-형 preference 후보와 비교해 낮은 쪽 선택.
- locked test paired bootstrap 10,000 — **총비용(cost.total 누적)** 1차 지표,
  mean/p95 대기·본선지연·guardrail (완료·backlog·invariant·P95≤+5%) 동시 보고.
- seed band: train 300000+N / val 310000+ / test 320000+ — 기존 전 대역과 분리 guard.

### 해석과 한계

1. **지형이 바뀌니 결론이 바뀐다**: 단일 YC·평균대기에서는 SPT 가 near-optimal
   (트랙 종결)이었으나, 다중 크레인·본선·레인·REPOSITION/PRE_REHANDLE 이 있는
   통합 지형에서는 순차 그리디 휴리스틱이 크게 준최적 — RL 이 큰 마진으로 승리.
2. **한계 (정직)**: (a) baseline 휴리스틱 2종은 통합 지형용으로 튜닝된 강자가
   아님 — 마진의 일부는 상대의 약함일 수 있음. 강화 휴리스틱/beam oracle 대조가
   후속 가치. (b) 비용 scale 전 항목 assumed placeholder (YR-038 — YR-002 후
   실측). (c) 합성 시나리오 + POC-MULTI 가정 프로파일 — 실운영 주장 불가 (YR-009 전).
3. DDQN 이 DQN 보다 낫지 않음 (과대추정 편향이 이 지형에서 크지 않음) — Dueling
   의 V/A 분해가 가장 유효. 기본 채택 후보 = **dueling**.
