# YR-030 — 계열 2 (Direct-Job Cost-Q) baseline 승격 결정

> 기록일: 2026-07-14 · 상태: **방향 결정 (사용자) — 구현 전** · 상세 보강: 2026-07-14 (작성 규약 반영)
> 관련: [YR-027 v1](2026-07-13-YR-027-exp1-direct-job-cost-q.md) · [v2 최소상태](2026-07-13-YR-027-exp1-direct-job-cost-q-minimal-state.md) · [YR-028 사전등록·결과](2026-07-14-YR-028-coverage-ablation-prereg.md)

## 1. 결정

신규 RL 실험의 baseline 정책 구조를 **계열 2 (Direct-Job Cost-Q)** 로 지정한다:
매 결정에서 실행가능 후보 Job 들의 `(GlobalState, CandidateFeature)` 스칼라 비용을
비교해 argmin 을 고르는 구조. 이후 실험은 이 골격 위에서 **상태 feature·파라미터·
학습설정을 추가**하며 진행한다.

계열 1 (rule-선택 tabular QL)은 **폐기가 아니라 동결**: Exp-1~3·YR-018·YR-023 의
PoC 증거(H1 예비지지, w_tail negative 등)는 유효한 실험 기록으로 보존·인용하되,
신규 실험 라인에서 제외한다.

## 2. 결정 경위 (타임라인)

1. **2026-07-12~13**: 계열 1 로 Exp-1~3C 완주. Exp-1 은 FIFO 대비 평균대기 -15.2%
   (12/12 seed 유의)로 예비 지지, 그러나 P95 +10.8% 악화 trade-off 관찰.
2. **2026-07-14 YR-023**: ARMG 프로파일 재실행 — 방향 유지(-10.4%)·P95 악화 확대
   (+17.4%, 유의). P95 문제가 프로파일 강건.
3. **2026-07-14 YR-018**: w_tail {0,0.1,0.3,1.0} × 예산 {e4,e10} × 프로파일 {POC,HJNC}
   전 grid 에서 **인접 가중치 유의차 0건**. 결정적 기제: w_tail 은 자기 지표(tail_area,
   SLA 초과 총량)를 실제로 개선(HJNC -6~-17%)하면서 p95 는 +17~+35% 악화 —
   **총량 적분 지표는 극단 분위수를 보호하지 못함 (지표 불일치)**.
4. **사용자 판정 (2026-07-14)**: "이 메커니즘은 폐기하는 게 맞아 — 상태마다 어떤
   액션을 선택해야 하는지 트레이드오프가 없고, 기준이 모호하잖아?" 이어서
   "지금은 계열 2 를 baseline 으로 지정해서 파라미터 및 학습설정을 추가하면서
   실험을 하는 게 맞아" — 본 결정의 직접 지시.

## 3. 근거 — 사용자 논리와 데이터 정합

**계열 1 (rule-선택)의 구조적 문제:**
- 정책 클래스가 "9개 priority rule 중 선택"으로 제한 — rule 이 표현하지 못하는 job
  선택은 학습이 완벽해도 접근 불가 (표현력 상한 = rule 어휘). 새 능력마다 rule
  수작업 추가 필요 (실례: Exp-3 용 EPA·PRE_REHANDLE rule 신설).
- 상태별 행동 간 트레이드오프가 학습 신호로 나타나지 않음 — YR-018 에서 가중치를
  10배 바꿔도 상태별 Q 순위가 불변(유의차 0)이 실증. 벌점이 (a) 희소하고 (b) 어떤
  행동을 하든 공유되며 (c) oldest_wait 4구간 상태로는 조건화 자체가 불가.
- rule 간 의미 중복(같은 job 을 고르는 경우 다수)으로 "어떤 rule 을 왜"의 기준 모호.

**계열 2 의 구조적 장점:**
- 결정 기준이 명시적: 후보별 예상 비용 비교 — 후보 간 트레이드오프가 결정 단위에서
  직접 표현·학습됨.
- 함수근사 정합: 구현계획 02 §5.2 의 함수근사 관측(`candidate_features[K,F_job]` +
  `candidate_valid_mask`)과 동일 골격 — YR-012 전환 시 테이블→네트워크 교체만.
- coverage 통제 수단 확보: YR-027 v2 가 최소상태로 fallback 55.04%→0.01% 실증,
  YR-028 이 "v1 상태도 gate 도달 가능(최소 0.5%)·주범은 선택규칙" 판정 — 남은
  병목은 **학습 Q 의 순서 품질**로 좁혀짐.

## 4. 검토한 대안과 기각 사유

| 대안 | 내용 | 기각 사유 |
|---|---|---|
| A. 계열 1 유지 + hard-guard 만 추가 | rule-선택 지속, P95 는 mask 제약으로 방어 | 기준 모호성·표현력 상한이 본질적으로 남음. P95 는 지켜도 rule 어휘가 개선 상한을 계속 결정 — 사용자 지적의 핵심 미해결 |
| B. 곧장 YR-012 (DQN/PPO) 직행 | tabular 를 건너뛰고 함수근사로 | 순서품질·coverage 문제의 **원인이 미규명**인 채 신경망으로 가면 실패 시 원인 분리 불가. tabular 계열 2 에서 원인을 규명한 뒤 같은 골격으로 전환하는 것이 디버깅 가능한 경로 |
| C. 보상형 P95 페널티 지속 (비선형/분위수) | 벌점 모양 변경 | YR-018 기제(희소·공유 벌점 + 상태 조건화 불가)가 그대로 적용 — 같은 함정. 사용자 명시 폐기 |

## 5. 실험 축 — "파라미터·학습설정 추가"의 구체화

| 축 | 내용 | 근거·주의 |
|---|---|---|
| 상태 v3 | v1(도달가능 ~17.6k 서명, YR-028 실측)과 v2(~120 키) 사이 — 후보 맥락(대기열 구성) 보강 후보 | 수렴진단 지표(fallback/thin) 동반 필수. YR-028 결과: coverage 는 병목 아님 — **순서품질 개선이 목적** |
| 학습설정 | α=n^-p (p grid), ε 스케줄, checkpoint 선택 = **R2(coverage-gate)** 채택 | YR-028 판정 반영 — R1(mean 최소)은 greedy 혼합 편향 |
| 초기화 | Q0=0 낙관 초기화 → **greedy 비용 추정치 ĉ prior** 검토 | 순수 Q 열세(YR-028 단조 표)의 1차 용의자 — "greedy 에서 출발해 개선분만 학습" 구조 |
| 제약 | SLA 임박 시 후보 필터 (P95 보호 — YR-029 재편성분) | 보상형 폐기에 따른 구조적 대체 |
| 평가 계약 | paired·locked test·guardrail(P95/completion/backlog)·강 휴리스틱 비교군 유지 | YR-027 계약 승계, seed band 는 실험별 신규 동결 |

## 6. 적용 범위와 되돌릴 조건

- **적용**: 신규 RL 실험 전부. 계열 1 산출물(Exp-1~3·YR-018·YR-023 리포트, replay 등)은
  증거로 유지하며 인용 시 "동결된 계열 1 기준" 임을 명시. UI replay 는 양 계열 지원 유지.
- **되돌릴 조건**: 계열 2 확장 실험(상태 v3·greedy-prior 포함)이 **연속 2회의 사전등록
  실험에서 shortest-service 를 유의하게 넘지 못하면**, "후보 단위 tabular 스코어링"
  자체의 한계로 보고 YR-012(함수근사) 직행 또는 문제 재정의를 재논의한다.
