## 1. 질문 (G0)

**"이길 여지가 있는가"** — NEW 계층목적을 최적화하는 중앙 공동계획기(JR_NEW)가
문헌 보정 v2 지형(현실 부하·피크)에서 SF-SPT 보다 트럭 대기를 일관되게 줄이는가.
통과 시에만 YR-073(증류) 착수. 실패 시 목적 재설계 재론 — knob 구제 금지.

## 2. NEW 계층목적 정의 (동결 — `integrated/objectives.py`)

- 비교 단위: rollout 시간창(600s) 누적 **항별 기여** `CostBreakdown.contributions()`
  (13항, RC=assumed_default — 항 내부 정규화·가중은 OLD 와 동일 상수 유지, 바꾸는
  것은 **항 간 비교 방식**뿐).
- **사전식(lexicographic) 4-tier 비교** (엄격 — 허용오차 없음, 반올림 9자리):
  - tier-A `truck_wait` — 트럭 누적대기 (1차 최적화)
  - tier-B `long_wait` — 긴 대기 (SLA 초과 구간)
  - tier-C `vessel_delay + depart_delay` — 본선 마감·지연
  - tier-D 나머지 9항 합 (rehandle·crane/empty_travel·interference·lane_cong·
    resequence·imbalance·sts_wait·transfer_wait) — 하위 운영비용
- tier-①(안전·미완료·backlog)은 창 내 비교 항이 아니라 **구조 보장**이다: 안전·물리는
  resolver mask, 완료는 mandatory+마감 강제 (YR-061 실측: 미완료 페널티 무발동).
  에피소드 판정에서 완료율·backlog guard 로 별도 검증.
- 원칙 구현: 사전식이므로 하위 tier 가 아무리 좋아도 상위 tier 열세를 뒤집을 수 없다.
- 미분류 신규 항이 생기면 tier-D 로 합산 (누락 방지 안전망).

## 3. 처방·비교군 (동결)

- `JointRolloutGreedy(objective=hierarchy_key)` = **JR_NEW** — 기존 JR 와 창(600s)·
  base_policy(SF-SPT)·max_combos(64)·후보·resolver 전부 동일, 조합 비교 key 만 교체.
  `objective=None` 기본 경로는 바이트 불변 (기존 JR·BEAM·골든 보존 계약).
- 지형: `build_calibrated_profile()` × `calibrated_load_params(level)` — mid(56)·
  high(80), 피크 2배·gate 210s. **YR-072 와 동일 지형** (OLD 대조 재사용 가능).
- arms (NOISY eta±300s): SF_SPT · JR_OLD(scalar) · JR_NEW(계층). PERFECT(eta 0):
  SF_SPT_P0 · JR_NEW_P0 — "ETA 가치 0" 의 NEW 재측정 (보고 축, 판정 아님).
- seed: **수준별 20개 신규** — mid 720000~720019 · high 720100~720119 (미사용 대역).
- 계산시간: 정책별 에피소드 wall-time 기록 (조건 3 — JR_NEW 채택 가능성 판단 재료).

## 4. 판정 기준 (동결)

- **G0 (primary)**: paired Δ평균대기(JR_NEW−SF_SPT) bootstrap 10k(seed 75168)
  95% CI **상한 < 0** 이 **mid·high 둘 다** 성립. 보고는 절대차(분)+CI (백분율 금지).
- **guard (전부 필요)**: ① 완료율 전 seed 1.0·backlog 0 ② P95 대기 paired Δ
  유의 악화 없음 (CI 하한 > 0 이면 실패) ③ 행동분포 보고 — swa·SERVE/REPO share
  (JR_NEW 의 REPO 급증+SERVE 급감 = 퇴화 경고, `assert_healthy_action_mix` 통과)
  ④ 결정론 — 수준별 선두 2 seed 재실행 바이트 동일.
- **부고 축 (판정 아님)**: JR_OLD 대비 행동분포 이동(SERVE↑·REPO↓ 예측), 완벽정보
  이득(NEW 기준), OLD 총비용 병기 (OLD 자로 재면 JR_NEW 가 나빠 보일 수 있음 — 예상 자체를 박제).
- **의미 제한 (조건 4)**: 통과 = "트럭 대기 목적 정렬 + 현 행동공간 헤드룸 확인".
  본선 보호 검증 아님 (본선지연 0 시나리오 — YR-041 별도).
- **실패 조건**: 어느 수준이든 G0 미달 → YR-073 착수 금지·목적 재설계 재론 박제.
  ε·tier 재구성 등 knob 조정으로 구제하지 않는다.
