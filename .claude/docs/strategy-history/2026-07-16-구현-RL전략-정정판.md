## 1. 목표와 현재 위치 (쉬운 말)

**목표는 하나다**: 사전 반출입정보·차량 ETA(도착예정시각)·컨테이너 장치상태·본선 작업계획·레인·
다중 야드크레인(YC) 상태를 한 번에 보고, 각 크레인의 다음 작업을 협조적으로 골라 **터미널 전체의
정규화 누적 운영비용을 최소화**하는 단일 통합 정책. 기능별(Exp-1~4) 정책을 따로 만들지 않고, 같은
정책에서 요소를 끄는 ablation(제거 실험)으로 기능 효과를 잰다 (YR-034, 사용자 결정).

**현재 위치**: 토대(계약→환경→후보/공동배정→비용→baseline)는 전부 가동 중이다 — 계약~비용

첫 학습 실험(YR-039)은 형식상 이겼으나 **무효**가 됐고, 그 원인 두 가지(목적함수 오염·비교상대
퇴화)를 코드에서 제거했다. **다음 한 수는 YR-045 — 정정판으로 잠금(locked) 재실험**. 단 착수 전
**YR-048**(통합 시나리오에 ETA 주입 — 없으면 선제정리 축이 통째로 비활성인 채 판정) 처리 권장.

> REPOSITION도 3.3배 늘어 두 효과를 분리해야 한다. SERVE 없는 한산기에 ETA만으로 결정을 여는

## 2. 변하지 않는 골격 (동결 계약)

| 축 | 동결 내용 | 근거 |
|---|---|---|
| 문제 형식 | 이벤트 기반 SMDP(작업마다 길이가 다른 준마르코프 결정) — 계약은 경과시간 `dt_s`만 저장, 할인 `γ^(dt/ref_s)`(γ=0.95·ref=60s)는 학습기가 파생 | 형식은 원본 §5, ref=60s 는 YR-039 매핑 §3 동결값(원본 §5 의 기준시간 5분을 조정) |

## 3. YR-039 가 가르친 것 → 정정된 설계

무효 사유는 둘이었다. ① 총비용의 97.6~99.9%가 잘못 구현된 불균형(imbalance) 항 하나였고(누적
완료건수의 pstdev(모집단 표준편차) × scale=1 방치 → 트럭 대기의 300배 스케일), ② 비교상대인
SPT(예상 처리시간이 가장 짧은 작업 우선 규칙)가 결정의 54%(val 5-seed)~81%(test seed)를 "일 안
하고 이동"에 쓰는 퇴화 정책이었다. **비교 기준이 무너지면 어떤 승리 주장도 성립하지 않는다.**

| 정정 | 내용 (정확한 현행 정의) | 재발 방지 장치 |
|---|---|---|
| 목적함수 (YR-043) | imbalance = 누적 완료건수 pstdev **폐기** → 현재 작업부하 `I(t)=(max−min)/ΣLoad ∈[0,1]`, rate=I/T_shift. feature 와 cost 가 같은 정의 공유. 실측: episode_raw 14,779.4→0.028 | **지배도 guard**: 단일 항 기여율 >70% → `CostDominanceError` 로 run 자체 실패 + 항목별 기여율 보고 의무 |
| 비용 스케일 | per-term scale 은 train seed 기준정책으로 fit 후 **동결** (val/test 재계산 금지). 현행 fit 결과: 활성 8항 각 ~12.5% 균등 | placeholder(assumed) config 는 계측 전용 — reward 로 쓰기 전 지배도 점검 의무 |
| 본선 축 분리 | λ_vessel(본선 비용 배율)=**1.0 중립 고정**, 동적 위험 비활성, 본선 KPI 는 기록만 (`neutral_lambda_config()`). 밴드 설계 {1.0, 2.5, 6.0}×동적은 YR-041 별도 | 고위험 성능 주장 금지 |
| 시나리오 | 가우시안 평균조건 — 기본값을 평균(μ)으로 두고 ±2σ 에서 잘라낸 정규분포(TruncatedNormal, σ=12%)로 seed 별 변주: 트럭 물량·장치율·본선 물량·STS 간격·ETA 오차 | 스트레스 축은 별도 실험 |
| baseline (YR-044) | 1차 = `JointRolloutGreedy`: 두 YC 공동 feasible 조합 열거 × **고정 시간창(600s) 누적비용** argmin, 행동 후 base 정책으로 시간창 끝까지 진행(1-step 정책개선). ETA 전 역사값 93.4 < base 96.6, YR-048 후 같은 seed 62.927 < 79.643. 강 baseline = `BeamLookahead` | **행동분포 건전성 계약**: 실작업 가능 시 SERVE 선택률(`serve_when_available`) <0.25 → 실행 실패. 퇴화 0.03~0.08 vs ETA 전·후 건전 0.46~0.58. 후보 조합 축소 횟수와 PRE_REHANDLE 선택을 별도 보고 |
| 발견 (YR-044) | 무효판정 §6.2 가 지정한 "즉시비용(다음 결정까지) argmin"은 **문자 그대로 구현하면 그 자체가 퇴화** — 짧은 행동일수록 대기 rate 비용이 덜 쌓여 같은 함정에 비용으로 도달 (실측: 완료율 41%·대기 119.8분). 진단 전용 `JointImmediateCostGreedy` 로만 보존 | 고정 시간창으로 재정의해 길이 편향 소멸 |

한 줄 해석: "지표 하나가 전부를 지배하지 않는가"와 "상대가 제정신인가"를 이제 코드가 자동으로 묻는다.

## 5. 학습·평가 프로토콜 (YR-045 기준)

- **학습**: Candidate Double DQN 기본(ddqn), dqn·dueling 두 변형과 비교. 탐험률 ε=1/√ep 감쇠,
  경험 재사용 버퍼(replay) 50k·배치 64·target 망 동기화 500 step, cost_scale 은 train baseline fit.
  결정마다 resolver 가 mandatory·feasibility·dry_run 을 전담 — 학습이 안전을 건드릴 수 없다.
- **판정 게이트 (§18 다중, 동시 충족)**: 평균대기 감소 · P95(상위 5% 대기) 비악화 · 본선 완료 비악화 ·
  STS/이송 비악화 · 이동/재조작 중 1+ 개선 · 완료 100% · 위반 0. **총비용 CI(신뢰구간) 단독으로는
  불채택.** + 항목별 기여율 보고 의무(지배도 guard)·행동분포 건전성 통과.
- **seed 위생**: 기존 test 대역(320000~)은 YR-039 진단에 사용돼 **소각**. YR-045 는 신규 locked seed
  대역 필수. val 대역은 config 기준 **310000~310019 (20 seeds)** — 그중 310000~310004 는 무효판정
  진단에, seed 310000 은 YR-044 건전성 문턱 보정에 사용됐으므로 **val/test 양쪽의 취급을 YR-045
  사전등록에서 명시**해야 한다.
- **비교군**: JointRolloutGreedy(1차)·BeamLookahead(강)·ServiceFirstSPT/FIFO(보조 진단).
  `candidate_dqn_experiment.BASELINES` 의 퇴화 SPT 교체는 YR-045 작업 범위.
- **ETA 기여 분리**: `NO_ETA / ETA_NO_PRE / FULL` 3-arm으로 위치선점(REPOSITION)과
  선제 재조작(PRE_REHANDLE)의 순효과를 따로 계산. seed는 train 400000~, val 410000~,
  locked test 420000~로 교체. 상세 동결값은 [YR-045 사전등록](2026-07-16-YR-045-corrected-locked-rerun-prereg.md).
- **승격 경로**: YR-050(ETA 결정 시점) → YR-045 통과 → YR-042(DGT·HJNC 프로파일 일반화
  재실행)·YR-013(QMIX — 착수 조건 재판정) → YR-014(locked 평가·ablation·운영 적용판정, YR-002/009 의존).

## 7. 되돌릴 조건·금지 사항

**되돌릴 조건 (YR-034 §6)**: Candidate DDQN 이 같은 정보·행동의 강한 휴리스틱(JointRolloutGreedy)을
