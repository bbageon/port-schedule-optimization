# 정정 트랙 열린 문제·대응 전략 (2026-07-16)

> 지위: [실행 전략 정정판](2026-07-16-구현-RL전략-정정판.md)이 "무엇을 왜"라면, 본 문서는 **"지금
> 무엇이 막혀 있고 어떻게 풀 것인가"**의 스냅샷이다. 원본 최종전략·정정판은 수정하지 않는다.
> 작업 순서 원본은 [Dashboard](../../Dashboard/README.md).

## 1. 한눈에 (쉬운 말)

토대(계약→환경→후보/공동배정→비용→baseline)는 다 섰고, YR-039 무효의 두 원인(목적함수·baseline)을
코드로 제거했다. 그런데 **다음 큰 실험인 YR-045(정정판 locked 재실험)는 아직 돌릴 수 없다** —
선결 문제 2건과 측정 설계 보강이 남았다. 이번 세션에서 사용자 질문 두 개("미래정보가 성능을
나쁘게 하나", "WAIT이 필요한가")가 각각 실제 baseline 버그와 미결 설계 결정을 끌어냈고, 그 과정에서
지금 무엇이 열려 있는지가 또렷해졌다.

**정직한 한 줄**: 지금 결과는 전부 합성·가정 조건의 구현 증거다. YR-045는 "이겼다"를 판정하는
자리가 아니라 "정정된 파이프라인이 여러 지표를 동시에 통과하는가"를 보는 자리이며, 그 전에 아래
문제를 닫아야 판정이 오염되지 않는다.

## 2. 파악된 문제 (뿌리로 묶음)

### A. ETA(선제정리) 축이 절반만 켜져 있다 — YR-050
ETA를 넣어 선제 재조작 후보가 0%→14~15%로 열렸지만(YR-048), 엔진은 **처리할 실작업(SERVE)이
있어야만** 결정 시점을 연다. 그래서 첫 트럭 도착 전·한산기처럼 "미리 치워두면 가장 이득인" 구간에
크레인이 놀아도 선제작업을 시작하지 못한다. 또 ETA가 이미 지난 트럭의 "늦음" 신호(음수)가 feature
스키마에서 0으로 잘려 "지금 도착"과 구분되지 않는다.

### B. rollout baseline이 '대기(WAIT)'를 과대평가한다 — YR-051(수정)·YR-052(미결)
같은 뿌리에서 두 증상이 나왔다. rollout은 고정 시간창 안에서 비용을 재는데, WAIT은 즉시 드는
이동비용을 뒤로 미뤄 창 안에서 싸 보인다 → 대기를 과하게 고른다.
- **파국(수정됨)**: 후보 조합 절단이 WAIT을 떨궈, 남은 "둘 다 실작업" 조합이 전부 충돌하면
  decide가 진행 가능한 조합을 못 찾고 무한 WAIT → 8 seed 중 4개 완료율 0%. YR-051이 절단에서
  WAIT 보존으로 수정(8 seed 100% 복구, seed 310000 총비용 62.927 불변).
- **잔여(미결)**: 파국을 고친 뒤에도 전략적 WAIT(할 일 있는데 대기)이 남아 baseline에 평균
  −4.24 손해(6 seed ablation, 5/6 개선). → YR-052 설계 결정 보류.

### C. YR-045 측정·귀속의 무결성 — 사전등록 보강
- **2채널 혼입**: ETA는 선제 재조작만이 아니라 위치이동(REPOSITION)도 3.3배 열어, "개선이 어느
  경로에서 왔나"가 섞인다 → `NO_ETA / ETA_NO_PRE / FULL` 3-arm 분리 필요.
- **seed 대표성**: YR-048 헤드라인(62.927<79.643)은 하필 정상 seed 하나였고 기본 설정에서 절반이
  파국이었다 → 신규 소각 안 된 seed 대역 필수, baseline 완주를 arm별로 재확인.
- **전략적 WAIT**: 위 각 arm에 `전략적 WAIT 유무`를 교차해 데이터로 판정(YR-052).

### D. 상위 게이트 — 실데이터 미충족 (상시)
YR-002(실측 프로파일·현행규칙)·YR-009(시뮬레이터 실측 검증) 전에는 모든 결과가 합성·가정
증거다. 부산항 실운영 개선·"실제 대비 개선율" 주장 금지.

## 3. 각 문제를 어떻게 닫나

| 문제 | 대응 | ID·상태 |
|---|---|---|
| A. 선제정리 축 절반 | 실작업 없어도 ETA+결정지평으로 idle 크레인 결정 시점 개방 + 연착 음수 신호 보존(호환 필드/명시 버전) | YR-050 🟠 (YR-045 선결) |
| B. 파국 WAIT | 절단이 WAIT 항상 보존 — **완료** | YR-051 ✅ |
| B. 잔여 전략적 WAIT | 코드 미변경, YR-045에서 `전략적 WAIT 유무` ablation으로 판정 후 결정 | YR-052 🟡 (보류) |
| C. 2채널·seed·baseline | 3-arm + 신규 seed(train 400k/val 410k/test 420k) + baseline 완주 선결조건 | YR-045 사전등록 반영 |
| D. 실데이터 | 실운영 주장 금지, locked 평가는 게이트 뒤 | YR-002·009·YR-014 |

## 4. 실행 순서 (의존관계 기준)

```
YR-050 (선제정리 결정 시점·연착 신호)         ← YR-045 선결
   ↓
YR-045 (정정판 locked 재실험)
   · 3-arm: NO_ETA / ETA_NO_PRE / FULL — H2 귀속 분리
   · 전략적 WAIT 유무 ablation — YR-052 판정
   · §18 다중 게이트(평균대기↓·P95·본선·STS·이송 비악화·이동/재조작 1+ 개선·완료 100%·위반 0)
   · 총비용 CI 단독 불채택 / 항목별 기여율·건전성 통과
   ↓
YR-042 (DGT·HJNC 프로파일 일반화)  ·  YR-013 (QMIX 협조 — 착수조건 재판정)
   ↓
YR-014 (locked 평가·ablation·운영 적용판정)   ← YR-002/009 게이트
```

**축을 켜는 별도 실험** (전략적 WAIT·본선 우선의 이득이 이론상 나타나는 곳):
- YR-041 — λ_vessel 밴드·본선 압박 시나리오 (지금은 λ=1.0 중립이라 본선지연 0).
- YR-021 — 고부하·고장치율·고재조작 강건성 (지금은 평균조건이라 혼잡 약함).
이 둘이 켜지기 전에는 전략적 WAIT·본선 우선을 "무용"으로 단정하지 않는다(현재 조건 한정 결론).

## 5. 판정 원칙 (반복된 실수의 예방)

이번 트랙에서 값비싸게 배운 것들 — YR-045는 이걸 계약으로 건다.
1. **seed 하나로 판정 금지**: seed 310000만 봤으면 baseline 절반 파국을 "건전"으로 오인했다(YR-051).
2. **단일지표 승리 금지**: 총비용 하나로 판정하면 YR-039 함정 재현. §18 다중 게이트 동시 충족.
3. **미실행 ≠ 통과**: 안 돌린 테스트는 미지다(YR-046). 환경 2원(Windows 288·WSL 329) 전수 실행.
4. **baseline 건전성**: 비교 상대가 퇴화·파국이면 승리는 무의미(YR-044·051). 행동분포 검출기 상시.
5. **정보 vs 행동 분리**: "미래정보가 나빠졌다"는 정보가 아니라 정보가 연 행동의 오용이었다 —
   기여를 섞지 말고 arm으로 가른다(YR-052 ablation).

## 6. 범위 밖·되돌릴 조건

- **범위 밖**: ETA 예측모델·탄소 직접최적화·물리 레일 재배치(설계 불변). 실데이터 수집·익명화는 YR-005.
- **되돌릴 조건**: (a) YR-041/021로 스트레스 축이 켜지면 전략적 WAIT·본선 우선을 재평가.
  (b) Candidate DDQN이 강한 휴리스틱(JointRollout)을 못 넘으면 QMIX 전에 feature·비용·후보 재감사.
  (c) 실서비스영역 자료가 독립 YC 구조를 보이면 협조 범위 축소.
