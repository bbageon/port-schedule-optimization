# 본선 위험도 λ_vessel — 잠정 정적 중간값 결정

> 기록일: 2026-07-15 · 상태: **사용자 결정 반영** · 원문 요지: "본선작업의 경우
> 위험도는 추가 별도 실험을 해야 할 것 같아서 일단 위험도 없이 중간 값으로 설정"

## 1. 결정

향후 통합 실험(YR-013 QMIX·YR-014 locked 평가)의 비용 config 는 본선 λ 를
**동적 위험도 밴드 → 정적 2.5 (밴드 1.0~6.0 의 중간값)** 로 잠정 고정한다.

- config: [`configs/costs/terminal_cost_v1_static_mid.yaml`](../../configs/costs/terminal_cost_v1_static_mid.yaml)
  (`TERMINAL-COST-V1-STATIC-MID`, `lambda_vessel: static 2.5`)
- 동적 밴드(§10.6 초기후보)·기존 정적 λ=1.0 config 는 보존 — YR-041 비교 arm.

## 2. 근거

1. **동적 밴드는 아직 미검증 가정**: risk 임계(0.2/0.4/0.6/0.8)·계수(1.5~6.0)
   전부 assumed — 위험도→가중치 사상의 형태 자체가 별도 실험 대상.
2. **YR-039 결과와의 정합**: 해당 run 의 시나리오에서 본선 위험도가 실질 0
   (본선지연 0, risk_max≈0 → 동적 λ 도 사실상 1.0 하한에 상주) — 동적화가
   결과에 기여한 바 없음. 정적 전환이 기존 승리 결론을 흔들지 않는다.
3. λ=1.0(본선 무가중)은 본선 항을 트럭 항과 동급으로 두어 과소보호 위험 —
   중간값 2.5 는 "본선을 상시 중시하되 위험 동적화는 보류"의 보수적 절충.

## 3. 파생

- **YR-041 (backlog)**: 본선 위험도 λ 설계 실험 — 정적 {1.0, 2.5, 6.0} ×
  동적 밴드 paired 비교, 본선 지연이 실제 발생하는 고부하/타이트 deadline
  시나리오에서 (현 시나리오는 본선지연 0 이라 판별력 없음 — 시나리오 강화 포함).
- YR-002 실측 후 λ 는 실제 지연 비용 단가로 재캘리브레이션 (YR-038 §10 계약).
