# YR-039 형식 승리 무효 판정 — imbalance 항 지배 + baseline 결함

> 기록일: 2026-07-15 · 상태: **판정 확정 (외부 진단 + 본 세션 독립 재현 일치)**
> 파급: YR-039 "통합 RL 첫 승리 / 총비용 −84%" 판정 무효, YR-042 run 중단,
> YR-013 착수 조건 미충족 회귀. 재설계: YR-043(목적함수)·YR-044(baseline)·YR-045(재실험).

## 1. 무효 사유 (2결함 + 파생 1)

### 결함 1 — BASELINE_SPT 가 "최단 서비스"가 아니라 "최단 행동"을 선택

`SPTPreference.rank` 가 `plan.duration_s` 를 SERVE·PRE_REHANDLE·REPOSITION
전 종류에 일괄 적용 (`candidate_dqn_experiment.py`). resolver 는 WAIT 만
제외하고 전 후보를 정렬하므로, 소요시간이 짧은 REPOSITION 이 SERVE 를
체계적으로 이긴다 (±1 bay 재배치는 미생성이라 0-루프는 아님 — 2 bay 이상
재배치가 반복됨).

**실측 (본 세션 재현)**: SPT 의 선택 행동 중 REPOSITION 이 val 5-seed 54%
(1,161회 vs SERVE 250회), test seed 320000 에서 81%. 그 결과 mean_wait
3.87분 — 서비스우선 SPT(0.415분)의 9배. 의도했던 "강한 휴리스틱"이 아니라
퇴화 정책이었다.

### 결함 2 — 총비용의 97.6~100% 가 imbalance 단일 항 (scale placeholder)

- `cost.py` ASSUMED_SCALE: `imbalance=1.0` vs `truck_wait=600`,
  `crane_travel=1000` — imbalance 만 물리량 그대로 비용화.
- `engine.py _refresh_rates`: imbalance rate = 크레인별 **누적** served_count
  의 pstdev, 시간적분(rate×dt). 누적 카운트라 리셋이 없다.
- 산술: 2크레인이 1건 차이(pstdev=0.5)로 1시간 유지 → 0.5×3600/1 = **1,800**.
  트럭 1대가 같은 1시간 대기 → 3600/600 = **6**. 300배 격차.

### 파생 결함 3 — 사용자 의도 ≠ 실제 reward

사용자는 크레인 처리건수 균등화를 목적으로 지정한 적이 없다. imbalance 는
YR-034 전환 때 13항에 포함되고 YR-038 에서 전 항목 assumed placeholder
(weight=1, imbalance scale=1)로 들어간 뒤, **사용자 목적과 대조 없이**
YR-039 학습 reward 로 사용됐다. 처리건수 균등화는 안전 제약(중복·레인충돌 —
resolver 가 이미 보장)과 다른 선택적 운영 선호이며, 빠른/가까운 YC 가 더
많이 처리하는 것이 오히려 정상일 수 있다.

**결정 근거 2가지**:
1. **imbalance 제외 시 정책 간 사실상 무차별 (102.8~132.9)** — dueling 의

2. mean_wait 기준으로도 dueling(0.505분 val)은 자명한 서비스우선
   SPT(0.415분)에 진다.

실질적으로 검증된 것: "Dueling 은 주어진 reward(≒처리건수 균형)를 잘
최적화한다" — 학습 파이프라인의 작동 증거일 뿐 운영 개선 증거가 아니다.

## 5. 정정 경위 — queue-area 회귀안은 철회됨 (중요)

외부 진단의 1차 결론("원 목적=평균대기 최소화이므로 queue-area 단일비용으로
회귀")은 **이후 스스로 철회**됐고 본 세션 검증도 철회가 옳다고 판정:
최종전략 §0 이 Exp-1~4 구분 제거·터미널 전체 운영비용 최소화·QMIX 를
명시한다. imbalance 도 "무조건 제거" 가 아니다 — §10.2 에
`w_imbalance × 크레인별 작업부하 불균형` 이 실재하는 항목. 문제는 그것을
**누적 완료건수 균등화로 잘못 근사 + scale placeholder(1.0)** 로 구현해
§10.3(기준값 정규화 후 가중)을 위반한 것. YR-039 의 Dueling 이 QMIX
정책이 아니라 로컬 utility 중간 단계라는 지적도 타당 — "최종 통합정책
첫 승리" 는 이중으로 과대해석이었다.

## 6. 재설계 방향 (사용자 확정 2026-07-15, 등록: YR-043·044·045)

1. **YR-043 목적함수·행동공간 정정** (최종전략 유지, queue-area 회귀 금지):
   - imbalance 재정의 — 누적 완료건수 pstdev 폐기. `Load_i(t)` = 현재 작업
     잔여시간 + 할당·예약 작업 예상 서비스시간,
     `I(t) = (max−min)/Σ Load` (Σ=0 이면 0, 값域 [0,1]),
     `C_imb = w·∫I(t)dt/T_shift` → 에피소드당 O(1).
   - per-term scale: 실측 없으면 train seed 기준정책으로 fit 후 **동결**
     (val/test 재계산 금지). **항목별 기여율 보고 의무 + 단일 항 지배
     (기준: baseline fit 상 >70%) 시 run 실패 처리.**
   - WAIT 를 실제 학습 행동으로 복구 (최종전략 Hold/Yield, line 520):
     Q 채점·replay 포함, 선택 시 다음 외생 이벤트/타 크레인 완료까지 대기
     (0초 재결정 루프 방지).
   - mask 는 물리·정보 제약만 (관측불가·blocker 부재·슬롯 없음·접근 불가·
     레인/안전/비통과·중복배정·고장·미래정보). §8.4 류 운영 트레이드오프
     (장기대기·ETA 신뢰도·도착 전 완료·본선위험·절감효과)는 mask 가 아니라
     **State·Cost 로 제공해 RL 이 학습** — "트럭 있으면 무조건 SERVE" 금지.
   - 시나리오: 가우시안 평균조건 — 기본값을 μ 로 TruncatedNormal(μ,σ,±2σ),
     σ≈10~15% (트럭 물량·ETA 오차·장치율·본선 물량·STS 간격·이송시간·
     작업시간). **본선 악화 축은 별도 실험으로 분리(사용자 결정)**:
     λ_vessel=1.0 고정·동적 위험 비활성·스트레스 시나리오 제외·본선 KPI 는
     기록만 (고위험 성능 주장 금지). → λ=2.5 잠정 결정은 본 트랙에서 대체
     ([정정 amendment](2026-07-15-vessel-lambda-static-interim.md)).
2. **YR-044 baseline 재설계**: 1차 = **Joint Immediate-Cost Greedy**
   (2-YC 공동 feasible 행동 열거, 동일 13항 비용의 즉시비용 argmin) +
   강 baseline = 2~3단 beam/rolling-horizon. FIFO·ServiceFirstSPT 는 보조
   진단군. 전 정책 동일 정보·후보·제약·비용 config. **행동 분포 건전성
   계약 테스트** (REPOSITION 지배 같은 퇴화 사전 검출) 포함.
3. **YR-045 재실험**: 정정판으로 locked 재실행 — **기존 test 대역(320000~)
   은 진단에 사용돼 소각, 신규 locked seed 대역** + §18 다중 게이트 동시
   판정 (평균대기 감소·P95 비악화·본선 완료 비악화·STS/이송 비악화·
   이동/재조작 ≥1 개선·완료 100%·위반 0 — 총비용 CI 상한<0 단독으로는
   불채택). 그 후에야 YR-042(프로파일 일반화)·YR-013(QMIX) 재개.

## 8. 교훈 (프로세스)

- placeholder scale 로 **학습**까지 진행한 것이 원인 — assumed config 는
  평가 계측까지만 쓰고, reward 로 쓰기 전 항목별 지배도 점검(항목 기여
  분해 리포트)을 의무화한다 (YR-043 산출물에 포함).
- baseline 은 성능 수치만이 아니라 **행동 분포**로 건전성을 검증한다
  (REPOSITION 81% 같은 퇴화는 총비용 표에선 보이지 않았다).
- 외부 피드백 검증 절차가 유효했다: 주장 → 코드 근거 → 독립 재현(수치
  일치) → 판정. 이 순서를 표준으로 유지.
