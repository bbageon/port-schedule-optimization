# YR-031 — Oracle 상한 측정 (평균대기 게임의 총 상금) 사전등록

> 기록일: 2026-07-14 · 상태: **사전등록 (실행 전 동결)** · 사용자 승인: "A 해봐" (2026-07-14)
> 배경: 사용자 문제제기 — "RL 이 immediate greedy 보다 개선도 없는데 계속하는 게 맞나?"
> 선행: [YR-012 결과](2026-07-14-YR-012-residual-delta-net-prereg.md) (잔여 격차 +0.083분)

## 1. 질문과 판별 설계

**질문**: 현 조건(단일 YC·정보 동등·결정적·SLA_OFF·평균대기 지표)에서 greedy
(IMMEDIATE_COST_GREEDY = SPT)를 이길 **여지가 애초에 얼마인가?**

**근거 정황** (2026-07-14 측정): 순서 의존 성분(크레인 이동)이 서비스시간의 **7%**,
평균 동시 후보 **2.8개** — SPT 평균 최적성 정리가 거의 그대로 적용되는 지형.

**방법**: 잠긴 test 100일 각각에 대해 **전지적 beam search** 로 달성 가능한
최선 순서를 탐색한다. 찾은 개선 = 상금의 **하한** (찾은 만큼은 확실히 가능).
greedy 궤적을 beam 에 시드해 `best_found ≤ greedy` 를 구성적으로 보장.

- 탐색이 최적을 놓칠 수 있으므로 "상금 없음"의 완전 증명은 아니다 — 단
  greedy 대비 폭넓은 탐색(아래 §2)이 개선을 못 찾으면 관행적 종결 근거로 삼는다
  (사전 합의). 반대로 큰 개선을 찾으면 그 순서 패턴 자체가 RL 개선의 지도가 된다.

## 2. 탐색 설정 (동결)

| 항목 | 값 |
|---|---|
| 알고리즘 | 결정 lockstep beam search — 각 결정에서 모든 feasible 후보로 분기, 누적비용(= 최종 mean_wait 와 항등) 순 상위 W 유지 |
| beam width W | **12** + greedy 궤적 상시 시드 (탈락 금지) → best_found ≤ greedy 보장 |
| 전지성 | 시뮬레이터가 전 도착을 알고 분기 평가 — 단 행동은 "그 시점 feasible 후보" 로 제한 (도착 전 서비스 불가, 현 env 계약 동일) |
| 대상 | **YR-012 와 동일한 잠긴 test 100일** (seed 160000~) — 사후 분석이며 학습이 아니므로 재사용이 목적에 부합 (격차 직접 비교) |
| 환경 | HJNC-ARMG · SLA_OFF · n_external 100 · v1_final env (결정 계약 동일) |
| 산출 | 일별 (greedy_mean, best_found_mean, improvement) + 총상금 = mean(improvement) |

## 3. 판정 (사전 동결)

| 총상금 (greedy − best_found 평균) | 판정 |
|---|---|
| **< 0.15분** | **평균대기 게임 사실상 닫힘** — greedy 를 이 조건의 준최적 정책으로 공식 인정, 평균 축 실험 동결. RL 지속 시 승부처 이동 (P95/정보우위/복잡 지형) 재논의 |
| **≥ 0.30분** | 게임 열림 — beam 이 찾은 개선 순서의 패턴 분석을 차기 과제로 |
| 0.15~0.30분 | 중간 — RL 잔여 격차(+0.083)와 대비해 사용자 논의 |

참고 대비값: RL(Δ-net) 잔여 격차 +0.083분 — 상금이 이보다 작으면 RL 은 이미
사실상 도달 가능한 바닥에 있는 것.

## 4. 검토한 대안과 기각 사유

| 대안 | 기각 사유 |
|---|---|
| 정확 최적 (MILP/DP) | 순서의존 setup + release time 의 총 흐름시간 최소화 — NP-hard, 100 job 비현실 |
| 선점 SRPT 완화 하한 | 순서의존 이동시간을 완화에 반영할 깔끔한 방법 부재 — 느슨한 하한은 판정에 무익 |
| rolling lookahead-k | beam 이 상위호환 (같은 계열, 더 넓은 탐색) |

## 5. 한계 (사전 명시)

- best_found 는 달성가능값 — 진짜 최적과의 잔여 간극은 미지 (상금의 하한임을 판정문에 명시).
- beam 은 결정 lockstep — 모든 궤적이 "결정 = 작업 1개 실행" 이라 단계 수 동일 (N_config), 시간축 불일치는 누적비용 비교로 흡수.
- 합성 시나리오 + assumed 프로파일 — 이 판정은 이 조건에 한정.
