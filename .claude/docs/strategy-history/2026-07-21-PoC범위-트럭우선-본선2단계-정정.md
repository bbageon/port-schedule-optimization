# PoC 범위 정정 — 1단계=트럭(완료) / 본선=계획된 2단계 (2026-07-21)

> 계기: 본선 조사 중 저(에이전트)가 순서를 오해해 "총비용인데 본선 뺀 건 결함"으로
> 과표현. 사용자 정정: **블록 기준 트럭 대기 최소화가 원래 1단계 PoC 이고, 본선은
> PoC 이후 2단계 확장으로 의도된 것.** 즉 본선 부재는 결함이 아니라 단계 분리다.

## 설계 순서 (사용자 의도 — 확정)

```
[1단계 · PoC]  블록 기준, 트럭 대기 최소화 학습 정책   ← 원래 PoC 범위 (완료)
      ↓
[2단계 · 확장]  본선(선박 정시성) 추가                 ← PoC 이후 계획
```

## 1단계 = 완료 (블록 기준 트럭 대기 최소화 PoC)

문헌 보정 시뮬레이션 조건에서, 블록 1개·2크레인 트럭 대기 최소화 학습 정책이
규칙(SF-SPT)을 초과함을 사슬로 확립:

- **목적 재정렬**(트럭대기 1차 계층) → 계획기 상금 확인 (YR-071 G0 통과)
- **순위 증류 + DAgger** → 학습 정책 최초로 규칙 초과 (YR-073-b 확증)
- **반사실 RL 미세조정** → 간섭 −31~35%, 채택본 FT 확정 (YR-074)
- **강건성**(크레인 고장 주입) 유지 (YR-077) · **재조작 목적지** H1 규칙 채택 (YR-075-a)

→ **PoC 핵심 주장 성립**: "블록 기준 트럭 대기 최소화에서 학습 정책이 규칙을 이긴다."
주장 게이트: 문헌 보정 조건 한정, 실운영 주장 아님(D5).

## 본선 = 계획된 2단계 확장 (결함 아님)

- 1단계는 **의도적으로 트럭 대기에 집중**했다. 본선을 목적에서 낮게 둔 것은 오류가
  아니라 **PoC 범위를 트럭으로 한정한 단계 분리**다.
- 앞선 "총비용인데 본선 뺀 게 잘못" 표현은 **순서 오해에 따른 과표현으로 철회**한다.
  (YR-080B/080d/080e/041 문서의 "결함·잘못" 어투는 본 정정으로 맥락 보정 — 결과
  수치·진단은 유효하되 "PoC 결함"이 아니라 "2단계 사전조사"로 재분류.)

## 이번 세션 본선 작업의 재자리매김 = 2단계 사전조사

지금까지의 본선 진단은 버릴 게 아니라 **2단계 확장의 실측 발판**이다:

- **환경**: 본선/트럭 긴장 시나리오 축 (busan 근사 4셀·ETD 조임)
- **재판정**: 균형 목적서 현 FT 위치 (트럭 이기나 본선 방치)
- **통제가능성**: 본선지연의 통제 여지 vs 구조 하한 (실측, 추정 없음)
- **λ 진단**: 규칙 신호의 부분 성공·한계 (보편 신호 없음 = 학습 영역 시사)

## 2단계 착수 조건 (PoC 이후)

- **선결**: 목적함수를 총비용(트럭+본선 공동 1차)으로 정식 확정 (YR-080 — 병렬 진행).
- **그 후**: 균형 목적으로 교사→증류→FT 재학습 → 본선/트럭 긴장 셀에서 검증.
- **역할 분담**: 본선 긴급도는 규칙 신호로 부분만 잡히므로(041c 실측) 학습이 주도,
  λ(마감 임박)는 보조.

## 한 줄

확장이며, 이번 세션은 그 2단계의 환경·진단 발판을 실측으로 깔았다.**
