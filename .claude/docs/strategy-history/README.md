> 실험 전략의 의사결정 시점별 원문을 보존한다. 현재 구현 상태는 Dashboard가 원본이며,
> 이 폴더의 문서는 설계 스냅샷이므로 구현·검증 완료를 뜻하지 않는다.

## 작성 규약 — **요약 금지, 상세 보존** (2026-07-14 사용자 지시)

- 요약은 board row·본 README index 의 몫이다. **히스토리 문서 자체는 결정 시점의
  전체 맥락을 나중에 재구성 가능하게 상세히** 적는다. 필수 포함:
  1. **결정 경위** — 무엇(실험 결과·사용자 지시·관찰)이 이 결정으로 이어졌는지, 수치 포함.
  2. **검토한 대안과 기각 사유** — 채택안만 적으면 안 된다.
  3. **동결 설정 전수** — seed·규칙·판정기준·비목표를 재실행 가능한 수준으로.
  4. **근거 데이터 스냅샷** — 참조가 아니라 핵심 수치를 본문에 박제 (원본 유실 대비).
  5. **적용 범위와 되돌릴 조건** — 결정이 어디까지 미치고, 무엇이 관찰되면 재검토하는지.
- 사전등록 문서는 실행 후 **원문 수정 금지** — 결과·해석은 별도 절(`## 실행 결과`)로
  append 하고 기록일을 명시한다 (YR-027 v2 문서의 §5 패턴).
- 200줄 초과가 예상되면 분할하되, 분할이 상세성 축소의 핑계가 되면 안 된다.
