# 전략 결정 — 일반화-우선 강화학습 아키텍처(B) 채택 (2026-07-23)

> **사용자 결정 (2026-07-23)**: A(계획법 직접 배포)가 아니라 **B(일반화하는 학습 정책)** 로 간다.
> 목표 = **"계획법의 결정 품질을, 학습 정책의 속도로, 어떤 규모·설정에서든."**
> 배경: [본선정책 설계 v2](2026-07-23-본선정책-설계-학습가속-중앙rollout.md) + "FT=계획법 모방" 판정.

## 1. 왜 B (근거)

- **A = 계획법(rollout/MPC)**: 매 결정 미래를 시뮬해 고름 → 결정 **품질은 어떤 설정에도 일반화**
  (매번 새로 계산). 그러나 **규모에서 계산 폭발**(42기 조합 지수·조합마다 상태 복제) → 실터미널
  실시간 불가. 그리고 학습이 아님.
- **B = 학습 정책**: 한 번 배우면 **어떤 규모든 즉답**(빠름). 크레인 수 무관 구조면 1·2·42대 공통.
- ∴ 사용자 판단: 연구 목표가 **일반화 성능**이라면 A(계획법)만으론 부족(느림·규모) → **B가 주축.**
  성배 = 계획법 품질 × 학습 속도 × 규모/설정 일반화.

## 2. 정직한 출발점 (지금 어디)

- **FT 는 순수 강화학습이 아니었다** — 계획법 교사 **모방학습(imitation)** 이 본체 + 반사실
  미세조정 한 조각(그 신호도 rollout 이 계산). "계획법 흉내내는 얇은 학습"이었고 **실패**(별칭).
- **B = 이 프로젝트 첫 '진짜' 일반화 강화학습** 시도 = 크고 불확실한 도박(단계 게이트로 손절).

## 3. 첫 관문 = 관측별칭 해소 (선결)

실패 진단 = **서로 다른 상황이 214/246 요약벡터에 같아 보임**. B 는 여기서 출발:
1. **별칭이 '표현 탓'임을 먼저 측정** — 같은 관측에 교사 선택이 갈리는 **라벨 충돌률**을 값/랭킹
   타깃에서 직접 재고, 리치 표현이 그걸 줄이는지 확인. (현 별칭 진단은 분류에만·미측정 — 적대검토
   지적. 안 줄면 원인이 용량·최적화일 수 있어 B 전체 재검토 게이트.)
2. **입력 표현 교체** — 요약벡터 → 전체 스택·작업 선후·이벤트 일정·선박별 잔여작업·버퍼·크레인
   활성계획을 보존하는 **집합/그래프 입력**.
3. **크레인 수 무관 구조** — 크레인별 공유 인코더 + **순열불변 집계**(순서 무관, 1·2·N대 공통).

## 4. 학습 방식 — 계획법을 스승으로, 여러 설정 섞어

- 교사 = 계획법(rollout, 품질 실증·본선 보호됨). B = 이 교사를 **새 표현으로 재증류** + 반사실
  미세조정. 가설: 표현이 충분하면 별칭이 풀려 이번엔 증류가 성공한다.
- 학습 분포 = **도메인 랜덤화**(터미널·혼잡도·크레인 수 랜덤) → 그 범위에 일반화.

## 5. A 는 버리지 않는다 (역할 재배치)

- 계획법(A) = **품질 교사** + **소규모 배포 안전망**(2크레인 기준점). B 가 규모에서 A 품질을
  따라잡음을 실증하기 전까지 A 가 fallback. 즉 A→B 는 대체가 아니라 **가속·확장**.

## 6. 순서·검증 게이트 (조기 손절점)

1. **라벨 충돌률 측정**(별칭=표현 문제인지 확정) — 값/랭킹 타깃. → 아니면 B 재검토.
2. **최소 집합/그래프 학생 1건** → 별칭 감소·교사 재현 개선 확인. → 안 되면 표현 가설 기각.
3. **크레인 수 무관 구조 + 도메인 랜덤화 학습** → 규모 스윕서 계획법 품질 추종·속도 우위 실증.
- 의존: YR-081(구조확장)·본선 설계 v2 §7(집합/그래프)·YR-082/085/086(설정 컴포넌트).
- 범위: 전 결과 **문헌 보정 시뮬 조건·실운영 주장 금지**.

## 7. 리스크 (박제)

- 별칭이 표현 탓이 아니면(용량·최적화 등) B 근거가 흔들림 → §6-1·2 가 조기 판정.
- 큰 재작성(새 인코더 + 새 학습 파이프라인 + 구조확장) — 계산·시간 큼. **단계 게이트 통과 시에만
  다음 단계** 착수(무한 투입 방지).
