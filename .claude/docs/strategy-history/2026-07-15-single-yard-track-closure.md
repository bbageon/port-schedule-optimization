> 범위: 단일 블록·단일 YC·외부트럭·평균 블록대기 최소화·`BLOCK_ENTRY` 정보경계

## 1. 트랙의 질문과 최종 답

- **질문**: 위 단순화 환경에서 학습 정책이 강 휴리스틱 greedy(shortest estimated
  service, = `IMMEDIATE_COST_GREEDY`)의 평균 블록대기를 유의하게 이길 수 있는가?
- **최종 답**: **아니오 — 이 환경에서 greedy 는 near-optimal.** robust 격차 ~+0.1분
  (fresh band), 어떤 선택으로도 좁혀지지 않으며 도달 불가 상한(최적 checkpoint)도
  greedy 미달. RL 의 순가치는 "평균 승리"가 아니라 tail(p95) 개선 방향과 oracle
  미회수 상금에 있다.

아니라 문제 자체의 성질**(동급 작업 미세 스왑 여지가 작음).

## 3. 무엇이 확정됐나 (재사용 가능한 지식)

1. **greedy(SPT)의 준최적성**: 현 정보·목적에서 anti-SPT 이탈은 결정의 4.2%,
   즉시 희생 중앙값 0.004분, service 차이 중앙값 +9.5초 — 회수 여지가 구조적으로
   작다 (YR-031-b).
2. **함수근사의 실효**: tabular→MLP 로 해상도세 2/3 제거 (YR-012), 집합 feature 로
   추가 절반 (YR-012-c) — 방향은 전부 옳았고 격차를 1/12 로 줄였으나 문턱은 못 넘음.
3. **방법론 교정**: 단일 test band 는 정책을 과대평가할 수 있다 (동일 정책 +0.035
   vs +0.111). 향후 판정 전 다중 band/대표본 필수 (YR-040 취소 — 결론이 이미
   2-band 로 확정).
4. **RL 잔존 가치**: p95(tail) 개선은 220k 에서 관측 (band 종속이나 방향 유효),
   oracle 상금 +0.182 중 미회수분 — 둘 다 **다요소 비용·다중 YC 환경에서 headroom
   재탐색**이 합리적 (통합전략 YR-034 본류로 이관).

## 5. 적용 범위와 되돌릴 조건

- **적용**: 단일 야드·평균대기·현 정보 경계에 한정. "greedy near-optimal" 은 이
  좁은 설정의 결론이지 야드 운영 일반이 아니다.
- **되돌릴 조건**: (a) 정보 경계 확장(ETA·사전 반출입·본선 연계) 시 anti-SPT 여지가

  둘 다 통합전략(YR-034)에서 자연히 다뤄짐.
