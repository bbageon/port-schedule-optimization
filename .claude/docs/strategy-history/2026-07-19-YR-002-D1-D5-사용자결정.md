## 주장 게이트 재정의 (연구 정체성)

- **가능**: H1~H5 알고리즘 가설 검증, "문헌 보정 시뮬레이션 상" 방법론 기여 주장 —
  이 분야 학술 논문의 표준 위치 (선행연구 대부분 시뮬레이션 기반).
- **불가 (영구)**: "실운영 대비 개선율" 주장·CURRENT_RULE 확보·운영사 실측 validation.
  report 경고문·표현 금지 규칙은 유지 (문구를 "문헌 보정 조건" 명시로 갱신).
- **YR-009 재정의**: "운영사 실측 validation" → **"공개 실측 대조"** — PNIT·HPNT
  공개 turn-time (반입 9~22분·반출 19~33분) 분포와 시뮬 KPI 대조 (결정자료 §5-5).

## 실행 계획 (재기준화된 YR-002)

1. **프로파일 v2** (`build_calibrated_profile`): ARMG 제원 (Kalmar gantry 4~5 m/s·
   PEMA 2~3분/작업 앵커)·10열 6단·부하별 속도 분리(Liebherr 근거 비율)·SLA 2단계
   (안전운임제 30/60분)·gate travel 하향(600→300s 대). **기존 진단 프로파일 동결
   유지** (골든·선행 실험 재현성) — v2 는 신설 opt-in.
2. **부하 현실화**: 문헌 4~10대/h/크레인 → 2기 블록 8h ≈ 64~160대 (현 40대는
   하한의 절반). 피크 패턴 8h 주기 캘리브레이션.
3. **turn-time 대조 리포트** (YR-009 1차): v2 baseline 실행 분포 vs 공개값.
4. **천장 재진단** (신규 **YR-072** — YR-071 은 목적함수 재정렬로 선점, 상보 축):
   YR-070 진단을 v2·현실 부하에서 재실행 — "천장이 낮았던 것은 한산한 시나리오 탓"
   가설 검정. 예측·조율 상금이 커지면 학습 트랙(YR-066 등) 재개 근거, 그대로
   작으면 문제 성질로 확정. (YR-071 = 목적 축, YR-072 = 프로파일·부하 축.)
