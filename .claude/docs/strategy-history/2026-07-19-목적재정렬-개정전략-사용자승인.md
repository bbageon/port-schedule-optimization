## 3. 조건 7건 (동결)

1. **G0 확정 게이트**: v2 프로파일·중간+고부하+피크·신규 seed·paired. 평균대기,
   긴 대기, 완료율·backlog, SERVE/REPO 행동분포, 결정론 재실행 전부 보고.
2. **NEW 목적 형식 동결**: 임시 가중합 금지 — 계층(사전식): ①안전·미완료·backlog
   ②트럭 누적대기 ③긴 대기 ④본선 마감·지연 ⑤재조작·이동·간섭. 원칙: **하위
   비용이 아무리 좋아도 트럭 대기가 나빠지는 계획은 선택 불가**.
3. **교사 천장 명시**: 증류는 JR_NEW 초과 불가 — 성공 문구 = "SF-SPT 초과·JR_NEW
   근접·JR_NEW 보다 빠른 결정". JR_NEW 자체가 충분히 빠르면 JR_NEW 가 채택 후보
   (계산시간을 YR-071 부터 측정).
4. **본선 항 미검증 명시**: 현 시나리오 본선지연 0 — YR-071 통과의 의미는 "트럭
   대기 목적 정렬 + 블록 정책 헤드룸 확인"으로 **제한**. 본선 보호는 YR-041 별도.
5. **가치망은 scalar 하나 금지**: 교사 **순위 보존**이 주목표 (선택 분류 / 후보쌍
   순위학습 / 목적별 다중 출력 중 택 — 권장: 순위학습 주 + advantage 보조).
   숫자 하나 회귀가 계층 순서를 깨면 "학습 목적≠실행 목적" 재발.
6. **분기상태 확보**: JR_NEW≠SF-SPT 인 결정의 비율·대기감소 기여·SERVE/REPO 갈림
   수·경합 상태 비율·학습 데이터 포함 비율 **보고 의무** + 오버샘플링. 학생 실행
   상태를 JR_NEW 로 재라벨하는 반복 증류(DAgger) 포함.
7. **YR-075 분리**: (a) 재조작 목적지 — 에피소드 내 효과 측정 가능 / (b) 반입
   장치위치 — **생애주기(반입→보관→미래 반출) 연결 후에만** (현 시뮬은 반입
   컨테이너가 같은 에피소드에서 재반출되지 않아 보관품질 검증 불가).
