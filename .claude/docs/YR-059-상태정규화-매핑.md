# YR-059 — 상태 scale-only 정규화 (itc-v4) + QMIX 사다리 재실행

> [적용전략 §4·§6-1](상태정규화-보상가중치-적용전략.md) 집행. 코드 원본:
> `contract/schema.py`(norm_ref)·`integrated/encoding.py`(StateNorm)·
> `experiments/yr059_state_norm.py`(fit·러너). 검증: `tests/integrated/test_yr059_state_norm.py`.

## 1. 무엇을 (쉬운 말)

신경망에 들어가는 상태 숫자들은 단위가 뒤섞여 있다(초 단위 수천, bay 좌표 수십, 비율 0~1).
QMIX 의 혼합기는 이 숫자로 다른 망의 가중치를 생성하므로 큰 값에 특히 취약하다 — 사다리
실험에서 "학습 50회째가 최고점, 이후 950회는 악화"라는 발산 서명이 나왔다. 각 숫자를 필드별
**동결 기준값으로 나눠(scale-only)** O(1) 범위로 맞추고 ±5 에서 잘라 넣는다.

## 2. 구현 계약

| 항목 | 동결 내용 |
|---|---|
| 방식 | `value / norm_ref` 후 `[-clip, +clip]` (clip=5) — **중심이동 없음** (결측=0 규약 보존, 0→0) |
| 기준값 1차 | 스키마 `FieldSpec.norm_ref` assumed (unit 기본: 초=3600·bay=40·count=10·비율=1, trolley_row=4) |
| 기준값 2차 | baseline(BaselinePreference) train 대역 선두 5 seed 에서 필드별 **\|값\| P90** fit → JSON 동결 (`state_norm.json`, basis=fitted_baseline_p90). P90=0(상수) 필드는 assumed 유지 |
| 적용 지점 | **학습 인코딩(fv_to_vec) 전용** — 계약 record 값·validate·resolver 경로 불변. known 지시자 채널 무변경 |
| 금지 | running 통계(순서 의존·골든 파괴)·val/test 재적합·centering |
| 계약 버전 | itc-v3→**itc-v4** (FieldSpec.norm_ref 추가 — 스키마 구조 변경 규약 준수, golden 3종 재생성. record 값 불변 → terminal record golden 은 버전 문자열만 변경) |
| 기본값 | `state_norm=None` (미적용) — **기본 on 전환(적용전략 P1)은 본 판정 통과 후 별도 결정** (병행 실험 YR-061 과의 간섭 방지 포함) |

## 3. 실험 프로토콜 (ON vs OFF paired)

- ON: 사다리 재실행 — YR-013 과 동일 프로토콜(2000ep·tier 500/1000/2000·lr 3e-4·dueling·
  seed 530k/540k/550k·val 20) + `state_norm` 만 추가.
- OFF 대조: **기존 yr013_qmix_ladder 결과를 그대로 사용** (같은 seed·프로토콜·커밋 계보 —
  차이는 정규화뿐). JR 행도 선행 run 재사용(JR 은 인코딩 미사용).
- 판정: 정책별(QMIX@tier·INDEP@tier) ON−OFF paired 총비용·interference·대기 + QMIX 발산
  서명(최적 checkpoint 위치·곡선 후반 악화) 소멸 여부.
- **변인 통제 주**: val 확대(20→40, YR-057 권고)는 paired 비교를 깨므로 이번 판정에서 제외 —
  프로토콜 동일성이 우선이다. val 확대는 YR-057 로 남긴다.

## 4. 결과 (2026-07-19 — ON vs OFF paired, 같은 seed·프로토콜)

**갈린 판정**: 정규화는 **독립 학습자를 유의하게 개선**했고, **QMIX 는 유의하게 악화**시켰다.

| 정책 | OFF test | ON test | Δ (ON−OFF) [95% CI] |
|---|---:|---:|---|
| INDEP@500 | 86.12 | 81.74 | **−4.39 [−7.44, −1.60]** |
| INDEP@1000 | 86.12 | 80.51 | **−5.61 [−8.80, −2.74]** |
| INDEP@2000 | 85.57 | 80.51 | **−5.06 [−7.16, −2.84]** |
| QMIX@500 | 116.80 | 137.64 | +20.84 [+15.28, +26.40] |
| QMIX@1000 | 116.80 | 139.25 | +22.45 [+17.42, +27.41] |
| QMIX@2000 | 104.80 | 133.30 | +28.51 [+24.33, +32.81] |
| (JointRollout) | 68.26 | 68.26 | 재사용 (인코딩 미사용) |

- **INDEP**: 전 tier 유의 개선 — 관측 정규화 "항상 켜라"(P1, Andrychowicz 2020)가 이 환경에서
  확증. **track TD-RL 최고 80.51** (YR-045 dueling 86~91 대비, JR 격차 +17.3→+12.2).
  선택 지점도 건강(ep300→975, 예산 따라 개선).
- **QMIX**: **발산 서명은 사라졌다** — 최적 checkpoint 가 ep50 병리에서 ep350→775→1675 로
  예산 따라 이동, 곡선도 안정(128~148 요동, 후반 폭주 없음). **그러나 안정화된 지점의 성능
  자체가 크게 나쁘다** (+21~+29). 즉 입력 스케일은 불안정의 한 원인이었지만, QMIX 열세의
  **주원인이 아니다** — 안정적으로 나쁜 정책을 배운다.
- 해석: YR-061 판정(병목 = **TD 신용 희석** — 팀 공유 return ~70 에서 행동 간 Q 차이가 표적
  분산에 묻힘)과 정합. mixer 는 신용 귀속을 해결하지 않으며 입력 정규화도 아니다.
  "QMIX 발산 1차 용의자 = 입력 표현" 가설은 **기각**.

**후속 순서 제안**: 형식상 PopArt(YR-060) 조건이 발동하나, 신용 귀속(YR-063)이 1차 용의자로
승격된 지금은 **YR-063 해결 후 QMIX 재도전**이 합리적 순서다. 이후 RL 실험에는 state_norm
적용을 권고(P1 확증 — YR-063 하네스에 반영 권고, 기본 on 전환은 별도 결정).

## 5. 검증

- norm 미지정 = 기존 인코딩과 완전 동일(회귀 0) · scale-only 계약(결측 0 보존·부호 보존·
  ±5 클리핑·known 채널 불변) · fit 결정론·양수 · zero-init 정책의 결정 불변(정규화는 망
  입력만 바꿈) — 전용 5 tests.
- golden itc-v4 재생성: 스키마 descriptor 에 norm_ref 포함, record 값 불변.
