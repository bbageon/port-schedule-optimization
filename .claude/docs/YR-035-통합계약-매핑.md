## 1. 범위 (계약은 무엇을 하는가)

  fixture, 직렬화 왕복, 정보누출·단위·결측·vessel·joint·cost 검증.
- **안 한다 (YR-036~039)**: FeatureVector 값 생성기·`realized_at` 공급·GROUND_TRUTH truth 구조

  (YR-038), tensor 화·공유 Q망·학습 (YR-039). 계약은 이들의 산출물을 **저장·검증만** 한다.

## 3. 데이터 source × time-of-knowledge (감사 축)

`FieldSpec` 이 필드마다 **source**(어디서) · **tok**(언제부터 알 수 있나) · **unit** · **ablation** ·
nullable · assumed_default 를 버전당 1회 고정한다. 인스턴스(`FeatureVector`)는 값과 무관하게
value/known/assumed 3채널만 운반한다.

| source | 의미 | 대표 필드 |
|---|---|---|
| TOS | 야드/컨테이너 상시 | backlog_external, blocker_count |
| VESSEL_PLAN | 본선 작업계획 | vessel_count, remaining_moves |
| ETA_PROVIDER | 외생 제공 ETA | predicted_arrival_gap_s, eta_confidence |
| EQUIPMENT | 크레인·이송장비 | crane_bay, sts_wait_accum_s |
| LANE | 레인 점유·혼잡 | lane_congestion_* |
| DERIVED | 파생 계산 | slack_s, expected_service_time_s |
| GROUND_TRUTH | 시뮬 진실 | **레지스트리 부재** — 정책 진입 금지, 비용정산 전용 |

| tok | 가시 시점 | 정책수준 상한 |
|---|---|---|
| ALWAYS / PLANNED | 상시 / 계획 존재 시 | 전 레벨 |
| BLOCK_ARRIVAL | 블록 도착 후 | Baseline·Exp-1 이상 |
| GATE_IN | 게이트 진입 후 | Exp-2 이상 |
| PRE_ADVICE | 사전 반출입정보 | Exp-3 만 |
| NEVER | 실현·oracle | **레지스트리 부재** |

`_LEVEL_CEILING`(leakage.py)이 실험 정보수준별 허용 tok 상한을 강제 — info_filter.py 의
`BLOCK_ARRIVAL ⊂ GATE_IN ⊂ PRE_ADVICE` 계승. `realized_at > now` 면 미실현으로 known=0.

## 5. Action·Cost 계약

- **CandidateKind** (§8.2): `SERVE`(외부트럭·본선 실작업) · `PRE_REHANDLE`(도착 전 재조작) ·
  `REPOSITION`(위치조정) · `WAIT`(양보/대기). 가변 K(Top-K 아님, mandatory 보존).
- **3-mask 분리**: `pad_mask`(구조 패딩) / `feasible_mask`(§8.5 실행가능) / `FeatureVector.known`(결측).
- **JointAction** (§8.6): `resolver_token` 동일성으로 중복배정, `lane_id` 로 비통과 충돌, `eligible_crane_ids`
  로 자격을 검사. Job ID 는 `ref_job_id` 감사필드로만 보존 — **net 입력 채널 아님**(신원 암기 금지).
- **Cost 13항** (§10.2): `COST_TERMS` 고정. `VESSEL_FAMILY`(sts/transfer/vessel_delay/depart_delay)에만
  λ_vessel 적용(§10.6). `contributions = λ·weight·raw/scale`, `total=Σcontrib`, `reward=-total`.
  실수치는 assumed config(configs/costs, YR-038) 위임 — 계약은 이름·순서·단위·항등식만 고정.

## 6. 직렬화·검증 계약

- `dumps = json.dumps(to_dict, sort_keys, ensure_ascii=False)`. 모든 float `_canon`(round 6·-0 정규화),
  **inf/nan 거부**(결측은 known=0·value=0 으로만 표현). 불변식: `loads(dumps(r))==r`,
  `dumps(loads(dumps(r)))==dumps(r)`. schema_version 불일치는 `SCHEMA_MISMATCH`.
- `validate_all` 순서: schema→forbidden→units→missing→assumed→leakage(FeatureVector)
  → vessel(§7.10 mode 불변식) → candidates(feasible⊆pad·패딩 zero·mask_reason) → joint(§8.6)
  → cost(항등식·scale>0). 오류는 `ValidationError`/`LeakageError` 로 던진다.

## 7. 최소 통합 fixture 커버리지 (`build_minimal_transition`)

YC 2기(YC-A/YC-B, 공유 본선 후보 1건) · 외부트럭 반출 2(도착완료·PRE_ADVICE 미도착[eta_confidence
결측]) + 반입 1 · 본선 양하=RISK(완료시각 확보)·선적=SYMPTOM(완료시각 결측) · STS·이송 누적대기 ·
레인 2+연결 · SERVE/PRE_REHANDLE/REPOSITION/WAIT + 불가후보 + 패딩 슬롯. **결측 표기**(eta_confidence·
완료시각) · **가정 표기**(remaining_service_time_s imputed·profile_assumed). 반환 전 `validate_all` 통과.
