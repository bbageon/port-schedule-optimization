## 핵심 구현 결정 (index §3)

- 연속시간 이산사건 시뮬레이션, non-preemptive, 의사결정 이벤트에서만 정책 호출. 문제형식은 가변 경과시간 **SMDP**.
- 행동 = 작업 자체가 아니라 **priority rule 선택** (9종: FIFO·LONGEST_WAIT·NEAREST_JOB·MIN_REHANDLE·VESSEL_PRIORITY·`EARLIEST_PROVIDED_ARRIVAL`·PRE_REHANDLE·SAME_BAY_BATCH·WAIT_YIELD) + Action Masking. 동점해소 결정론적.
- 정책 단계: Tabular Q-learning(집계 bucket) → 필요성 입증 시에만 Masked DQN/PPO → Exp-4 중앙집중형 → 필요 시에만 Multi-Agent.
- Reward = 정규화 Core Cost 의 음수 (Scale 은 train Baseline 로 고정). 안전·도달영역·슬롯적합·비통과는 Hard Constraint 로 2중 차단.
- ETA 는 **Exp-3 에서만 공개되는 제공 입력값** — 예측모델·정확도 평가 미구현 (`eta_prediction_model: null` 명시).
- 탄소는 정책 목적 아님 — 실측자료 있을 때만 사후 정량화. UI 는 시뮬레이터와 분리된 **읽기 전용 replay**.
- 비교: 동일 초기상태·실제 도착흐름·이벤트·난수표본 (paired).

## 구현 시 주의 원칙

- TerminalProfile 하드코딩 금지, 임시값 `assumed: true` + 근거 (01 §2).
- 정보 노출은 InformationFilter 로 제한 — 미공개 작업 유출 방지가 invariant 테스트 항목 (05 §1.2). recorder 도 필터 결과를 저장 (04 §3.2).
- CURRENT_RULE 미확보 시 "실제 운영 대비 개선율" 표현 금지 (03 §1.1).
- 날짜 단위 train/val/test 분리, test 재튜닝 금지. 시뮬레이터 validation 미충족 시 RL 평가 진행 금지 (05 §1.5 게이트).
- Baseline/RL 비교 run 은 manifest hash (scenario·seed·event·initial state) 일치 검증 (04 §3.1).
