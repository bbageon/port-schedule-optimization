## 2. imbalance 재정의 (핵심)

```
Load_i(t) = 현재 작업 잔여시간 (available_at − now; idle=0)
I(t)      = (max Load − min Load) / Σ Load      # Σ=0 → 0, 값역 [0,1]
rate      = I(t) / T_shift                       # ∫rate dt = ∫I dt/T_shift → 에피소드 O(1)
```
- **폐기된 것**: `pstdev(누적 served_count)` — (a) 처리건수 균등화는 사용자 목적이 아니었고(무효 §1 파생결함 3),
  (b) 누적이라 리셋 없이 단조 증가, (c) scale=1.0 placeholder 라 미정규화(§10.3 위반) → 300배 격차.
- **feature 정합**: global `load_imbalance` 도 동일 `I(t)` 사용 — cost 와 feature 가 같은 정의여야 RL 이
  올바른 신호를 본다 (이전엔 feature=pstdev/cost=pstdev 로 둘 다 오정의).
- **실측**: minimal fixture episode_raw `imbalance 14779.431 → 0.028186`. **event hash 불변**(`970b45a…`) —
  비용 정의 변경이지 이벤트 흐름 변경이 아님.

## 3. 지배도 guard (재발 방지 장치)

```python
DOMINANCE_THRESHOLD = 0.70
contribution_shares(costs) -> {term: share}      # 항목별 기여율 — 보고 의무
assert_no_dominance(shares)                      # >70% 단일 항 → CostDominanceError (run 실패)
```
`freeze_fitted_config` 이 **fit → 동일 baseline 재채점 → shares → guard** 를 강제하고, per-term rep 에
`share` 를 박제한다. per-term fit(§10.3, per-interval 평균) 후 활성 8항이 각 ~12.5% 로 균등 — 정규화가
의도대로 작동함을 확인. YR-039 패턴(imbalance 97.6%)은 즉시 실패.

## 4. WAIT 복구 (행동공간)

- **resolver**: `pairs` 에서 WAIT 제외를 삭제 → 정책이 선택 가능. `dry_run` 수용 판정은 **실행 배정만**
  비교(`set(proj.plans)==work`)해 WAIT 를 joint feasibility 에 중립 처리. 계약 `candidate_id=None ⟺ WAIT`
  유지(`chosen_candidate_id=None`).
- **baseline 보존**: `BaselinePreference.rank(WAIT) = (2,…)` 최하위 tier → 실행 가능 작업이 있으면 항상 수행.
  기존 거동·golden 불변.
- **학습**: `encoding.actionable = selectable` (WAIT 포함). 배제 근거였던 *"WAIT 는 resolver 가 pair 에서
  배제해 회귀 표적을 못 받으므로 backup argmin 오염"* 전제가 소멸. `wait_pos` 를 노출하고 학습기가
  WAIT 결정을 `pos=wait_pos` 로 replay 표본에 포함.
- **0초 루프 없음**: WAIT 선택 시 engine 이 `yielded=True` → 다음 외생 이벤트/타 크레인 완료(`_clear_yields`)
  까지 결정 제외 (YR-036 기존 계약).

## 5. mask 정정

- **유지(물리·정보)**: 관측불가(info_level·`_visible_eta`)·blocker 부재·슬롯 없음·service range·
  rehandle 용량·레인/corridor/중복배정(ReservationTable)·고장·미래정보.
- **제거(§8.4 운영 트레이드오프)**: PRE_REHANDLE 의 `pre_window`(≥600s "도착 전 완료 가능") 게이트.
  대신 `predicted_arrival_gap_s` 를 candidate feature 로 제공 → RL 이 선처리 가치를 학습.
  계약이 tok=PRE_ADVICE 로 게이팅하므로 누출 없음. 결정 지평(§8.3 탐색축소)은 후보공간 bound 로 유지.
- `mandatory`(SLA 임박)는 mask 아님 — **pruning 금지 보장**일 뿐 선택 강제가 아니다 (유지).

## 6. 시나리오 (평균조건)

`TerminalGenParams(gaussian=True, sigma_frac=0.12)` — 기본값을 μ 로 seed 별 TruncatedNormal(μ,σ,±2σ) 추출:
트럭 물량·장치율·본선 물량·STS 간격 + ETA 오차(게이트 소요 μ=600s). RNG 는 생성에만(엔진 결정론 소비).
`gaussian=False` 면 μ 고정. **본선 악화 축 분리**: 스트레스 시나리오 제외·λ=1.0·본선 KPI 기록만
(고위험 성능 주장 금지) — 고부하/타이트 deadline 은 YR-041.
