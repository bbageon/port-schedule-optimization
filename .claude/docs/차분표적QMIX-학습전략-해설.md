# 차분 표적 QMIX 의 학습전략 — 무엇이고, 모방학습과 어떻게 다른가

> YR-013c(65.97, G1 통과) 승자 구성의 방법론 규정. 사용자 질문
> "모방학습이 아닌 어떤 학습전략인가" (2026-07-19) 에 대한 참조 문서.
> 구현: `integrated/qmix.py` DiffQmixLearner · [사전등록](strategy-history/2026-07-19-YR-013c-차분표적QMIX-prereg.md).

## 1. 한 줄 규정

**시뮬레이터 기반 반사실 신용(counterfactual credit)을 표적으로 쓰는 가치기반
강화학습 + 단조 가치분해(QMIX) 보정 — CTDE 패러다임.**
학계 용어로 이름 붙이면 **Difference-anchored QMIX** (차분 앵커 QMIX).

## 2. 네 가지 구성 성분

| 성분 | 무엇 | 우리 구현 |
|---|---|---|
| **가치기반 RL** | 행동의 "가치(비용) 점수" Q 를 배우고, 실행은 argmin Q | CandidateQNet — 후보 작업별 비용 점수 |
| **반사실 신용** (difference rewards) | 보상을 팀 결과가 아니라 "내 행동이 만든 차이"로 | D_i = 실제 창비용 − 나만 WAIT 한 세계의 창비용 (시뮬레이터 rollout 으로 직접 측정) |
| **가치분해** (QMIX) | 개별 가치를 단조 혼합기로 팀 가치와 정합 | mixer 는 D 가 놓친 크레인 간 교차항의 **보정항** (λ=1) |
| **CTDE** | 학습은 중앙(특권정보), 실행은 분산 | 학습 시 rollout·전역상태 사용, 실행은 크레인별 Q + 중앙 안전 resolver |

손실: `L = Σᵢ huber(Qᵢ, Dᵢ) + λ·huber(Mixer(Q, g), 팀 창비용)` — 1-step 회귀,
TD 부트스트랩·target 망 없음.

## 3. 모방학습(BC)과의 차이 — 핵심 표

| | 모방학습 (BC) | 본 전략 (차분 표적 QMIX) |
|---|---|---|
| 정답(표적)의 출처 | **선생(규칙 SF-SPT)의 선택** | **시뮬레이터가 측정한 비용 차이** (선생 불필요) |
| 배우는 것 | "선생이라면 뭘 골랐을까" | "이 행동이 팀 비용을 얼마나 바꾸는가" |
| 성능 상한 | **선생을 구조적으로 못 넘음** (사용자 판정 2026-07-19 — 최적화 경로 제외 사유) | 상한 없음 — 원리상 규칙 초과 가능 |
| 학습 데이터 | 선생의 시연 | **자기 정책의 경험** (ε-탐험 포함) |
| 협조 개념 | 없음 (선생의 단일 결정 복제) | 명시 신용(D) + 혼합기 보정으로 팀 상호작용 반영 |
| 현재 성적 | 56.25 (선생 53.12 미만) | 65.97 (비-BC 학습 신기록, 개선 여지 = 확전·개량) |

한 줄 대비: **BC 는 "누구를 따라할까"를 배우고, 이 전략은 "무엇이 실제로 비용을
줄이는가"를 배운다.** 전자는 선생이 천장이고, 후자는 환경이 천장이다.

## 4. 학습 신호가 만들어지는 파이프라인

```
결정 시점 (크레인 1~2대)
  → 각 크레인 Q 점수 → 중앙 resolver 가 충돌 없는 배정 확정 (실행)
  → [학습 전용] 시간창(2400s) 시뮬레이션 2~3회:
       실제 세계 비용 C_actual
       크레인 i 만 WAIT 한 세계 비용 C_cf_i   (상대 행동 고정)
  → 표적 생성:  Dᵢ = C_actual − C_cf_i   (WAIT 선택 시 0 — 자기 앵커)
               팀 표적 = C_actual        (추가 계산 0회 — 부산물)
  → 손실 역전파: 개별 앵커(Σ huber(Qᵢ,Dᵢ)) + 혼합기 보정(huber(Q_tot, C_actual))
```

## 5. 표준 기법 대비 우리 변형의 독자성 3가지

1. **TD 부트스트랩 제거 (1-step 회귀)**: 고전 Q-learning 은 "다음 상태의 Q"로
   표적을 만드는데(부트스트랩), 이 프로젝트의 진단(YR-061~062)이 그 통로가 팀 비용
   균등귀속과 결합해 **신용 희석**을 일으킴을 확정 — 그래서 자기완결 표적(D·창비용)
   으로 교체. target 망도 불필요해져 구조가 단순해짐.
2. **반사실을 망이 아니라 시뮬레이터로 계산**: COMA 등은 반사실을 신경망 추정으로
   근사하지만, 우리는 이산사건 시뮬레이터가 있으므로 **직접 굴려 측정** (모델기반
   요소). 정확한 대신 rollout 비용 지불 — 학습 시에만.
3. **혼합기의 역할 강등**: 표준 QMIX 는 혼합기가 신용 배분을 통째로 배우는 주
   학습기 — 이 환경에선 3중 실패(+24.8/+21~29 악화). 우리는 명시 신용(D)을 앵커로
   두고 혼합기를 **교차항 보정 전용**으로 씀 — 이 역할 분담이 첫 유의 개선(−9.41)
   의 정체다.

## 6. 계보 (관련 문헌 좌표)

- 가치기반 RL / DQN 계열 — Q(s,a) 회귀·argmin 실행
- Difference rewards (Wolpert & Tumer) · COMA 의 counterfactual baseline —
  "내 기여만 남기는" 신용 배분의 원형
- QMIX (Rashid et al.) — 단조 가치분해·IGM
- CTDE — centralized training, decentralized execution 패러다임

부산항 실운영 개선 주장이 아니다 (YR-002/009 게이트). 규칙(SF-SPT 53.12) 초과도
아직 미달(+12.85) — 다음 관문은 YR-068(본 시나리오 확전).
