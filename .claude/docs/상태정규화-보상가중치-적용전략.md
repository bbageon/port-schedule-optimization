# 상태 정규화·보상 가중치 적용전략 (기존연구 근거)

> 작성 2026-07-18. QMIX(YR-013) 검증 발산 진단 중 확인된 **상태 입력 정규화 부재**와
> **보상 가중치(scale/weight) 미확정·지배 전례**를 기존 연구에 근거해 우리 계약(itc-v3)에
> 맞게 적용하는 전략. 판단 근거 문헌은 §7. 실측 확정(YR-002)과 무관한 **구조·기법** 결정만 다룬다.

## 1. 한 줄 결론

- **상태**: 값을 그대로(초·비율·개수 혼재) 망에 넣는 지금 구조를, **필드별 고정 정규화 상수(scale-only)로
  나눠 O(1) 범위로 맞추고 클리핑**한다. 온라인 running-mean/std(VecNormalize)는 **쓰지 않는다** — 우리
  골든 결정성·CTDE와 충돌한다(§4). 이미 있는 **비용 scale 기계(provenance·baseline-fit)를 상태에 재사용**한다.
- **보상**: `scale`(통계적 크기 정규화)과 `weight`(업무적 우선순위)를 **분리**한다. 지금은 둘 다 assumed 로
  뭉뚱그려져 과거 imbalance 항이 총비용의 97.6~99.9%를 지배했다. scale=baseline 크기, weight=명시적 정책선호.
- **QMIX**: MARL 성능차의 대부분은 정규화·구현트릭에서 온다(Hu 2021). **정규화를 넣기 전엔 QMIX 열세를
  구조적 결론으로 확정하지 말 것.**

## 2. 왜 지금 (진단 요약)

- 상태 인코딩 `fv_to_vec`는 clip 만 하고(스키마 clip_lo/hi) **정규화가 없다** — 초 단위 필드 대부분은
  `lo=0.0`만 있고 상한이 없어 수천~수만 값이 0~1 비율·한 자릿수 count 와 같은 벡터에 섞인다.
  코드 전체에 z-score·running-mean·obs 정규화 0건(입력 정규화 부재, grad_clip 만 존재).
- 스케일이 큰 소수 필드가 학습을 지배 → 특히 여러 크레인 Q를 섞는 QMIX는 입력 스케일에 민감.
  YR-013 검증 발산(ep150 후 val 115→155)의 유력한 1차 용의자.
- 보상은 `total = Σ λ·weight·raw/scale`인데 scale/weight 전부 assumed placeholder. 크기와 선호가
  분리 안 돼 한 항이 총합을 지배하는 사고가 실제로 발생(YR-039).

## 3. 기존 연구가 말하는 원칙 (근거 → 우리 적용)

| # | 연구가 밝힌 것 | 우리 적용 |
|---|---|---|
| P1 | 관측 정규화는 성능에 가장 큰 영향 중 하나 — "항상 켜라" 권고 (Andrychowicz 2020 [A1]) | 상태 정규화를 **기본 on** 으로 도입 |
| P2 | 값의 크기가 여러 자릿수로 바뀌면 학습이 스케일에 민감 → 타깃 적응정규화(PopArt) (van Hasselt 2016 [A2]) | 입력 정규화로 부족하면 **QMIX 타깃에 PopArt** 보완 |
| P3 | MARL 성능차의 상당 부분이 정규화·code-level 트릭에서 옴 — 정규화한 QMIX가 SOTA (Hu 2021 [A3]) | 정규화 전 QMIX 열세를 **구조 결론으로 확정 금지** |
| P4 | 보상 스케일·시드에 성능이 크게 흔들림 (Henderson 2018 [A4]) | scale 을 **재현가능·문서화**된 상수로 고정, 지배도 guard 유지 |
| P5 | 선형 스칼라화에서 weight 는 선호를 인코딩 — 크기 정규화와 개념 분리 (Roijers 2013 [A5]) | **scale=크기 정규화 / weight=업무 선호** 분리 |
| P6 | 단순 reward clipping 은 스케일 정보를 버려 목적을 왜곡 (Mnih 2015 [A6]) | 보상은 **clip 대신 scale 정규화** — 항 간 크기만 맞춤 |

## 4. 상태 정규화 적용전략 (우리 구조 반영)

**핵심 제약 2가지가 방법을 결정한다.**

1. **3채널 `[value⊙known ‖ known]` — 결측=0 규약을 깨면 안 된다.** 평균을 빼는 정규화(centering)는
   known=1 값을 0 근처로 옮겨 "결측(0)"과 수치가 겹친다. → **중심이동 없는 나눗셈(scale-only)** 만 쓴다.
   `value/scale` 은 0→0 을 보존하므로 결측 규약이 그대로 유지된다. 정규화 후 known 마스크 재적용.
2. **골든 결정성·CTDE.** 온라인 running-mean/std(VecNormalize)는 상태가 흘러가며 통계가 갱신돼
   **순서 의존·비결정**이라 우리 골든 fixture(PYTHONHASHSEED 격리·바이트 재현)를 깨고, CTDE 다중
   크레인 배치에서 통계 공유가 꼬인다. → **동결된 필드별 상수**를 쓴다.

**설계**: 스키마 `FieldSpec` 에 필드별 `norm_ref`(양수 정규화 기준값)를 추가하고, 이미 있는
`Provenance`(basis: assumed→fitted_baseline→measured)로 출처를 추적한다. **비용 `scale`을 다루는 기계와
동일 구조**를 상태에 재사용하는 것이다(cost_config.py 의 with_scale·Provenance 패턴).

- `norm_ref` 는 baseline(FIFO/SPT) 분포에서 **필드별 대표 크기**로 fit — robust 하게 상위 백분위(예: P90)
  또는 평균. 초기값은 도메인 상식(초→/3600, count→/typical_max)으로 assumed 시작.
- 정규화 후 **클리핑**으로 이상치 상한을 둔다(예: `value/norm_ref` 를 [0, 5]로). 상한 없는 초 필드 방어.
- 스키마 필드 추가이므로 **SCHEMA_VERSION bump + golden 재생성**(규약 준수). ablation 그룹 거동 불변.
- 망 보강(선택): context/candidate MLP에 **LayerNorm** — MARL 구현트릭의 표준(P3), 스케일 잔차에 강건.

## 5. 보상 가중치 적용전략 (scale ↔ weight 분리)

원칙(P5·P6): **scale 은 "얼마나 큰가"를 지우고, weight 는 "얼마나 중요한가"를 넣는다.** 지금은 둘이
섞여 크기 큰 항이 곧 중요한 항이 돼버린다.

- **scale_k = baseline 기준 크기** — 학습 시나리오에서 FIFO/SPT baseline 의 항별 raw 크기 대표값
  (에피소드당 평균 또는 P90). 그러면 각 항의 정규화 기여가 baseline 에서 O(1)로 **비교가능**해진다.
  이것이 이미 스코프된 **baseline-fit(YR-038 기계 done, 수치 fit 은 실측/시나리오 후)** 이다.
- **weight_k = 명시적 업무 선호** — 안전운임·본선지연 등 정책 우선순위를 사람이 정하고 provenance 문서화.
  기본 1.0 에서 근거 있을 때만 조정. λ_vessel 동적밴드는 별도 실험(YR-041)에서 확정.
- **지배도 guard 유지·강화**: 단일 항 > 70% → run 실패(YR-043). 항별 기여 로깅을 상시 켜 재발 감시.
- **주의**: 실측 확정 전 scale/weight 는 assumed. 이 단계 결과는 **"부산항 개선" 주장이 아니라 구조 검증**
  (주장 게이트 YR-002/009 준수).

## 6. 실행 순서 (싼 것·의심 큰 것부터)

1. **상태 scale-only 정규화 + 클리핑** (§4) → **QMIX 예산사다리(500/1000/2000) 재실행**. 정규화 on/off
   를 ablation 으로 대조해 "발산이 정규화 부재 탓인지" 실측. — 가장 싸고 의심 큼, **최우선**.
2. 발산이 남으면 **QMIX 타깃 PopArt**(P2) 도입 — 학습 중 return 크기 변화(비정상성) 대응.
3. **보상 scale baseline-fit + weight 분리**(§5) — 실측/시나리오(YR-002/YR-041) 의존, 병행 준비.
4. 각 단계 **paired·seed 다중·golden 불변** 확인 후 커밋(전역 규약).

## 7. 근거 문헌

- [A1] Andrychowicz et al. (2020) *What Matters In On-Policy Reinforcement Learning? A Large-Scale
  Empirical Study.* arXiv:2006.05990 — 관측 정규화는 대부분 환경에서 성능에 핵심, "항상 사용" 권고.
- [A2] van Hasselt et al. (2016) *Learning values across many orders of magnitude* (PopArt), NeurIPS —
  타깃 적응정규화로 크기 변화에 강건.
- [A3] Hu et al. (2021) *Rethinking the Implementation Tricks and Monotonicity Constraint in
  Cooperative MARL* (RIIT), arXiv:2102.03479 — 정규화·code-level 트릭이 MARL 성능차의 상당 부분.
- [A4] Henderson et al. (2018) *Deep Reinforcement Learning that Matters*, AAAI — 보상 스케일·시드
  민감성, 재현성 권고.
- [A5] Roijers et al. (2013) *A Survey of Multi-Objective Sequential Decision-Making*, JAIR 48:67–113 —
  선형 스칼라화 weight 는 선호 인코딩.
- [A6] Mnih et al. (2015) *Human-level control through deep reinforcement learning*, Nature — reward
  clipping 의 스케일 정보 손실 트레이드오프(왜 clip 대신 정규화인지의 반례).
