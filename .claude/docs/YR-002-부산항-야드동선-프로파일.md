# YR-002-a 부산항 실제 야드 동선·인계 레인 공개근거 프로파일

> 조사일 2026-07-15 · 대상 HJNC(신항 3부두), DGT(신항 7부두) · 상세 설정:
> [HJNC](../../configs/terminals/hjnc_public_topology.yaml) · [DGT](../../configs/terminals/dgt_public_topology.yaml)

## 1. 결론

`L1/L2`를 “실제 차로 두 개”로 해석하면 안 된다. 실제 모델에는 아래 두 자원이 따로 필요하다.

- `road_segment`: 차량이 이동하는 도로 조각. 방향·통행차종·용량을 가진다.
- `transfer_point`: 차량이 멈추고 YC와 컨테이너를 주고받는 배타적 작업 위치다.

DGT는 공개자료가 육측 `LSTP`와 해측 `WSTP`, 외부트럭 후진 정차 절차까지 보여주므로
통합 시뮬레이터의 실제 부산항 기준선으로 가장 구체적이다. HJNC는 수평형 비교 프로파일로 유지한다.

## 2. DGT — 수직형·육해측 완전 분리

```text
외부트럭: Gate ─ 육측도로 ─ LSTP ─ 육측도로 ─ Gate
                              ↕ 육측 ARMG
                         [컨테이너 블록]
                              ↕ 해측 ARMG
본선흐름:  STS  ─ AGV/FMS ─ WSTP ─ AGV/FMS ─ STS
```

| 항목 | 공개근거 판정 |
|---|---|
| 블록 방향 | 안벽에 수직, 육측·해측 작업구간 분리 — 공식 확정 |
| YC | 46기, 블록당 2기 → 23블록 유도 |
| 역할 | 육측 ARMG=외부트럭, 해측 ARMG=AGV — 공식 확정 |
| 단면 | 최대 10열·6단 — 공식 확정 |
| 외부트럭 인계 | `LSTP`; 진입로에서 후진 정차 — 2026-05 공식 가이드 |
| 내부 인계 | `WSTP`; 이후 안벽까지 무인 AGV 구역 — 공식 작업 Flow |
| AGV 동선 | FMS가 경로·작업·충전을 관리 — 공식 확정 |
| 일방통행 | 주도로의 정확한 방향 edge는 미공개 |
| 인계점 수·용량 | 블록별 정확한 개수와 동시작업 대수 미공개 |

LSTP에서는 차량번호 자동인식, 정지 신호, 운전자 하차·콘 해제, `차량 준비 완료` 버튼,
작업완료 녹색 신호 순으로 진행된다. 따라서 LSTP는 단순 통과차로가 아니라 **예약·점유되는
작업 berth**다. `REVERSE_IN`이므로 이를 “한 방향으로 통과하는 L1”로 모델링하면 틀린다.

### DGT hard constraint

- 외부트럭은 WSTP·해측 무인구역에 진입할 수 없다.
- AGV는 외부트럭용 LSTP 후보가 될 수 없다.
- 육측/해측 ARMG 역할은 공개 구조를 따르며 임의 공동풀로 시작하지 않는다.
- LSTP 점유 중 같은 인계점에 다음 차량을 넣지 않는다. 정확한 병렬 위치 수는 파라미터로 둔다.
- 주도로 일방통행은 도면·FMS 자료 확보 전 강제하지 않는다.

권장 ID는 `B01-LSTP`, `B01-WSTP`다. 숫자뿐인 `L1/L2`는 금지한다.

## 3. HJNC — 수평형·양측 캔틸레버

```text
외부차량 loop ─────────┐  블록 측면 인계
                       ↓
              [CANT-A | 10열×6단 | CANT-B]  (안벽과 평행)
                       ↑
내부 YT loop ──────────┘  블록 측면 인계
```

| 항목 | 공개근거 판정 |
|---|---|
| 블록 | 21개, 안벽과 평행 — HJNC 공식 확정 |
| YC | 자동 C-ASC 42기, twin cantilever → 블록당 2기 유도 |
| 크기 | 10열·6단, 길이 52 TEU — HJNC/PEMA |
| 차량 | 외부트럭과 내부 YT를 RFID로 블록 진입점에서 인식 |
| 인계 | C-ASC 양쪽 캔틸레버 아래에서 차량 처리 — PEMA |
| 방향성 | HJNC 공식 배치 그림에 방향 화살표가 있으므로 directed graph 필요 |
| 미확인 | 색상별 차량 매핑, 정확한 차로 수·용량·edge 방향, 두 YC 분담/통과 규칙 |

HJNC 게이트의 반입 6·반출 4레인은 **터미널 출입 게이트 수**다. 블록 하나의 작업 레인이
10개라는 뜻이 아니다. 블록 인계자원은 우선 `B01-CANT-A/B`로 쓰고, 외부트럭·YT의
사이드 고정은 운영자료 전까지 확정하지 않는다.

## 4. 계약 변경점

현재 `itc-v1`의 무방향 `LaneGraph` 하나로는 실제 구조를 충분히 표현하지 못한다.

```text
RoadGraph(edge: from, to, allowed_vehicle, capacity, travel_time)
TransferPoint(id, block, side, allowed_vehicle, capacity, control)
CraneRole(crane, service_side, allowed_job_type)
```

- `RoadGraph`는 이동·혼잡·단방향을 계산한다.
- `TransferPoint`는 같은 작업 위치의 동시점유 충돌을 막는다.
- `CraneRole`은 DGT의 육측/해측 작업 분리를 강제한다.
- 공개되지 않은 값은 `null + assumed=true`로 두며, 임의값은 별도 합성 시나리오에서만 넣는다.
