## 1. 총괄 판정 (3렌즈 설계 합의)

**단일 YC `sim/engine.py` 는 동결**하고 순수 프리미티브(`travel_time`·`stack`·`kpis`·`constraints`)만
재사용, 다중 YC 통합 엔진은 sibling 패키지 `src/yard_rl/integrated/` 에 신규 조립. 근거: 다중 YC 는
크레인 A 의 진행 중 이동을 B 가 관측하면 정보시점 위반 → **deferred commit**(dispatch=예약·완료=스택
커밋) 필수인데, 단일 YC 의 "dispatch 시점 일괄 커밋" 등가성은 크레인 1기에서만 성립. `sim/` 무수정으로
기존 golden(n_events=325) 회귀 원천 차단.

## 2. 범위

- **한다(YR-036)**: 이벤트 엔진(다중 YC·본선·이송·레인), 4종 자원 lock, deferred commit, 구간 raw
  비용 적분, clear-out, 정보시점 누출 0, 결정론, 계약 어댑터(sim→TransitionRecord), 참조 디스패처.
- **안 한다**: 후보 생성기 정교화·feasible mask·중앙 resolver·deadlock yield (YR-037); 비용 scale/weight/λ
  실수치·인과 ledger (YR-038 — sim 은 raw 물리 delta 만); Q망·학습 (YR-039); 실측 validation (YR-002/009).
- 후보 열거(`candidates_for`)는 결정적 SERVE 최소 열거 — YR-037 교체 seam. 비용은 assumed placeholder.

## 4. 메인 루프·의사결정

- `run_until_decision() → TerminalDecision(crane_ids) | None`: 동시각 이벤트 소진 후 idle·미yield 크레인
  집합을 반환. `assign(cid, a)` 로 크레인별 순차 배정(앞 예약 반영) → `close_decision()`.
- 동시각 처리순위(`_PRIORITY`): 완료(0)·장비(1)·이송/STS(2)·도착/release(3)·개시(4)·계획변경(5)·ETA(6)·marker(7).
- 결정론 tie-break 전 경로 고정: 크레인=`sorted`, 후보=`sorted(job_id)`, `find_slot` 동률=`(cost,bay,row)`,
  엔진 RNG 없음, `EventQueue._seq` plain int (itertools.count 금지 → deepcopy 안전, YR-031 beam).

## 5. deferred commit·자원예약

- `_plan` 은 스택 deepcopy 에 moves 를 미리 계산(스택 미변형)하고 예약만 잡는다. 물리 실현(remove/place)은
  `JOB_COMPLETED` 의 `_complete` 에서 moves 순차 커밋 → **진행 중 이동 비관측**(`observable_stacks` = 커밋 스택).
- 4종 lock(`ReservationTable`): job token(이중배정)·slot(`find_slot(exclude=reserved_slots)`)·corridor(bay
  구간+`safety_gap` 비통과·간섭)·lane. 두 겹 방어: `candidates_for` 가 `can_reserve` 로 1차 필터, `assign`
  의 `reserve()` 가 임의 joint(YR-037)를 2차 검증(DUP_JOB·LANE_CONFLICT·CRANE_INTERFERENCE·SLOT_CONFLICT).
- 해제는 완료 이벤트로만(부동시간 암묵해제 = 비결정 금지).

## 6. 구간 비용 적분 (Σ cut == episode)

- **truck_wait·long_wait**: 재사용 `KpiTracker` 정확 적분(queue_area·tail_area) 증분을 `_advance` 에서 accrue.
- **rate 항**(sts_wait·transfer_wait·lane_cong·interference·imbalance): `_refresh_rates` 가 이벤트마다 `set_rate`,
  `_advance` 가 `rate×dt` accrue.
- **delta 항**(crane_travel·empty_travel·rehandle·vessel_delay·depart_delay·resequence): 이벤트 임펄스 accrue.
- 각 accrual 이 정확히 한 구간에 귀속 → `Σ cut()[k] == episode_raw()[k]` (test 로 검증, 오차 <1e-6).
- scale/weight/λ 는 assumed placeholder(`cost.py`, YR-038 위임). `make_cost` 가 항등식(total=Σcontrib) 보장.

## 7. 계약 어댑터·정보시점·clear-out

- `record_episode` 가 결정마다 s_k 캡처 → 참조 디스패처 배정 → t_{k+1} 진행 → `cut()` raw → s_{k+1} 캡처 →
  `TransitionRecord` 조립 → **매 record `validate_all` 통과 + canonical 왕복**(전수 보장).
- 두 조회 표면 분리: `observable_stacks()`(feature 전용) vs `cum_wait`·`VesselTruth`(realized_at 게이팅·비용 전용).
  `own_oldest_wait_s`·candidate `cum_wait_s` 는 트럭 진실 도착시각을 `realized_at` 로 넘겨 미실현을 known=0 중화.
- 본선 어댑터: `resolve_mode` 로 RISK(완료시각 확보)↔SYMPTOM(결측) 배타, risk/징후 산식은 assumed.
- clear-out(`_finalize`): end 까지 적분(신규도착 없음), 미완료 본선 vessel_delay·검열 대기 계상, 종료.
- ablation: `ablation_off` 를 전 그룹 `build_feature_vector` 에 배선. §16.3 필수필드 포함 그룹(LANE·MULTI_YC)
  ablation 지원 위해 계약 `validate_missing_fv` 가 ablation 필드를 REQUIRED_MISSING 에서 면제(YR-036 반영).
