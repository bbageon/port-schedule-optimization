# YR-042 — 다른 2크레인 블록 profile-shift 일반화 게이트

- **Epic**: Exp / **Priority**: 🟠 / **재정의**: 2026-07-20
- **배경**: 현재 `FT+H1`은 24×10×6 단일 블록·동일 크레인 2대에서만 학습됐다. 다만 YR-080·075-c가 목적과 목적지 규칙을 바꿀 수 있으므로 이 작업은 그 뒤 동결된 정책 bundle을 평가한다. 기존 DGT·HJNC 근사 프로파일은 실제 수치가 거의 같고 구 정책용이라 일반화 증거가 아니다. 부산항 공개자료도 대체로 Level 0~1이며, 실제 블록 좌표·크레인 규칙·레인 구조는 빠져 있다.
- **목표**: 동결 정책이 실제 수치가 다른 단일 블록·2크레인 구조에 무재학습으로 이식되는지 판정하고, 실패할 때 제한된 재적응으로 전이 가능한지도 별도 판정한다.

## 범위

- 단일 블록, 크레인 정확히 2대만 다룬다.
- 기준환경과 실제로 다른 프로파일을 최소 2개 구성한다. bay·row·tier, 이동속도, 작업구역, 레인 또는 이송장비 중 둘 이상이 수치로 달라야 한다.
- 각 필드에 출처·연도·단위·불확실성·`confirmed/derived/assumed/unresolved`를 기록한다. 임의 수치 프로파일은 **구조 stress test**이며 터미널 일반화로 부르지 않는다.
- 터미널 이름이나 `fleet_kind`만 다른 프로파일은 인정하지 않는다.
- YR-082가 Level 2로 자격을 부여한 **같은 구조군**만 무재학습 성능 비교에 넣는다.
  잠재 후보군은 수평 RMG/ARMG이지만, 현재 공개조사만으로 자격을 충족한 실제
  2크레인 블록은 0개다.
- DGT의 육측·해측 역할분리와 AGV, BNCT·BCT의 S/C처럼 후보·이송·안전계약이 다른 구조는 이름만 바꿔 넣지 않는다. 현 계약으로 물리를 표현할 수 없으면 YR-081 구조 확장으로 보낸다.
- 크레인 1대·3대 이상과 다중 블록은 YR-081 범위다.

## 프로파일 자격 게이트

- 선택 블록의 ID·좌표·Bay/Row/Tier·진입/출구·작업면 확인
- 실제 2크레인 배치·담당 Bay·역할·통과 여부·최소 안전거리 확인
- 방향·길이·용량·허용차종이 있는 레인 또는 인계구조 확인
- 크레인 속도·작업시간과 큐 등록 시점 확인
- `unresolved` 핵심필드가 남으면 실제 터미널 arm을 열지 않고 stress arm으로 강등
- YR-083 compiler·mask·resolver가 위 구조를 실제 실행하고 dead field 변이검사를 통과

앞의 물리항은 Level 2다. 실제 속도·작업시간 분포와 큐 등록 시점은 Level 3이며,
없으면 문헌·가정 운영조건을 쓴 **구조 이식 시험**으로만 보고한다.

## A. 무재학습 시험

1. YR-009·080·075-c 뒤 채택된 체크포인트, 상태 정규화, 목적지 규칙, 목적함수와 resolver를 동결한다.
2. 목표 프로파일 자료는 평가 시나리오 생성에만 쓰고 정책 학습·선택에는 쓰지 않는다.
3. 각 프로파일에서 동결 정책과 동일 목적지 규칙을 쓴 SF-SPT를 신규 paired seed로 비교한다.
4. ETA·마감·known-mask 계약이 기준환경과 같은 의미·시점으로 제공되는지 검사한다.
5. A/B 슬롯은 크레인 ID 문자열이 아니라 물리 역할로 매핑한다. 동일 환경에서 이름만 바꾼 회귀검사 또는 역할 매핑 동결을 통과해야 한다.

## hard guard와 진단

- 모든 seed 완주율 1.0·backlog 0·제약위반 0
- 평균 대기 개선과 P95 대기 사전등록 비열등
- 판별력 있는 셀에서 본선·레인·이송 지표 악화 없음
- SERVE·REPOSITION·WAIT 행동비율과 이동·재조작·간섭 퇴화 없음
- episode 계산시간·메모리의 배포 한계 통과
- frozen normalization의 clipping/saturation 비율, 후보 pruning률, 공동조합 수 보고 및 사전등록 한계 통과

## B. 제한된 재적응

A가 실패할 때만 실행한다. 목표 터미널의 train/validation/test를 먼저 분리하고,
고정된 작은 예산으로 정규화 재적합·증류·미세조정을 허용한다. test 결과를 본
뒤 설정을 고치지 않는다. 성공해도 명칭은 **전이 가능**, 무재학습 일반화가 아니다.

## 최종 분류

- **ZS PASS**: 모든 근거 프로파일에서 무재학습 guard와 사전등록 성능기준 통과
- **ADAPT PASS**: 무재학습은 실패했지만 제한된 재적응 후 locked test 통과
- **PROFILE-SPECIFIC**: 처음부터 재학습해야 하거나 guard 실패

Level 2 실제 구조자료로 만든 프로파일에서만 `cross-terminal structure`를 붙인다.
assumed profile만 통과하면 `profile-shift stress PASS`로 제한한다. 실제 운영로그로
보정한 Level 3가 없으면 터미널 운영 개선율은 주장하지 않는다.

정확한 비열등 한계·seed 수·신뢰구간은 실행 전에 사전등록한다. 고적재율 0.65에서
FT+H1이 20개 중 2개 seed를 완주하지 못한 원자료 결과를 회귀 셀에 포함한다.

- **의존**: YR-009 환경 동결, YR-080 목적 동결, YR-075-c 뒤 최종 정책 bundle,
