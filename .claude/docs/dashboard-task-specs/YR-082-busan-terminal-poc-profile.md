# YR-082 — 부산항 터미널 프로파일 자격심사·PoC 전환 게이트

- **Epic**: Data / **Priority**: 🟠 / **등록일**: 2026-07-20
- **입력**: 사용자 제공 `부산항_터미널별_강화학습_환경정보_상세조사.md`
- **배경**: 조사에는 신항 7개·북항 3개 터미널의 시설·장비·일부 구조와 미확인값이
  구분돼 있다. 그러나 공개자료만으로는 대체로 규모 비교(Level 0)와 가정 격자
  시뮬레이션(Level 1)까지만 가능하다. 현재 `SNP-ARMG-STD`도 특정 터미널 재현이
  아니라 문헌 보정 단일 블록 환경이다.
- **목표**: 공개자료를 바로 실행값으로 오해하지 않도록 10개 터미널의 증거 레지스트리를
  만들고, 다른 터미널 시험·현장 PoC를 열 수 있는 자료 수준을 필드별로 판정한다.

## 구조군

| 구조군 | 터미널 | 현재 정책과의 관계 |
|---|---|---|
| `PROVISIONAL_RMG_ATC_YT` | PNIT·PNC·HJNC·HPNT | 장비 기준 잠정군. HJNC만 수평형 확인; 나머지는 방향부터 미확인 |
| `VERTICAL_ARMG_AGV` | DGT | 육측·해측 크레인 역할과 AGV 흐름을 표현해야 하므로 동일 후보풀 적용 금지 |
| `VERTICAL_TC_SC` | BNCT·BCT | S/C 이동·인계 규칙이 AGV·YT와 달라 별도 환경군 |
| `CONVENTIONAL_MIXED_PROVISIONAL` | BPT 신선대·감만, HKT | 잠정군. 장비 형식·블록 이동·최신 운영구조부터 현장 확인 필요 |

구조군이 다르면 같은 네트워크 **구조**를 후보로 검토할 수는 있지만 동결 가중치
재사용을 전제하지 않고, 성능을 한 평균으로 합치지 않는다. DGT·BNCT·BCT를 현
2크레인 공동경합 환경에 이름만 바꿔 넣는 시험은 금지한다.

## 프로파일 증거 계약

각 필드는 숫자만 저장하지 않고 다음 메타데이터를 가진다.

```text
value · unit · status(confirmed/derived/assumed/unresolved)
source_url · source_authority · retrieved_at · source_date · valid_from/to
derivation · conflict_group · verification_required
```

- 충돌한 자료는 최신 근거로 해소하기 전 `unresolved`로 둔다.
- 부두·안벽·철도·전체 야드 길이를 차량 레인 길이로 대체하지 않는다.
- CY 면적에서 블록 길이·Bay 수를 역산하지 않는다.
- 전체 크레인 수를 블록 수로 나눈 값을 블록당 대수로 확정하지 않는다.
- 연간 처리능력·전체 장비 수를 블록별 도착률·장치율로 환산하지 않는다.

## 단계

### A. 공개자료 레지스트리

1. 10개 터미널을 위 4개 구조군으로 분류한다.
2. 시설·블록·크레인·게이트·도로·이송장비 필드를 증거 계약으로 정규화한다.
3. 기존 HJNC·DGT YAML과 조사 원문을 대조해 아래 하한보다 높게 확정하지 않는다.
   - PNIT·PNC·HPNT 배열방향은 `unresolved`다. PNIT 장비 수 충돌도 한 값으로 합치지 않는다.
   - HJNC는 21블록·42기만 확인이다. 블록당 대수와 52 TEU는 `unresolved/source_conflict`,
     42÷21=2는 참고 유도값이다. YT·샤시의 BPA/운영사 값도 따로 보존한다.
   - DGT는 블록당 2기·육해측 역할만 확인이다. 23블록은 유도 후보이며 확정값이 아니다.
   - PNC의 T/C 69기 중 철송 RMGC 4기 포함·일반야드 배분은 `unresolved`다.
   - DGT `hard_vehicle_separation`, HJNC `separate_vehicle_paths_supported`도 물리 방식이
     확인되기 전에는 추론값 이상으로 올리지 않는다.
4. 터미널별 Level 0~4 자료충족도와 빠진 필드를 표로 남긴다.

### B. 실제 블록 자격심사

YR-042에 들어갈 블록은 최소한 다음 값이 확인돼야 한다.

- 블록 ID·좌표·Bay/Row/Tier·진입/출구·작업면
- 실제 크레인 배치·담당 Bay·역할·통과 여부·선후관계·최소 안전거리
- 방향·길이·용량·허용차종이 있는 레인과 인계점

이 수준은 구조 재현(Level 2)이다. 현재 공개조사만으로 이 자격을 모두 충족한
터미널은 0개다. 문헌 시간값은 `assumed`로 넣어 구조시험만 할 수 있다. 터미널별
성능을 주장하려면 TOS/VBS/RTLS의 실제 속도·작업시간 분포, 큐 등록규칙, 도착·ETA·
재조작·고장·현행 배차로그를 별도 날짜로 보정한 Level 3가 추가로 필요하다.

### C. 현장 PoC 진입

Level 4는 실시간 데이터 연계, 운영자 Shadow Mode(실제 제어 없이 권고만 병행),
안전 중지·기존 방식 복귀 절차가 있어야 한다. 현재 운영사 접근권한이 생겼다고
가정하지 않는다. 대상 터미널·블록·협력 범위가 확정될 때 별도 현장 PoC row를 등록한다.

## 수용 기준

- 10개 터미널 manifest와 구조군·Level·미확인 필드 표가 재현 가능하다.
- `derived/assumed`가 `confirmed`로 승격되는 오류와 임의 0 채움이 0건이다.
- YR-042 대상은 Level 2 자격과 YR-083 런타임 지원 여부를 판정하고, 미달이면
  `profile-shift stress`로만 표시한다.
- 같은 구조군의 Level 2 적격 2크레인 블록 프로파일이 2개 이상이면 YR-042의
  `cross-terminal structure` arm을 연다. 2개 미만이면 `DATA-BLOCKED — NO ELIGIBLE
  PROFILE`로 판정하고 assumed profile stress arm만 허용한다.
- 실제 운영 성능 주장은 Level 3, 현장 PoC는 Level 4를 통과하기 전 금지한다.
- 원문 938줄은 사용자 소유 로컬 자료로 보존한다. YR-082 착수 때 200줄 이하 index와
  터미널별 분할 snapshot·기계가독 manifest로 원격 재현본을 만든다.

이 문서에서 기존 단일 블록 시험은 **시뮬레이션 PoC**, Level 4만 **현장 실증 PoC**로 부른다.

- **산출물**: `configs/terminals/busan/` manifest, 증거·충돌·자료충족도 표,
  YR-042 입력 자격 판정, 현장 자료요청 체크리스트
- **의존**: YR-002 공개 토폴로지·문헌 보정 프로파일, 사용자 제공 터미널 조사
- **연계**: YR-009는 기준환경 한계를 동결하고, YR-042는 Level 2 동형 구조만 시험하며,
  YR-083은 Level 2 자료를 런타임 계약으로 만들고, YR-081은 수직형·다중 블록 구조 확장을 담당한다.
- **착수 순서**: A 공개자료 레지스트리는 YR-009·080과 병행 가능하다. B Level 2
  자격 충족은 YR-042 직전 필수다.
- **범위 밖**: 미확인값 추정, 실제 운영 개선율 주장, 협력기관 확정 전 현장 제어
