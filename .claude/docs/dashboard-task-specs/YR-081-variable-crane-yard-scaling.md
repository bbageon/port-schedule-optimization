# YR-081 — 가변 크레인 수·다중 블록 야드 확장

- **Epic**: RL / **Priority**: ⚪ / **등록일**: 2026-07-20
- **배경**: 현재 공동가치망은 `전역 + 크레인 A + 크레인 B`의 214개 입력과 단일 블록 자료구조에 고정돼 있다. 1대·3대 이상이나 여러 블록은 성능 회귀가 아니라 실행 구조 불일치다. 부산항 조사에서는 RMG/ATC-YT, 수직 ARMG/AGV, 수직 T/C/S/C, 북항 혼합형의 물리가 달라 한 환경으로 묶을 수 없음도 확인됐다.
- **목표**: 단일 블록 PoC를 보존하면서, 야드 규모에서 블록 간 공유 병목을 다루는 계층형 관제 구조를 설계·검증한다.

## 1차 후보 구조

```text
상위 야드 관제기: 블록별 예상부하·여유공간·공용자원 병목을 보고 작업/차량/이송장비 배분
  ↓
블록별 실행기: YR-014에서 채택된 최종 2크레인 bundle 또는 강한 규칙
  ↓
안전 resolver: 블록·레인·이송장비 예약과 물리 제약 보장
```

- 블록 간 공유 자원과 작업 재배분이 없으면 블록별 독립 실행을 기준선으로 둔다.
- 공유 레인·이송장비·도착 물량 재배분이 있으면 상위 관제기의 추가효과를 측정한다.
- 고정형 ARMG는 블록 간 이동 불가를 기본으로 한다. 야드크레인 재배치는 레일·도로와 운영절차상 물리적으로 가능한 터미널 근거가 있을 때만 별도 축으로 연다.
- YR-082의 4개 구조군을 별도 환경군으로 둔다. DGT는 육측·해측 역할과 AGV/FMS, BNCT·BCT는 S/C 이동·인계, 북항은 혼합장비·블록간 이동부터 각각 모델링한다.
- 구조군별 독립 기준선·학습·판정을 유지하며 터미널 전체 평균 하나로 합치지 않는다.
- 크레인 수 {1, 2, 3+}, 이기종 속도·작업구역, 다중 블록을 단계적으로 연다.
- 공동조합이 작으면 중앙 직접평가, 폭발할 때만 set/attention/GNN 또는 QMIX 계열을 비교한다.
- 안전·물리·마감은 계속 mask와 resolver가 담당한다.

## 판정 원칙

단일 블록 성능을 그대로 야드 성능으로 주장하지 않는다. 먼저 독립 블록 기준선과
계층형 관제를 비교해 블록 간 조정의 상금이 실제로 있는지 측정한다. 상금이 없으면
상위 RL을 만들지 않는다. 상금이 있을 때만 학습 구조를 확장한다.

- **산출물**: 가변 agent 계약, 다중 블록 시뮬레이터, 독립 대 계층관제 천장 실험, 조건부 학습정책
- **의존**: YR-082 터미널 구조군·프로파일 자격, YR-083 Level 2 런타임 계약,
  YR-042 구조 일반화 분류, YR-014 단일 블록 최종 판정
- **연계**: 공동조합이 커질 때 YR-069(QMIX 재검토)을 조건부로 연다.
- **범위 밖**: 단일 블록 PoC와 다른 터미널 2크레인 무재학습 시험
