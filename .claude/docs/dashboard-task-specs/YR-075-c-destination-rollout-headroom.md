# YR-075-c — 재조작 목적지 K후보·30분 국소 rollout 헤드룸

- **Epic**: RL / **Priority**: 🟠 / **등록일**: 2026-07-20
- **배경**: YR-075-a의 oracle은 `future_blocked` 프록시이며 실제 총비용을 평가하지 않는다. H1이 이 프록시를 따라잡았다는 사실만으로 목적지 결정의 잔여 헤드룸이 0이라고 할 수 없다.
- **목표**: 재조작 목적지를 행동 차원으로 열고, K개 후보와 30분 지평 안에서 동결 목적의 rollout 비용을 비교해 H1의 **국소 잔여격차**를 추정한다. 전 행동공간·전 에피소드의 수학적 상한은 아니다.

## 설계

1. YR-009 환경과 YR-080 목적·scale을 먼저 동결한다.
2. 물리적으로 가능한 목적지 중 규칙 생성기가 K개를 만들고 `(작업, 크레인, 목적지)` 조합을 평가한다.
3. 30분 중앙 rollout이 각 목적지 뒤의 트럭·본선·레인·이송·재조작 비용을 같은 계약으로 계산한다. 지평 뒤에는 동결 base policy를 쓴다.
4. 완벽 미래정보 rollout은 **낙관적 국소 benchmark**에만 쓰고 배포 정책 입력에는 넣지 않는다.
5. H1, greedy, K후보 rollout benchmark를 paired seed로 비교하며 high·fill0.65 미완주 회귀 셀을 포함한다.

## 판정

- K후보 benchmark가 H1을 사전등록 기준으로 유의하게 이기면 검사한 후보·지평 안에 목적지 결정 상금이 있다. 그때만 관측정보 기반 규칙 또는 학습정책을 개발한다.
- benchmark가 H1을 넘지 못하고 모든 guard를 통과하면 **검사한 K후보·30분 지평·조건에서만** H1 충분으로 판정한다.
- 어느 arm이든 완주율 1.0·backlog 0·제약위반 0을 먼저 통과해야 평균비용을 해석한다.
- 최종 목적지 규칙이 H1에서 바뀌면 교사·증류·FT 성능을 기준환경에서 재확인한다.
