# YR-002 — Phase 0: 대상 터미널·자료 확정 + TerminalProfile 초안

- **Epic**: Data / **Priority**: 🔴 / **등록일**: 2026-07-12
- **배경**: 코드는 부산항 구조를 하드코딩하지 않고 TerminalProfile 설정파일을 읽는다 ([01 §2](../../../docs/구현계획/01_범위_아키텍처_데이터.md)). 크레인 통과가능·안전거리·역할고정 등은 실측자료 확보 전 확정값 금지.
- **목표(수용 기준)**: 실제 구조를 반영한 profile 과 데이터 샘플이 검토됨 ([05 §4 Phase 0](../../../docs/구현계획/05_테스트_로드맵_산출물.md)). 모든 임시값에 `assumed: true` + 근거 기록.
- **범위 밖(non-goal)**: ETA 예측모형, 자료 자동수집 시스템.
- **계획**:
  1. 대상 터미널·블록·YC 확정 (후보: 05 §6 사실·자료 체크리스트)
  2. profile 사실값/가정값 구분 기록
  3. 현행 dispatch rule·작업분류 운영사 확인
  4. 데이터 사전·익명화·반출절차 합의
- **산출물**: `configs/terminals/target_terminal.yaml` 초안, 데이터 사전, 익명화 방안.
- **상태 메모**: 사용자/운영사 결정 필요 — 미확정 동안 가정 프로파일(`poc_single_crane.yaml`)로 개발 선행 가능.
