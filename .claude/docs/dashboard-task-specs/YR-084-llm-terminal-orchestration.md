# YR-084 — 장기 운영화: LLM 감독형 터미널 오케스트레이션·관제

- **Epic**: Infra / **Priority**: ⚪ / **등록일**: 2026-07-22
- **상태**: 장기 북극성만 박제. **현재 착수하지 않는다.**
- **사용자 결정**: 단일 블록 정책을 야드·터미널 단위로 순차 확장한 뒤, 변화가 잦은 실운영 상황을 LLM 감독 에이전트가 해석·조율하는 운영화를 최종 목표로 둔다.

## 배경

현재 채택 후보는 단일 블록·고정 2크레인 공동가치 정책이다. YR-081이 다중 블록의
공유 YT·레인·공간 병목을 다루는 상위 야드 관제를 별도 확장으로 등록했다. 실제 운영화에는
그 위로 선석·QC·YT/AGV·게이트·야드 처리능력을 함께 다루는 결정론적 터미널 계획기와,
계획 변경·복합 고장·비정형 운영 메시지를 안전하게 처리하는 감독 관제가 필요하다.

LLM 자체가 수치 최적화기나 장비제어기가 되는 구조는 채택하지 않는다. 블록·야드·터미널
최적화기는 검증 가능한 코드·수리계획·학습정책으로 유지하고, LLM은 상황에 맞는 도구와
승인된 대응절차를 선택·호출·설명하는 상위 감독자다.

## 동결할 목표 구조

```text
운영자·권한정책
  ↓
LLM 감독 에이전트: 예외 해석·재계획 요청·대안 설명·승인 흐름
  ↓
결정론적 터미널 계획기: 선석·QC·YT/AGV·게이트·야드 용량 배분
  ↓
야드 관제기(YR-081): 블록·차량·공유도로·버퍼 배분
  ↓
블록 정책: 현재 2크레인 공동가치 정책 또는 검증된 규칙
  ↓
resolver·ECS/PLC: 물리 가능성·충돌방지·인터록·실제 장비제어
```

조직도상 LLM은 상위 정보계층이지만 실행 권한은 가장 제한한다. 최종 권한은 운영자,
업무정책, 결정론적 안전계층에 남긴다.

## LLM 허용 범위

- 선사 이메일·관제 메모·상충 알림 같은 비정형 정보를 정형 이벤트 후보로 변환
- 최신 상태와 근거를 조회하고 영향 범위·재계획 지평을 제안
- 승인된 `RUN_SCENARIO`, `REQUEST_REPLAN`, `COMPARE_PLANS` 도구 호출
- 승인된 대응절차(playbook) 선택, 대안별 예상 KPI와 불확실성 설명
- 영향이 크거나 근거가 부족하면 운영자에게 기권·에스컬레이션

## 영구 금지 범위

- YC·YT·AGV·QC에 원시 이동·모터 명령 전송
- 안전거리·물리 mask·인터록·위험물 규칙 변경 또는 우회
- 비용계수·운영목표를 승인 없이 즉석 변경
- 임의 SQL·셸·범용 쓰기 API 사용
- 상태 버전·유효기간 확인 없이 계획 확정
- 결정론적 검증과 필요한 운영자 승인을 건너뛴 TOS 작업지시 발행

## 개방 선결조건

1. YR-080·YR-075-c 이후 단일 블록 정책 bundle과 목적·비용계약 동결
2. YR-082·YR-083·YR-042·YR-014로 구조자료·런타임·단일 블록 일반화 판정
3. YR-081에서 독립 블록 대비 야드 상위관제의 실제 상금 확인
4. 선석·QC·YT/AGV·게이트를 다루는 결정론적 터미널 계획기와 versioned API 마련
5. YR-082 Level 3 운영로그와 Level 4 읽기전용 Shadow Mode 범위 승인
6. 사건 버스·불변 상태 snapshot·계획 버전/TTL·감사·rollback·rule fallback 계약

선결조건이 없으면 착수하지 않고 장기 backlog에 유지한다. 특히 LLM을 먼저 만들고 호출할
안정적인 도구를 나중에 붙이는 순서는 금지한다.

## 향후 연구 질문과 비교군

```text
A. 블록별 독립 운영
B. 결정론적 야드·터미널 계층관제
C. B + 규칙 기반 돌발상황 관리자
D. B + LLM 감독 에이전트
```

- A→B: 계층관제가 공유 병목에서 실제 가치를 만드는가
- C→D: LLM이 규칙 관리자보다 복합 예외 해석·재계획 착수를 개선하는가
- D가 우월하지 않으면 LLM은 설명·검색·관제 UI로만 남기고 실행 오케스트레이션은 채택하지 않는다.

## 수용 기준 초안

- 안전·물리 위반과 무단 도구 실행 0
- 오래된 상태·허위 장비·불가능 계획은 실행 전 100% 거부
- 기존 TOS/규칙 fallback이 LLM·네트워크 장애 중에도 계속 운영
- 총 터미널 비용, 트럭 평균·P95 대기, 선석 초과, 복구시간을 함께 평가
- 잘못된 사건분류·기권·운영자 수정·승인 부담·응답기한 초과를 별도 보고
- 정상·복합장애·상충정보·데이터 지연·간접 프롬프트 공격 시험 포함
- 실제 쓰기 권한 전 오프라인 replay → 추천 전용 Shadow → 운영자 승인 단계를 순서대로 통과

정량 우월 임계값은 실제 운영자 비용·허용위험을 확보한 뒤 사전등록하며 지금 임의로 정하지 않는다.

## 선행 학습 범위

- ISA-95/IEC 62264의 사업계획·운영관리·감독제어·물리제어 경계
- TOS–ECS–FMS–VBS–PCS 이벤트와 권한·책임 분리
- rolling-horizon 재계획과 디지털 트윈 검증·불확실성 관리
- LLM 도구 최소권한, 정형 출력, prompt injection·memory poisoning 방어
- Human-in-the-loop 관제, 승인·반려·설명·수동복귀 UI
