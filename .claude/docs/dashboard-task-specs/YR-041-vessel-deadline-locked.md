# YR-041 — 본선 마감 조건 일반화 locked 시험

- **Epic**: Exp / **Priority**: 🟡 / **재정의**: 2026-07-20
- **배경**: 기존 시나리오는 본선 지연이 대부분 0이라 목적함수의 본선 tier를 판별하지 못했다. 목적 계층과 본선 weight 선택은 YR-080의 train/validation 타이트 마감 셀에서 끝낸다.
- **목표**: YR-089와 YR-075-c 뒤 동결된 최종 정책 bundle이 보지 않은 타이트 본선 마감에서도 본선 작업을 보호하는지 평가한다.

## 계약

- YR-080이 동결한 본선 목적 계층·scale·weight·λ_vessel과 YR-089가 동결한 트럭 시간계약을 그대로 쓴다.
- 이 작업에서는 λ grid 탐색이나 정책 선택을 하지 않는다.
- 신규 seed와 새 마감 강도·본선 작업량을 사용하되, 정확한 격자와 비열등 한계는 실행 전에 사전등록한다.
- 동결 정책과 같은 목적지 규칙을 쓴 SF-SPT를 paired 비교한다.
- 부두 길이·연간 하역능력·안벽크레인 수만으로 본선 마감과 작업량 분포를 만들지 않는다. YR-082 Level 3 선박·작업로그 전에는 문헌 stress 조건으로 표시한다.

## guard

- 모든 seed 완주율 1.0·backlog 0·제약위반 0
- 본선 미완료 0, vessel delay·STS wait 사전등록 비열등
- 트럭 평균·P95 블록 처리시간과 gate-to-gate 턴타임, 이송·레인 지표의 허용 한계 통과
- 본선을 보호한다는 이유로 SERVE가 붕괴하거나 REPOSITION이 퇴화하지 않음

- **산출물**: locked 본선 마감 보고서, seed별 원자료, 목적 계층 충돌 사례
- **의존**: YR-080 본선 목적 동결, YR-089 트럭 시간계약·정책 재판정, YR-075-c 뒤 최종 정책 bundle
- **범위 밖**: test 결과 기반 λ_vessel·weight 재선택, 실제 선사 SLA 주장, 공개 시설규모에서 본선 작업분포 역산
