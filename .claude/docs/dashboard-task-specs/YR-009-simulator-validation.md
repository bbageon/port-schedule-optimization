# YR-009 — 기준환경 동결: 공개 실측 대조와 현실성 한계 확정

- **Epic**: Sim / **Priority**: 🟠 / **등록일**: 2026-07-12 / **재정의**: 2026-07-20
- **배경**: 운영사 협약이 없어 실제 운영 로그 validation은 불가능하다. 현재 문헌 보정 v2는 공개 turn-time과 부분 정합한다. 반입 P50은 9.9~12.1분으로 공개 9~22분 범위지만, 반출 P50은 12.7~18.6분으로 공개 19~33분보다 짧고 작업시간은 3.9~4.0분으로 PEMA 자료 2~3분보다 길다.
- **목표**: 다른 터미널 일반화 시험 전에 기준환경의 보정 범위·남은 차이·사용 가능한 주장을 동결한다.

## 계획

1. 공개자료의 정의·표본기간·터미널 범위와 시뮬레이션 지표를 같은 단위로 맞춘다.
2. 게이트 처리와 장치율 등 공개자료만으로 식별 가능한 모형 수정만 후보로 둔다.
3. 여러 보정안이 같은 공개 통계를 설명하면 임의로 하나에 과적합하지 않고 한계로 기록한다.
4. 환경을 수정하면 기존 `FT+H1`을 진단용으로 재실행해 성능·완주 변화부터 확인한다.
5. 이 단계는 환경 프로파일·생성기·공개 근거의 해시를 동결한다. 최종 정규화·목적·체크포인트·목적지 규칙은 YR-080·075-c 뒤 정책 bundle로 동결한다.

## 수용 기준

- 공개 근거, 변환식, 비교 분위수와 남은 불일치를 재현 가능하게 기록한다.
- 보정하거나 수용한 각 불일치의 이유를 명시한다.
- 기준환경에서 채택 정책의 완주율 1.0·backlog 0·제약위반 0과 규칙 대비 대기 성능을 재확인한다.
- 결론은 **문헌 보정 시뮬레이션 기준**으로 제한하며 실제 운영 검증이라고 부르지 않는다.

- **산출물**: 공개 실측 대조 보고서 v2, 기준환경 manifest, 주장 가능/불가 표, 현 정책 영향 진단
- **의존**: YR-002 문헌 보정 프로파일과 기존 YR-009 1차 보고서
- **범위 밖**: 실제 운영 개선율, 비공개 TOS/VBS 로그 검증, 다른 터미널 정책 성능(YR-042)
