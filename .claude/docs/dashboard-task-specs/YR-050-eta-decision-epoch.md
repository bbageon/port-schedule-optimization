# YR-050 — ETA 선제작업 결정 시점·연착 신호 계약

- **발견**: YR-048은 ETA 데이터와 후보를 열었지만, 엔진은 실행 가능한 SERVE가 있을 때만 결정을
  만든다. 그래서 ETA가 이미 보이는 한산기와 첫 트럭 도착 전에는 PRE_REHANDLE을 고를 수 없다.
- **추가 문제**: ETA가 지났지만 실제 트럭이 아직 오지 않은 최대 300초 동안
  `predicted_arrival_gap_s` 음수가 itc-v1 범위에서 0으로 잘려 “지금”과 “연착”이 같아진다.
- **목표**: 실제 도착시각을 읽지 않고 `provided_eta`와 결정 지평만으로 idle 크레인 결정 시점을
  깨우며, 연착 신호를 학습 입력에 손실 없이 전달한다.
- **설계 경계**: PRE_ADVICE에서만 동작, 낮은 정보수준은 event hash·후보 0 계약 불변. itc-v1
  범위를 조용히 바꾸지 말고 호환 필드 또는 명시적 계약 버전을 선택한다.
- **필수 검증**: 첫 실제 도착 전 PRE_REHANDLE 결정 가능, 첫 트럭 혜택 경로 존재, 같은 시각
  무한 WAIT 재결정 없음, 정보 누출 0, 음수 연착값 보존, 결정론·완주·golden 영향 명시.
- **의존관계**: YR-045 locked 재실험의 선결조건. 완료 전 H2의 “조기 선제작업 완전 활성화”를
  주장하지 않는다.
