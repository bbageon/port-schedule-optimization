# YR-019 — ETA 품질 조건 일반화 locked 시험

- **Epic**: Exp / **Priority**: 🟡 / **재정의**: 2026-07-20
- **배경**: 현재 정책은 실제 블록 도착 `B`에 ±300초를 더한 ETA에서 개발됐다. 완벽 ETA 이득 0도 이 구 `B` 예측과 기존 행동공간에서만 확인됐으며, 현실의 게이트 예약정보 가치까지 기각한 결과가 아니다.
- **목표**: 동결 정책 bundle이 현실적인 ETA 오차·편향·누락·노후 정보에서 규칙 대비 이득과 운영 guard를 유지하는지 평가한다.

## 조건

- `EMPIRICAL`, `BIASED`, `NO_SHOW`, `STALE`을 신규 paired seed로 평가한다.
- `PERFECT`는 정보가치 진단용 상한 arm일 뿐 배포 성능 주장에 넣지 않는다.
- 정책에는 각 arm에서 실제로 발행된 예약창과 단계별 `estimated_block_arrival`·known-mask만 주고 실제 도착시각은 누출하지 않는다.
- 오류 크기·누락률·갱신주기와 사전등록 성능기준을 test 실행 전에 동결한다.
- 터미널 공개 시설자료에는 ETA 오차·노쇼·갱신주기가 없으므로 이를 운영사 실측 분포라고 부르지 않는다. YR-082 Level 3 로그 전에는 조건 stress test다.

## 실행 선결 계약

YR-089 이전 `Job`은 `provided_eta` 한 값만 가지므로 예약과 ETA 갱신 이력, 취소 이벤트를 표현하지
못한다. 따라서 YR-089 시간계약 뒤 `STALE/NO_SHOW` 실행 전에 다음을 구현·검증한다.

- ETA 제공·갱신시각과 값의 이력, 정책이 각 결정시점에 볼 수 있는 최신값
- 예약 취소·노쇼 확정시각과 그 전까지는 미래 결과를 숨기는 정보규칙
- 같은 이벤트를 두 번 소비하지 않는 결정성, 취소 후 후보·wake 제거
- 현 계약으로 표현할 수 없으면 해당 arm을 `CONTRACT-BLOCKED`로 판정

## guard

- 모든 seed 완주율 1.0·backlog 0·제약위반 0
- 평균·P95 블록 처리시간과 gate-to-gate 턴타임, 본선·이송·레인 지표 비열등
- ETA가 나빠질수록 PRE_REHANDLE·REPOSITION·결정 wake가 퇴화하지 않음
- 정보품질별 계산시간·후보 수 보고

- **산출물**: ETA 갱신·취소 계약과 테스트, 품질별 paired 보고서, 누출검사, 실패 경계
- **의존**: YR-089 시간계약, YR-009 재검증, YR-080 목적, YR-075-c 뒤 최종 정책 bundle
- **범위 밖**: test별 재학습, 완벽 ETA를 배포 입력으로 사용, 공개 시설규모에서 ETA 품질 분포 추정
