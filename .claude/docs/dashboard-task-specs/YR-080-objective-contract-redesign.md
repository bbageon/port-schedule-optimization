# YR-080 — 목적함수 문헌정합 재설계와 채택 정책 재판정

- **Epic**: RL / **Priority**: 🟠 / **등록일**: 2026-07-20
- **배경**: YR-071은 잘못된 간섭 지배를 제거하고 트럭 대기를 줄이는 데 성공했다. 이후 문헌 감사에서 본선측 지표 병기, 기준정책 기반 정규화, 간섭의 제약 분리, 13항 축소가 아직 필요하다는 결론이 나왔다. 이는 YR-071 이전 게이트가 아니라 **완료된 YR-071을 재감사하는 후속 게이트**다.
- **목표**: 실제로 판별 가능한 계층 목적과 학습용 스칼라를 동결하고, 목적 변경이 현재 `FT+H1` 채택 판정을 바꾸는지 확인한다.

## 프로토타입 증거와 한계

`be1c056`의 3-seed·600초 진단은 interference 기여 0, 본선 항 활성화와 포화 셀
본선지연 721.92→190.63분을 보였다. 동시에 JR 완주율은 0.956으로 guard를 깨고,
트럭대기 우위도 줄었다. 따라서 방향성 증거일 뿐 목적·정책 채택 결과가 아니다.

## 필수 설계

1. 안전·완주를 하드 guard로 두고, 본선 정시성·트럭 평균/P95 대기·하위 운영비의 우선순위를 명시한다.
2. 본선 지연이 0인 현 시나리오만으로 본선 tier를 정하지 않는다. 이 작업의 train/validation용 타이트 마감 설계 셀에서 본선 계층·λ를 선택해 동결한다. YR-041은 그 뒤 신규 seed로만 검사한다.
3. 비용 scale은 목표 test를 보지 않고 기준정책(SF-SPT)의 train 분포에서 고정한다.
4. weight는 정규화 뒤 업무 우선순위만 표현하며 scale과 섞지 않는다.
5. interference는 안전·물리 충돌이면 mask/resolver로 처리한다. 실제 시간손실 성분을 남길 경우 중복계상 여부를 분해해 증명한다.
6. 13개 원항은 기록에 보존하되 최적화 목적은 육상서비스·본선정시성·재조작·장비효율 등 4~5묶음으로 줄인다.

## 재판정

- 현 NEW 목적과 문헌정합 후보를 동일 시나리오·동일 계획기에서 먼저 비교한다.
- 계층 순위나 교사 행동이 실질적으로 바뀌면 현재 체크포인트는 자동 승계하지 않는다. 교사 재수집→증류/DAgger→FT 재검증을 다시 한다.
- 목적이 행동을 바꾸지 않고 guard·성능도 같을 때만 기존 체크포인트를 유지한다.
- 모든 선택은 train/validation에서 끝내고 locked test를 보고 수정하지 않는다.

## 수용 기준

- 목적 계층, 정규화 출처, weight 근거, hard guard와 scalar target을 각각 명시
- 본선·트럭·운영비의 충돌 사례에서 의도한 순서대로 선택함을 계약 테스트로 확인
- 채택 체크포인트의 유지·재학습·기각 중 하나를 명시적으로 판정
- YR-009 환경 보정이 생기면 변경 환경에서도 같은 재판정을 반복

- **산출물**: 목적 계약, scale provenance, 지배도·상충 보고서, 채택 정책 재판정과 새 manifest
- **의존**: 문헌근거 문서, YR-071 결과. YR-009과 병행 가능하나 최종 manifest는 두 작업 뒤 동결
- **후속**: YR-075-c 목적지 국소 rollout 헤드룸 → YR-041 locked 본선 시험 → YR-042 구조 일반화
- **범위 밖**: 임시 가중치 탐색으로 test 최고값 선택, 실제 운영 우선순위 확정 주장
