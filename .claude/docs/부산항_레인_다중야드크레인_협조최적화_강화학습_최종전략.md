# 부산항 스마트항만형 레인–다중 야드크레인 협조 최적화 강화학습 최종 전략

## 0. 문서 목적

본 문서는 기존의 Exp-1, Exp-2, Exp-3, Exp-4 단계 구분을 제거하고, 처음부터 다음의 **최종 통합 정책**을 구현하기 위한 설계안이다.

> **사전 반출입정보·차량 ETA·컨테이너 장치상태·본선 작업계획·연결 레인·다중 야드크레인 상태를 통합하여, 터미널 전체 운영비용이 최소화되도록 각 야드크레인의 다음 작업을 협조적으로 결정한다.**

초기 구현부터 최종 State·Action·Reward 구조를 모두 포함하고, 이후 실제 데이터와 시뮬레이션 결과를 바탕으로 다음 세부사항을 순차적으로 확정한다.

- 상태값의 구체적인 범위와 이산화 기준
- 후보 작업 수 `K`
- 비용 항목별 가중치
- 본선 위험도 임계값
- 장기대기 기준시간
- ETA 신뢰도 기준
- 재조작 선처리 허용조건
- Q-learning 계열 알고리즘과 학습 파라미터

본 연구는 ETA를 새로 예측하는 연구가 아니다. 부산항에서 제공되거나 별도 시스템에서 산출한 ETA와 사전 반출입정보가 정상적으로 확보된다는 전제하에, 해당 정보를 야드 운영 의사결정에 활용한다.

# 1. 최종 목표

## 1.1 정책 목표

최종 강화학습 정책은 야드크레인의 제한된 작업 처리 자원을 다음 작업 중 어디에 배분할지 결정한다.

- 외부 트럭 반입 작업
- 외부 트럭 반출 작업
- 본선 양하 연계 작업
- 본선 선적 연계 작업
- 반출 컨테이너 재조작
- 차량 또는 본선작업 도착 전 선제 정리
- 야드크레인 위치 조정
- 연결 레인 혼잡 완화를 위한 작업 양보 또는 재배정

정책의 목적은 특정 지표 하나만 줄이는 것이 아니라 다음 비용을 종합적으로 최소화하는 것이다.

```text
외부 트럭 대기
+ 장기대기
+ 야드크레인 이동 및 빈 이동
+ 반출 재조작
+ STS·AGV·SC·YT 대기
+ 본선작업 및 출항 지연
+ 레인 혼잡
+ 크레인 간 상호 대기
+ 작업순번 변경
```

## 1.2 최종 알고리즘의 명칭

제안서 및 연구 문서에서는 다음 명칭을 사용한다.

> **ETA 기반 레인–다중 야드크레인 협조 최적화 강화학습 알고리즘**

여기서 “최적화”는 시뮬레이션과 학습을 통해 총 운영비용을 낮추는 정책을 찾는다는 의미이다. 수학적으로 모든 상황에서 전역 최적해를 보장한다는 의미로 사용하지 않는다.

## 1.3 물리적 배치가 아닌 운영상 동적 배치

본 연구에서의 레인·야드크레인 배치는 물리적으로 레일을 새로 설치하거나 크레인을 재배치하는 문제가 아니다.

다음과 같은 **운영상의 동적 할당과 순서 결정**을 의미한다.

```text
- 어떤 작업을 어느 야드크레인이 담당할지
- 어느 차량 또는 이송장비를 어느 작업 레인으로 유도할지
- 각 야드크레인이 다음에 어떤 작업을 수행할지
- 차량 도착 전에 어떤 반출 재조작을 선처리할지
- 본선작업과 외부 트럭 작업 사이에서 처리 자원을 어떻게 배분할지
- 연결 레인의 혼잡과 크레인 간 상호 영향을 어떻게 완화할지
```

# 2. 참고 논문의 적용 구조

## 2.1 참고 논문의 핵심

김선용 교수의 박사학위논문에서는 스마트 에너지 빌딩을 다음과 같은 MDP와 Q-learning 문제로 구성하였다.

| 구분 | 참고 논문 |
|---|---|
| State | ESS 잔량, 순에너지 수요, 시간대 |
| Action | 전력 구매, ESS 충전, ESS 방전, 전력 판매 |
| Cost | 전력 거래에 따른 건물 에너지 운영비용 |
| 목표 | 현재와 미래의 장기 누적 에너지 비용 최소화 |
| 학습 방식 | 현재 행동 결과와 다음 상태를 관측하여 Q값 갱신 |

논문의 핵심은 미래의 전력수요, 신재생 발전량 및 가격을 완벽하게 알지 못하더라도, 현재 관측정보와 반복 경험을 통해 장기 비용이 낮아지는 정책을 학습한다는 것이다.

## 2.2 항만 문제로의 적용

논문의 행동을 항만 행동과 억지로 일대일 대응시키는 방식은 사용하지 않는다. 특히 “전기를 산다”를 “작업을 큐에 등록한다”로 치환하면 부정확하다.

작업 큐 등록은 야드크레인이 선택하는 행동이 아니라, 새로운 작업 수요가 발생하여 **State가 바뀌는 이벤트**이다.

본 연구에서는 논문의 문제 구성 방식만 다음과 같이 적용한다.

| 구성 | 참고 논문 EMS | 부산항 적용 |
|---|---|---|
| 관리 자원 | ESS에 저장된 에너지 | 야드크레인의 작업시간·작업 슬롯·이동 여력 |
| 수요 | 건물 및 V2G 에너지 수요 | 외부 트럭 및 본선 연계 작업 수요 |
| State | ESS·수요·시간대 | 차량·컨테이너·크레인·레인·본선 상태 |
| Action | 구매·충전·방전·판매 | 수행 가능한 후보 작업 중 다음 작업 선택 |
| Cost | 에너지 운영비용 | 대기·이동·재조작·본선지연 등 야드 운영비용 |
| 목표 | 장기 에너지 비용 최소화 | 장기 터미널 운영비용 최소화 |

쉽게 표현하면 다음과 같다.

```text
참고 논문
현재 에너지 상태를 보고 지금 어떤 에너지 행동을 해야
현재와 미래의 전체 비용이 줄어드는지 학습

본 연구
현재 야드 상태를 보고 지금 어떤 작업을 해야
현재와 미래의 전체 대기·이동·지연 비용이 줄어드는지 학습
```

## 2.3 참고 논문에서 계승하는 요소

- MDP 기반 상태·행동·비용 정의
- 모델프리 Q-learning 계열 접근
- 현재 비용뿐 아니라 미래 비용을 고려하는 할인율
- 탐색과 활용을 병행하는 정책
- 실제 또는 준실제 데이터 기반 시뮬레이션 검증
- 비용을 직접 최소화하거나 `Reward = -Cost`로 변환하는 구조

## 2.4 항만환경에 맞게 추가하는 요소

- 차량 ETA와 ETA 신뢰도
- 외부 트럭 누적 대기시간
- 반입·반출 작업 구분
- 반출 컨테이너의 blocker와 예상 작업량
- 본선작업 목표 완료시각과 잔여 물량
- 본선 Slack 및 지연 위험도
- STS·AGV·SC·YT 대기
- 연결 레인 점유와 혼잡
- 다중 야드크레인 간 작업 경쟁 및 상호 대기
- 작업 수행 가능 여부를 제한하는 Action Masking

# 3. 적용 범위와 기본 가정

## 3.1 적용 범위

최종 정책은 수직형 또는 수평형 야드 구조에서 다음 작업을 통합하여 다룬다.

| 작업 유형 | 설명 |
|---|---|
| 외부 트럭 반입 | 트럭이 가져온 컨테이너를 야드에 장치 |
| 외부 트럭 반출 | 야드에 있는 컨테이너를 인출하여 트럭에 상차 |
| 본선 양하 연계 | 선박에서 내려온 컨테이너를 야드에 장치 |
| 본선 선적 연계 | 야드의 컨테이너를 선박 선적 흐름으로 인출 |
| 재조작 | 대상 컨테이너 인출을 위해 blocker를 이동 |
| 선제 정리 | 예상 작업 전에 blocker 이동, 빈 슬롯 정리, 크레인 위치 조정 |
| 레인·작업 할당 | 연결 레인과 다중 크레인 상태를 고려한 작업 분배 |

## 3.2 핵심 가정

- 야드크레인은 정해진 물리적 이동 제약 안에서 운행한다.
- 강화학습은 레일의 방향을 변경하지 않는다.
- 현재 수행 중인 크레인 작업을 중간에 중단하지 않는다.
- 의사결정은 현재 작업 사이클이 끝난 뒤 수행한다.
- 차량 ETA는 외부 입력값이며, ETA 신뢰도를 별도로 관리한다.
- 본선작업 위험도를 계산하려면 계획 본선작업 완료시각이 필요하다.
- 계획 완료시각이 없으면 출항예정시각에서 출항 준비 버퍼를 차감한 대체값을 사용하되 `assumed: true`로 관리한다.
- 목표 완료시각조차 확보되지 않으면 “본선 지연 위험도”가 아니라 “본선 지연 징후”만 산출한다.
- 여러 야드크레인이 연결된 레인 또는 공용 작업 후보에 상호 영향을 줄 수 있다고 가정한다.
- 실제 장비 직접제어보다 초기에는 다음 작업 추천 또는 작업 할당 추천 방식으로 적용한다.

# 4. 최종 운영 흐름

```text
사전 반출입정보·ETA·본선 작업계획 수신
→ 외부 트럭·본선·컨테이너·레인·야드크레인 상태 갱신
→ 각 야드크레인별 수행 가능한 후보 작업 생성
→ 불가능·위험 작업 Action Masking
→ 다중 야드크레인이 후보 작업을 협조적으로 선택
→ 중복 선택·레인 충돌·작업 경쟁 조정
→ 야드크레인 이동 및 작업 수행
→ 대기·이동·재조작·본선지연 비용 계산
→ 공동 Reward 산출
→ Q값 또는 Q-network 갱신
→ 다음 작업 사이클에서 반복
```

# 5. 의사결정 시점

## 5.1 이벤트 기반 의사결정

참고 논문은 5분 단위의 고정 시간 간격을 사용했지만, 야드크레인 작업은 작업별 소요시간이 다르다. 따라서 본 연구는 **작업 완료 이벤트 기반 의사결정**을 기본으로 한다.

주요 의사결정 이벤트는 다음과 같다.

- 야드크레인 현재 작업 완료
- 외부 트럭 블록 도착
- 게이트 진입정보 및 블록 도착예상시각 갱신
- 사전 반출입정보 또는 ETA 변경
- 본선 연계 작업 신규 발생
- 본선작업 목표 완료시각 또는 작업계획 변경
- STS·AGV·SC·YT 대기 발생
- 레인 혼잡도 임계치 초과
- 장비 장애 또는 크레인 작업 불가 상태 발생

## 5.2 시간 길이가 다른 작업의 할인 처리

작업별 소요시간이 다르므로, 고정된 한 단계의 할인율만 적용하면 3분 작업과 15분 작업을 동일하게 취급하는 문제가 생긴다.

기준시간을 5분으로 둘 경우 실제 적용 할인율은 다음과 같이 보정한다.

```text
effective_gamma = gamma_base ^ (실제 경과시간 / 5분)
```

예시:

```text
기본 γ = 0.95

5분 작업  → 0.95
10분 작업 → 0.95²
15분 작업 → 0.95³
```

이를 통해 긴 작업이 미래 상태와 비용에 미치는 영향을 시간 길이에 맞게 반영한다.

# 6. 최종 MDP 정의

| MDP 요소 | 최종 정의 |
|---|---|
| Agent | 각 야드크레인 |
| Global State | 터미널 전체 차량·컨테이너·본선·레인·크레인 상태 |
| Local Observation | 해당 크레인의 위치·후보 작업·인접 레인·주변 크레인 상태 |
| Action | 해당 크레인이 다음에 수행할 후보 작업 또는 대기·위치조정 선택 |
| Transition | 작업 수행 후 대기열, 컨테이너 위치, 크레인 위치, 본선 및 레인 상태 변화 |
| Reward | 터미널 전체 운영비용의 음수 + 필요한 경우 크레인별 지역 비용 |
| Policy | 전체 비용이 낮아지는 크레인별 협조 작업 선택 규칙 |
| Episode | 일자·교대·본선 작업구간 등 하나의 운영 시뮬레이션 구간 |

# 7. State 설계

State는 모든 원시 데이터를 무조건 입력하는 것이 아니라, 의사결정에 필요한 값을 정규화·요약하여 사용한다.

## 7.1 야드크레인별 상태

```text
- 크레인 ID
- 현재 Block / Bay / 위치
- 작업 가능 여부
- 현재 작업 유형
- 현재 작업 예상 종료시각
- 직전 이동 방향
- 현재 적재 여부
- 다음 후보 위치까지의 이동거리와 예상 이동시간
- 최근 일정시간 처리량
- 최근 일정시간 빈 이동시간
- 현재 할당 작업 수와 예상 작업부하
```

## 7.2 외부 트럭 상태

```text
- 차량 ID
- 반입 / 반출 구분
- 공컨 / 적컨 구분
- 사전 반출입정보 등록시각
- 게이트 진입 여부와 진입시각
- 블록 도착 여부와 도착시각
- 목적 블록·레인
- 차량 ETA 또는 블록 도착예상시각
- ETA 신뢰도
- 현재 누적 대기시간
- 장기대기 기준 초과시간
- 작업순번 변경 횟수
```

## 7.3 컨테이너 상태

```text
- 컨테이너 ID
- 작업 유형: 반입 / 반출 / 본선 선적 / 본선 양하
- 현재 Bay / Row / Tier
- 대상 블록
- 작업 가능 여부
- 상부 blocker 수
- blocker 이동 가능 슬롯
- blocker 복원 필요 여부
- 예상 재조작 횟수
- 예상 총 취급 횟수
- 예상 서비스 시간
- 현재 크레인별 접근거리
- 본선 마감 또는 차량 ETA까지 남은 시간
```

## 7.4 반출 작업량 산정

반출 작업은 차량 한 대를 작업 한 건으로 단순 계산하지 않는다.

### 기본 예상 취급 횟수

```text
Expected Handling Count
= 대상 컨테이너 인출 1회
+ blocker 이동 횟수
+ blocker 복원 횟수
```

blocker를 다른 유효 슬롯에 이동한 뒤 복원하지 않는 경우:

```text
예상 취급 횟수 = blocker 수 + 1
```

blocker를 임시 이동 후 복원해야 하는 경우:

```text
예상 취급 횟수 = 2 × blocker 수 + 1
```

실제 값은 blocker의 이동 위치, 복원 필요성, 다른 작업과의 연계 가능성에 따라 달라지므로 시뮬레이터가 작업 단위로 산출한다.

### 예상 서비스 시간

```text
Expected Service Time =
크레인 접근 이동시간
+ blocker 이동시간
+ 대상 컨테이너 인출시간
+ 트럭 상차시간
+ blocker 복원시간
```

RL에는 단순 blocker 수뿐 아니라 다음 값을 함께 입력한다.

- `estimated_handling_count`
- `estimated_rehandle_time`
- `estimated_total_service_time`

## 7.5 반입 작업의 처리

반입 컨테이너는 트럭 도착 전 야드에 존재하지 않으므로 컨테이너 자체를 미리 재조작할 수 없다.

반입 작업에 대해서는 다음을 State로 반영한다.

```text
- 지정 장치 위치
- 장치 위치 사용 가능 여부
- 지정 위치까지의 크레인 이동시간
- 기본 상하차 처리시간
- 주변 레인 혼잡도
```

장치 위치 자체를 정책이 선택하도록 확장할 경우, 향후 반출 재조작 위험도를 별도의 장치 위치 비용으로 추가할 수 있다. 그러나 최초 구현에서는 기존 TOS가 지정한 장치 위치를 입력으로 사용하는 것을 기본으로 한다.

## 7.6 블록·레인 상태

```text
- 블록별 현재 대기열 길이
- 레인별 차량·AGV·SC·YT 대기 대수
- 레인별 평균·최대 대기시간
- 레인별 점유 여부
- 레인별 통과 가능 여부
- 레인 연결 그래프
- 레인별 혼잡도
- 레인 진입 예정 차량 수
- 레인별 예상 해소시간
- 특정 작업으로 인해 다른 크레인이 받는 영향
```

## 7.7 다중 크레인 상호 상태

```text
- 전체 야드크레인 위치
- 각 크레인의 현재·예정 작업
- 각 크레인의 예상 작업 종료시각
- 작업별 수행 가능 크레인 목록
- 동일 작업을 선택할 수 있는 경쟁 크레인 목록
- 인접 또는 연결 레인 사용 계획
- 크레인별 누적 작업량
- 크레인별 예상 부하
- 크레인 간 상호 대기 가능성
```

## 7.8 본선작업 상태

본선작업을 우선해야 하는 이유를 학습시키려면 “본선작업 있음/없음”만으로는 부족하다.

최소한 다음 파라미터가 필요하다.

```text
- 선박·항차 ID
- 본선 작업유형: 선적 / 양하
- 계획 본선작업 시작시각
- 계획 본선작업 완료시각
- 출항예정시각
- 출항 준비 버퍼시간
- 본선 전체 작업량
- 본선 잔여 작업량
- 최근 본선 처리생산성
- 목표 처리생산성
- 본선 잔여 예상 처리시간
- 예상 본선작업 완료시각
- STS 누적 대기시간
- AGV·SC·YT 대기 대수 및 누적 대기시간
- 긴급 본선작업 수
- 본선 Slack
- 본선 지연 위험도
```

## 7.9 본선 Slack 계산

```text
Vessel Slack =
계획 본선작업 완료시각
- 현재시각
- 예상 잔여 처리시간
```

또는 예상 완료시각을 기준으로:

```text
예상 본선 지연시간 =
max(0, 예상 본선작업 완료시각 - 계획 본선작업 완료시각)
```

### 계획 완료시각 확보 우선순위

1. TOS 또는 본선 작업계획의 목표 완료시각
2. 운영계획상 계산된 계획 완료시각
3. `출항예정시각 - 출항 준비 버퍼시간`
4. 운영자가 설정한 임시 마감시간

3~4번을 사용할 경우 가정값임을 명시한다.

## 7.10 본선 위험도

본선 위험도는 다음 요소를 이용해 0~1 또는 0~100 범위로 정규화한다.

```text
Vessel Risk =
f(
  Slack 부족 정도,
  예상 본선 지연시간,
  STS 대기시간,
  AGV·SC·YT 대기시간,
  잔여 작업량,
  목표 생산성 미달 정도
)
```

초기 단계 구분 예시는 다음과 같다.

| 위험도 | 상태 | 운영 의미 |
|---:|---|---|
| 0~20 | 정상 | 외부 트럭과 본선작업 균형 처리 |
| 21~40 | 주의 | 본선작업 우선도 일부 상승 |
| 41~60 | 경계 | 본선 연계 작업 우선 고려 |
| 61~80 | 위험 | 본선작업 강한 우선 |
| 81~100 | 지연 또는 매우 위험 | 본선 지연비용 최대 수준 반영 |

목표 완료시각이 없는 경우 이 값은 “위험도”가 아니라 STS·이송장비 대기와 처리량 저하를 기반으로 한 “지연 징후 점수”로 별도 관리한다.

## 7.11 시간 관련 상태

```text
- 현재시각
- 운영 시간대
- 차량별 누적 대기시간
- ETA까지 남은 시간
- 블록 도착예상시간까지 남은 시간
- 본선 목표 완료시각까지 남은 시간
- 예상 본선 잔여 처리시간
- 본선 Slack
- 현재 작업 예상 종료까지 남은 시간
- 레인 예상 해소시간
```

# 8. Action 설계

## 8.1 작업 큐 등록은 Action이 아님

외부 트럭 또는 본선 연계 작업이 큐에 등록되는 것은 새로운 작업 수요가 발생한 것이다.

```text
신규 작업 큐 등록
→ State 변경
→ 후보 작업 목록 갱신
→ RL이 다음 작업을 선택
```

강화학습의 실제 Action은 **등록된 작업 중 어떤 작업을 어느 크레인이 다음에 수행할지 결정하는 것**이다.

## 8.2 후보 작업 기반 Action

각 야드크레인마다 현재 수행 가능한 후보 작업을 생성하고, 상위 `K`개 후보 중 하나를 선택하도록 한다.

```text
Action_i ∈ {
  Candidate Job 1,
  Candidate Job 2,
  ...
  Candidate Job K,
  Pre-marshalling Job,
  Reposition,
  Hold / Yield
}
```

후보 작업에는 다음 작업유형이 포함된다.

| Action 유형 | 설명 |
|---|---|
| 외부 트럭 반입 처리 | 블록에 도착해 작업 가능한 반입차량 처리 |
| 외부 트럭 반출 처리 | 대상 컨테이너를 인출하여 반출차량에 상차 |
| 본선 양하 연계 처리 | 선박에서 내려온 컨테이너 야드 장치 |
| 본선 선적 연계 처리 | 선적 대상 컨테이너 인출 및 이송 연계 |
| 반출 재조작 선처리 | ETA가 확인된 반출 대상의 blocker 사전 이동 |
| 반입 위치 주변 정리 | 지정 장치 위치의 사용 가능 상태 확보 |
| 크레인 위치 조정 | 향후 작업을 위해 특정 Bay로 사전 이동 |
| 작업 양보·대기 | 연결 레인 또는 다른 크레인의 작업을 위해 대기 |
| 작업 재배정 | 다른 크레인이 수행하는 것이 전체 비용상 유리한 작업을 이관 |

## 8.3 후보 작업 우선 필터

전체 작업을 모두 Action으로 넣으면 행동공간이 지나치게 커지므로 다음 기준으로 상위 후보를 만든다.

```text
Candidate Score =
차량 누적 대기
+ 본선 위험도
+ 작업 마감 임박도
- 크레인 접근시간
- 예상 서비스 시간
- 예상 재조작 비용
- 레인 혼잡 영향
```

이 후보 점수는 최종 정책이 아니라 탐색 대상 축소용 필터이다. 최종 선택은 Q-value가 수행한다.

## 8.4 재조작 선처리 허용조건

차량 도착 전 재조작 선처리는 다음 조건을 모두 또는 일정 수준 이상 만족할 때만 Action 후보에 포함한다.

```text
- 반출 대상 컨테이너가 이미 야드에 존재
- blocker가 존재하여 사전 작업 효과가 있음
- 차량 ETA 신뢰도가 기준 이상
- 차량 도착 전 작업 완료 가능
- 현재 크레인이 대상 위치와 가깝거나 유휴 상태
- 현재 장기대기 차량을 과도하게 악화시키지 않음
- 본선 위험도를 악화시키지 않음
- blocker 임시 이동 슬롯 확보
- 사전 이동 후 추가 재조작을 유발하지 않음
```

따라서 정책은 “현재 차량을 무조건 미루고 미래 차량을 준비”하지 않는다. 현재와 미래의 총비용을 비교하여 사전 재조작이 유리할 때만 선택한다.

## 8.5 Action Masking

현실적으로 불가능하거나 안전하지 않은 작업은 Q값과 관계없이 선택하지 못하게 한다.

```text
- 아직 블록에 도착하지 않은 반입 컨테이너 상하차 금지
- 작업 불가능 상태 컨테이너 선택 금지
- 현재 수행 중인 작업 중단 금지
- 물리적 이동 제약 위반 금지
- 레인 통과·점유 제약 위반 금지
- blocker 이동 슬롯이 없는 재조작 금지
- 다른 크레인이 확정한 동일 작업 중복 선택 금지
- 본선 또는 외부 트럭의 필수 마감 제약 위반 금지
- 정비·장애 상태 크레인 작업 선택 금지
```

## 8.6 다중 크레인 공동 Action

각 크레인이 독립적으로 후보를 선택하면 같은 작업을 중복 선택하거나 연결 레인에 병목을 만들 수 있다.

따라서 최종 공동 Action은 다음처럼 정의한다.

```text
Joint Action =
[a_YC1, a_YC2, ..., a_YCn]
```

공동 Action은 다음 조건을 만족해야 한다.

- 동일 작업 중복 배정 없음
- 레인 점유 및 통행 제약 충족
- 작업별 수행 가능 크레인 조건 충족
- 본선·외부트럭 우선순위와 마감 제약 충족
- 크레인별 동시 수행 가능 작업 수 충족
- 전체 비용 기준으로 상호 양보·재배정 가능

# 9. Action의 핵심 트레이드오프

| 선택 | 줄어드는 비용 | 증가할 수 있는 비용 |
|---|---|---|
| 본선작업 우선 | STS·이송장비 대기, 본선 지연 | 외부 트럭 대기 |
| 외부 트럭 우선 | 기사 대기와 민원 | 본선 지연 |
| 근접 작업 우선 | 이동거리와 빈 이동 | 먼 위치 장기대기 |
| 장기대기 우선 | 공정성과 장기대기 | 이동거리와 처리시간 |
| 짧은 작업 우선 | 단기 처리량 | 복잡한 반출 작업의 기아현상 |
| 재조작 선처리 | 미래 반출차량 대기 | 현재 큐 처리 지연 |
| 작업순서 변경 | 전체 효율 | 기사 안내 혼선 |
| 레인 분산 | 혼잡 | 특정 크레인의 이동 증가 |
| 개별 크레인 최적 | 해당 크레인 효율 | 터미널 전체 병목 |

강화학습 적용의 핵심은 한 가지 규칙을 고정 적용하는 것이 아니라, 현재 상황에 따라 이 트레이드오프의 균형점을 학습하는 것이다.

# 10. Reward 설계

## 10.1 기본 구조

비용 최소화 문제로 정의한다.

```text
Reward_t = - Normalized Total Cost_t
```

비용이 작을수록 Reward가 커진다.

## 10.2 전체 비용함수

```text
Total Cost_t =

w_truck_wait       × 외부 트럭 대기시간 증가분
+ w_long_wait      × 장기대기 초과시간
+ w_crane_travel   × 야드크레인 이동시간
+ w_empty_travel   × 야드크레인 빈 이동시간
+ w_rehandle       × 재조작 시간 또는 횟수
+ w_sts_wait       × STS 대기시간
+ w_transfer_wait  × AGV·SC·YT 대기시간
+ w_vessel_delay   × 예상 본선 지연시간
+ w_depart_delay   × 출항 지연시간
+ w_lane_cong      × 레인 혼잡시간
+ w_interference   × 크레인 간 상호 대기시간
+ w_resequence     × 작업순번 변경비용
+ w_imbalance      × 크레인별 작업부하 불균형
```

필요한 경우 정상적인 작업 완료와 처리량에 작은 양의 보상을 추가할 수 있지만, 대기시간 감소와 처리량 증가를 동시에 중복 보상하지 않도록 주의한다.

## 10.3 비용 정규화

단위가 분, m, 회, 대수 등으로 다르기 때문에 원시값을 그대로 합산하지 않는다.

```text
normalized_metric =
현재값 / 기준값
```

기준값 예시:

- 트럭 대기 기준: 기존 운영 평균대기시간
- 장기대기 기준: 30분 또는 운영 기준
- 이동 기준: 블록 길이 또는 평균 이동거리
- 재조작 기준: 차량당 평균 재조작 횟수
- 본선 지연 기준: 계획 완료 대비 허용 지연시간
- 레인 혼잡 기준: 레인 처리용량 또는 평균 점유시간

가중치는 정규화된 비용에 적용한다.

## 10.4 장기대기 비용

95퍼센타일 대기시간은 최종 평가에는 적합하지만, 매 작업 단계의 Reward에서 직접 계산하기 어렵다.

학습 중에는 차량별 임계시간 초과분을 이용한다.

```text
Long Wait Cost =
Σ max(0, 차량별 현재 대기시간 - 장기대기 기준시간)
```

필요한 경우 초과시간이 길어질수록 비용이 비선형적으로 증가하도록 한다.

```text
Long Wait Cost =
Σ [max(0, wait_i - threshold)]²
```

이를 통해 작업시간이 짧은 차량만 계속 처리하여 복잡한 반출 차량이 무한정 밀리는 기아현상을 방지한다.

## 10.5 반출 재조작 비용

```text
Rehandle Cost =
재조작 횟수
또는
재조작에 소요된 실제 크레인 점유시간
```

가능하면 횟수보다 실제 점유시간을 우선 사용한다.

추가로 재조작으로 생성된 미래 blocker 또는 임시 장치공간 점유도 비용으로 반영할 수 있다.

## 10.6 본선 비용

```text
Vessel Cost =

a × STS 대기시간
+ b × AGV·SC·YT 대기시간
+ c × 예상 본선 지연시간
+ d × 목표 생산성 미달량
+ e × 긴급 본선작업 미처리량
+ f × 예상 출항 지연시간
```

본선 작업의 위험도에 따라 본선 비용 가중치를 동적으로 조정한다.

```text
Total Cost =
Truck Cost
+ λ_vessel(risk_t) × Vessel Cost
+ Yard Crane Cost
+ Lane / Coordination Cost
```

초기 후보:

| 본선 상태 | `λ_vessel` 초기값 |
|---|---:|
| 정상 | 1.0 |
| 주의 | 1.5 |
| 경계 | 2.5 |
| 위험 | 4.0 |
| 실제 지연 | 6.0 |

이는 확정값이 아니라 민감도 분석을 위한 초기값이다.

## 10.7 동적 본선 우선도

본선작업이 밀리면 다음 과정으로 우선도가 올라간다.

```text
Slack 감소 또는 음수
→ 예상 본선 지연시간 증가
→ 본선 위험도 증가
→ λ_vessel 증가
→ 본선작업을 미루는 Action의 비용 증가
→ 각 야드크레인이 본선 연계 작업을 선택할 가능성 증가
```

현재 수행 중인 작업을 즉시 중단하지 않고, 현재 사이클이 끝난 뒤 다음 공동 Action에서 우선순위를 조정한다.

## 10.8 지역 Reward와 전역 Reward

다중 야드크레인은 자기 이동거리만 줄이는 지역 최적화에 빠질 수 있다.

따라서 각 Agent의 Reward는 다음처럼 구성한다.

```text
Reward_i =
β_global × Global Terminal Reward
+ β_local × Local Crane Reward_i
```

전역 Reward:

```text
- 전체 외부 트럭 대기
- 전체 본선 지연
- 전체 이송장비 대기
- 전체 레인 혼잡
- 전체 재조작
```

지역 Reward:

```text
- 해당 크레인의 이동·빈 이동
- 해당 크레인 큐의 장기대기
- 해당 크레인의 불필요한 대기
```

초기에는 전역 Reward 비중을 높게 두어 개별 크레인의 이기적 선택을 방지한다.

# 11. Q-learning 계열 알고리즘

## 11.1 Tabular Q-learning을 그대로 쓰기 어려운 이유

최종 정책은 다음 요소를 포함하여 상태와 행동 공간이 크다.

- 다수 차량과 컨테이너
- 차량별 ETA·대기시간
- 컨테이너별 Bay·Row·Tier와 blocker
- 본선 잔여 작업과 위험도
- 연결 레인 상태
- 여러 야드크레인의 공동 Action

따라서 참고 논문의 단순 Q-table을 그대로 적용하면 상태·행동 공간이 폭발할 가능성이 높다.

## 11.2 최종 권장 구조

최종 구현은 Q-learning 계열의 **협조형 Multi-Agent Q-learning**을 사용한다.

우선 검토 알고리즘:

> **QMIX 기반 중앙집중 학습·분산 실행 구조**

선정 이유:

- 각 야드크레인의 행동이 이산적인 후보 작업 선택으로 구성됨
- 터미널 전체 공동비용을 최소화해야 함
- 개별 크레인 Q-value를 전역 공동 Q-value로 결합 가능
- 학습할 때 전체 상태를 사용하고, 실행할 때 크레인별 관측값으로 행동 가능
- 연결 레인과 작업 경쟁에 따른 협조 학습에 적합

비교 후보:

- Independent DQN: 협조 없는 기준 모델
- Centralized DQN: 전체 작업을 중앙에서 한 번에 선택하는 기준 모델
- QMIX: 최종 우선 후보
- MAPPO: 연속·복합 행동으로 확장할 때 비교
- 제약 기반 스케줄러 + RL: 운영 안전성과 마감 제약을 강하게 보장할 때 결합

## 11.3 중앙집중 학습·분산 실행

```text
학습 시:
터미널 전체 State, 전체 공동 Action, 전체 비용을 사용

실행 시:
각 야드크레인이 자기 관측값과 공유 요약정보를 보고
자신의 다음 후보 작업을 선택
```

실제 운영 초기에는 분산 실행 대신 중앙 추천 서버가 모든 크레인의 다음 작업을 한 번에 산출하는 방식도 가능하다.

## 11.4 Q-learning 기본 구조

비용 최소화 형태:

```text
Q(s_t, a_t) ←
(1 - α) Q(s_t, a_t)
+ α [
  Cost_t
  + effective_gamma × min_a Q(s_{t+1}, a)
]
```

Reward 최대화 형태:

```text
Reward_t = -Cost_t

Q(s_t, a_t) ←
(1 - α) Q(s_t, a_t)
+ α [
  Reward_t
  + effective_gamma × max_a Q(s_{t+1}, a)
]
```

QMIX에서는 개별 Agent의 Q값을 Mixing Network가 전역 `Q_tot`으로 결합한다.

# 12. 학습 파라미터 초기값

참고 논문은 다음 값을 사용하였다.

```text
학습률 α = 0.1
할인율 γ = 0.95
탐색률 ε = 0.2
```

본 연구의 초기 구현도 이를 기준점으로 사용하되, 다음 범위에서 고정값을 선정한다.

| 파라미터 | 초기 기준 | 검토 범위 |
|---|---:|---:|
| 학습률 `α` | 0.10 | 0.05 / 0.10 / 0.20 |
| 기본 할인율 `γ` | 0.95 | 0.90 / 0.95 / 0.98 |
| 초기 탐색률 `ε_start` | 0.20 | 0.10 / 0.20 / 0.30 |
| 최종 탐색률 `ε_end` | 0.02 | 0.01 / 0.02 / 0.05 |
| 후보 작업 수 `K` | 5 | 5 / 10 / 15 |
| 장기대기 기준 | 30분 | 운영 기준에 따라 조정 |
| 기준시간 | 5분 | 실제 평균 사이클 기반 조정 |

탐색률은 학습 진행에 따라 점진적으로 감소시킨다.

```text
ε_start → ε_end
```

실제 운영에서는 탐색 행동을 그대로 적용하지 않고, 시뮬레이션에서 학습된 정책의 활용 모드 또는 운영자 승인 모드로 적용한다.

- 사전 재조작 Action 제거
- 본선 위험도 제거
- 레인 혼잡 비용 제거
- 다중 크레인 협조 제거
- 장기대기 비용 제거

이는 별도의 Exp-1~4 정책을 만드는 것이 아니라, **동일 최종 정책에서 요소별 효과를 검증하는 Ablation Test**이다.

# 18. 성공 판단 기준

최종 정책은 기존 운영 또는 도착순·고정우선순위 기반 스케줄러와 비교한다.

최소 성공조건은 다음과 같다.

```text
- 평균대기시간 감소
- 95퍼센타일 대기시간 악화 없음
- 본선 계획 완료시간 악화 없음
- STS·AGV·SC·YT 대기 악화 없음
- 재조작 또는 크레인 이동시간 중 하나 이상 감소
- 안전·물리·마감 제약 위반 없음
```

더 높은 수준의 성공조건은 다음과 같다.

```text
외부 트럭 대기 감소
+ 본선 지연 감소
+ 크레인 이동·재조작 감소
+ 레인 혼잡 감소
```

한 지표만 좋아지고 다른 핵심 지표가 크게 악화되는 정책은 최종 정책으로 채택하지 않는다.

# 20. 제안서용 최종 문장

> 본 연구는 김선용 교수의 Q-learning 기반 에너지관리 연구에서 제시된 MDP·장기 비용 최소화 구조를 부산항 야드 운영에 확장 적용한다. 사전 반출입정보와 차량 ETA, 반입·반출 구분, 반출 컨테이너의 Bay·Row·Tier 및 blocker, 야드크레인 위치, 연결 레인 혼잡, STS·AGV·SC·YT 대기, 본선 잔여 작업량과 계획 완료시각을 통합 State로 구성한다. 각 야드크레인은 수행 가능한 외부 트럭·본선 연계·재조작·위치조정 작업 중 다음 작업을 Action으로 선택하며, 외부 트럭 대기, 장기대기, 크레인 이동, 재조작, 내부 이송장비 대기, 본선 및 출항 지연, 레인 혼잡을 통합한 비용의 음수를 공동 Reward로 학습한다. 최종적으로 중앙집중 학습·분산 실행 기반의 협조형 Multi-Agent Q-learning을 통해 개별 크레인의 지역 최적화가 아닌 터미널 전체 운영비용이 최소화되는 레인–다중 야드크레인 협조 정책을 도출한다.
