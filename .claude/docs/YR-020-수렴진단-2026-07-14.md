# 계열 1 수렴 진단 (2026-07-14) — 설계 결함 vs 학습량 부족 판별

> 질문: "Exp 결과 부진이 설계 문제인가, 상태공간이 커서 수렴이 안 된 것인가?"
> 방법: 기존 산출물(Q-table 방문통계·train_log·test 12 seeds 재실행 fallback 실측)만으로 진단.
> 스크립트: scratchpad `diag_convergence.py` (본 문서 표가 원 수치 박제본).

## 결론 (요지)

1. **Exp-1 조건에서 상태공간은 병목이 아니다** — 유효 12,288칸, 평가 중 미방문 fallback 은 결정의 4.6~10.3%뿐 (mask+휴리스틱 fallback 설계가 의도대로 방어).
2. **그러나 전 조건이 수렴 전(under-trained) 상태다** — 사용된 (s,a) 방문 중앙값 3~6회 (α=0.1 고정 기준 태부족), 평가 결정의 24~42%가 방문<5 인 얇은 추정에 의존, 예산 2.5×(e10)에서 성능·밀도 모두 개선 지속. **현재 수치(-15.2%/-10.4%)는 "수렴된 설계의 성능"이 아니라 하한**으로 읽어야 한다.
3. **Exp-2/3 열세는 상태공간 희석이 정량 확인됨** — 유효 196,608칸에 커버 1.3~2.5%, 방문 중앙값 2회. "정보 무익" 판정은 아직 불가 — 예산 통제 실험(YR-020)이 필요하다.

## [2] 학습 곡선 (exp1_hjnc, 120 episodes)

- states_known: 63(ep1) → 887(ep30) → 1,093 → 1,226 → 1,336(ep120). 마지막 20% 에피소드에서도 +77 (5.8%) — 감속 중이나 미포화.
- 에피소드 return 평균: 전반부 -169.16 → 후반부 -169.19 (평평). ε 혼입·시나리오 편차로 return 은 학습 신호를 보여주지 못함 — 학습 진행 판단에는 paired 평가만 유효.

## 시사점 (후속 작업 반영)

- **YR-018 (w_tail grid)**: 현 예산(4ep)으로 grid 를 돌리면 "수렴 전 스냅샷"끼리의 비교가 됨 — **학습예산 축(최소 e10)을 grid 에 동반**하고, 본 진단의 fallback/thin 지표를 리포트에 포함해야 해석 가능.
- **YR-020**: 희석 가설의 방문통계 증거 확보됨. 남은 판별은 "예산을 EXP2/3 에 크게 줘도 못 따라오는가"(정보 무익) vs "따라오는가"(희석) — 3A e10 격차 축소(16.56→15.14분)는 희석 쪽 정황.
- **수렴 관점 개선 후보**: α=0.1 고정 → 방문수 감쇠(n^-p, 계열 2 방식)가 tabular 수렴 보장 조건에 부합 — YR-018/020 실험 시 옵션으로 검토.
