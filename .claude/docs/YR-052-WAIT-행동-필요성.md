# YR-052 — WAIT 행동은 필요한가 (근거 분석 + 결정)

> 계기: 사용자 질문 "WAIT 동작이 필요한 근거가 있어?"
> 결론: **강제 WAIT(양보)는 구조적으로 필수. 전략적 WAIT(할 일 있는데 대기)은 현재 조건에서
> 실증 이득 0이고 baseline에 오히려 독**.
> **결정 (2026-07-18, 사용자 승인)**: RL 행동공간에서 전략적 WAIT **기본 제외** — §5.

## 1. WAIT은 두 종류다

| 종류 | 정의 | 근거 |
|---|---|---|
| **강제 WAIT(양보)** | 지금 할 수 있는 실작업이 없음 — `NO_FEASIBLE`(할 일 자체 없음) 또는 `LOST_CONTENTION`(다른 크레인이 그 작업을 선점) | **필수** |
| **전략적 WAIT** | 실작업이 가능한데도 일부러 대기 (더 나은 기회·양보) | 현재 조건 근거 없음 |

## 2. 강제 WAIT — 반드시 필요 (증명·실측)

크레인 2대가 같은/충돌하는 작업을 원하면 한 대는 반드시 양보한다(둘이 같은 작업 동시 불가).
그 진 크레인에게 줄 유효한 행동이 WAIT뿐이다 — 없으면 엔진의 결정 계약(모든 대기 크레인은
반드시 배정)이 깨져 `DECISION_COVERAGE` 위반으로 크래시한다.

**실측 (ServiceFirstSPT, seed 310000~310005, 크레인-결정 548건)**:
- WAIT 91건(16.6%) — **전부 `LOST_CONTENTION`** (경합 패배). 전략적 WAIT 0건(고정규칙은 구조상 불가).
- 이 91건은 WAIT 없으면 배정 불가능. 다중 크레인이 공유 작업을 두고 경쟁하는 한 WAIT은
  joint resolver를 성립시키는 no-op으로 필수다.
- 부수: 양보한 크레인은 다음 외생 이벤트까지 결정에서 제외(`yielded`) → 같은 시각 무한 재결정 방지.

## 3. 전략적 WAIT — 현재 조건에서 정당화 안 됨 (ablation)

JointRollout에서 "실작업 조합이 있으면 WAIT 못 고르게" 금지하고 비교(구조적 WAIT은 보존):

| seed | 원본 총비용 | 전략WAIT 금지 | 차이 |
|---|---:|---:|---:|
| 310000 | 62.93 | 61.68 | −1.25 |
| 310001 | 74.63 | 74.22 | −0.41 |
| 310002 | 60.16 | 60.66 | +0.50 |
| 310003 | 82.53 | 68.38 | **−14.15** |
| 310004 | 71.00 | 68.95 | −2.05 |
| 310005 | 80.39 | 72.29 | **−8.10** |

→ **6개 중 5개에서 전략적 WAIT 금지가 더 낫다 (평균 −4.24)**. 대기시간도 대부분 감소.
지금 rollout baseline은 전략적 대기를 오용하며(YR-051 라이브락과 같은 뿌리 — rollout이 대기를
과대평가), 막는 게 이득이다. 비용함수는 이미 대기에 벌점(yielded 크레인당 interference)을 물린다.

## 4. 범위와 보류된 설계 결정

전략적 WAIT이 **이론상 쓸모 있는** 상황은 현재 시나리오에 비활성이다:
- **본선 긴급**: 곧 나올 긴급 본선 작업 위해 지금 긴 외부작업을 미룸 — λ_vessel=1.0 중립이라 본선
  압박 0 (YR-041에서 활성화).
- **레인/크레인 혼잡**: 교착 회피 양보 — 평균조건이라 혼잡 약함 (YR-021 고부하에서).

따라서 "전략적 WAIT 무용"이 아니라 "**현재 평균조건·λ=1.0에서 실증 이득 0, baseline에 −4.24 독**"이
정확하다. 강제 WAIT(양보)은 어느 결정에서도 항상 유지된다.

## 5. 결정 (2026-07-18, 사용자 승인) — RL 행동공간에서 기본 제외

- **판정 데이터 (YR-045 신규 seed 대역)**: 전 정책·전 arm 에서 금지가 같거나 우월 —
  JR −0.8~−1.6, RL −4.0~−15.7. RL 미완주 6건 전부 허용 조건에서 발생 (오용 확정).
- **구현**: `dqn_learner.run_episode` 의 `forbid_strategic_wait` 기본값 False→**True**.
  RL 학습·평가 경로(candidate_dqn_experiment 포함)가 기본으로 제외를 상속.
  `False` 는 ablation 재현 전용. 회귀 가드: `test_strategic_wait_excluded_by_default_yr052`.
- **범위**: 결정·구현 모두 **RL 행동공간 한정**. JR/BEAM baseline 의 flag 는 opt-in 유지 —
  baseline 정의 변경은 locked 결과(YR-045) 재현성을 깨므로 본 결정 범위 밖.
  구조적 WAIT(경합 양보·NO_FEASIBLE)는 계약상 어느 경로에서도 보존 (§2).
- **재개 조건**: §4 의 비활성 상황이 활성화되면 (λ_vessel 동적 — YR-041, 고부하 — YR-021)
  전략적 WAIT 재도입을 해당 실험의 ablation 으로 재검토한다.
